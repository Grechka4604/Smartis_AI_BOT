Есть 3 функции конфигуратора:

  • Управлять группировками 
  • Создание категорий кампаний
  • Настройка телефонных номеров

Управление группировками

Управление группировками позволяет создать группировку CRM данных по выбранному полю.
Это может быть полезным при построении кастомных отчетов, где необходимо определенным сгруппировать данные по нескольким показателям, даже если искомое поле есть не во всех CRM сущностях, которые нужно представить в отчете.

Можно создать группировку по любому полю срм, по любой сущности. 

Алгоритм создания группировки

В качестве примера разберем создание группировки по полю "Полное имя ответственного", которая позволит строить отчет по всем CRM сущностям клиента от интереса до ДКП.
Список сущностей для отчета: Интерес, Активности, Визиты, ДКП.

  1. В конфигураторе выбираем первый показатель - Интерес.
    Далее задаем название новой группировки - "Полное имя ответственного"
    Группировать по - выбираем Коннектор, указываем название искомого поля "Полное имя ответственного". Сохраняем

Теперь группировка доступна для показателя Интересы. Но нам необходима группировка, применимая и к другим показателям. Поэтому переходим на следующий шаг.

В конфигураторе есть окно Связанная сущность (опционально) - если вы хотите создать группировку по полю в выбранном показателе, то на это окно нажимать не нужно. То есть, как в примере - мы создаем группировку для сущности Интерес по полю, которое есть в данной сущности, поэтому мы не выбираем еще и связанную сущность.

бывают ситуации, когда нам нужно создать группировку для сущности по полю, которого в этой сущности нет, но оно есть в связанном срм элементе. Тогда мы кликаем на окно Связанная сущность (опционально) и выбираем эту сущность

Рассмотрим пример: 

  • нам нужно создать группировку для показателя Звонки по полю, которое есть в сущности Контакты
  • выбираем показатель "Звонки" 
  • выбираем коннектор
  • далее кликаем на окно связанной сущности: в выпадающем списке нажимаем на название нашей сущности Звонки 
  • далее откроется еще один список. В нем нужно выбрать сущность, в которой хранится нужное нам поле - это сущность Контакты. Сущность звонки еще раз выбирать не нужно!
  • Далее мы кликаем на окно "поле" и выбираем нужное поле


  • Если у вас сложная связь между сущностями (например, нам нужно поле для звонков, которое есть в сущности Встречи). Мы видим, что у Звонков нет напрямую связи с Встречами, но эта связь есть у сущности Контакты. Контакты связаны со звонками. Тогда мы настроим связь, выбрав Контакты и далее - Встречи. После чего укажем в окне Поле название нужного поля из Встреч.

Итоговая настройка будет выглядеть так:

2. В конфигураторе выбираем другой показатель - Активности.
В списке группировок уже не создаем новую, а выбираем нашу - "Полное имя ответственного"
Группировать по - выбираем Коннектор, указываем название искомого поля "Полное имя ответственного". Сохраняем

3. В конфигураторе выбираем следующий показатель - Визиты.
В списке группировок снова выбираем нашу - "Полное имя ответственного"
Группировать по - выбираем Коннектор
Далее мы уже не можем просто указать название искомого поля по имени ответственного, так как данное поле не используется в сущности Визиты.
Поэтому нужно воспользоваться полем связанной сущности.
Строим связку: Визиты - Интерес (так как в сущности Интерес нужное поле есть).
После того, как указали связанную сущность, можно выбрать поле "Полное имя ответственного" - в данном случае поиск выполняется уже не в сущности Визиты, а в сущности Интерес. Сохраняем



4. В конфигураторе выбираем следующий показатель - Договоры купли-продажи.
В списке группировок снова выбираем нашу - "Полное имя ответственного"
Группировать по - выбираем Коннектор
Далее нужно снова воспользоваться полем связанной сущности.
Строим связку: Договоры купли-продажи - Интерес (так как в сущности Интерес нужное поле есть).
После того, как указали связанную сущность, можно выбрать поле "Полное имя ответственного". Сохраняем

Теперь в конструкторе отчетов можно проверить, как работает группировка - строим отчет по выбранным показателям, выбираем нашу созданную группировку. Видим, что все работает корректно.

  • Если при добавлении показателей группировка из списка исчезает, нужно убедиться, что она была активирована при создании в конфигураторе.



Создание категорий кампаний

В Smartis с помощью конфигуратора есть возможность создавать кастомные группировки по категориям кампаний. Категория - признак, который указан в названии рекламной кампании.

Для каких данный применяется функционал:

  • все данные интегрированных рекламных кабинетов (тк у них есть кампании с названиями, в которых есть признаки для группировки)
  • данные по обращениям (так как у них тоже есть информация о названии РК)
  • CRM данные (так как кампания, с которой пришли CRM конверсии определяются в Smartis с помощью атрибуции)
  • данные о посещении сайта

Данные, которые импортировали из файлов (расход по площадкам, факт по площадкам, плановые значения по звонкам/лидам) всегда будут попадать в категорию "Остальное", тк у этих данных нет признака рекламной кампании, а значит разделить их в такой категории нельзя)

Категории кампаний могут быть любыми, это общее название. Примеры категорий:

  • бренд и не-бренд
  • поиск и РСЯ
  • типы кампаний: гео, конкуренты и другие
  • часто клиенты используют группировки категорий при АВ-тестировании (в таком случае в нейминг кампаний добавляют признаки, по которым можно создать группировку)

Категории кампаний выделяются на основе параметров, которые указываются в названии рекламной кампании, которое приходит из рекламного кабинета.

Чтобы воспользоваться функционалом, откройте Администрирование→ Конфигуратор → Группировки категорий кампаний

Как создать новую группировку

  1. В окне "группировка" выберите "Новая группировка" и введите ее название
  2. Переведите ее в статус "Активно"
  3. Добавьте категории

  4. Как добавлять категории:
    1. Ввести название (будет отображаться в отчете)
    2. Выбор условия 
      • Содержит
      • Соответствует регулярному выражению
      • Не соответствует регулярному выражению

      • если требуется в группировке использовать одновременно условие "не содержит" (например, .*tgb.*) и "содержит" (например,.*paral.*), то требуется последовательно добавить несколько таких условий, которые будут собирать в себя те данные, которые содержаться не должны и тем самым исключать их.

        что пишем в условиях группировки (скриншот):

        первым условием выбираем "соответствует регулярному выражению" и указываем, что содержать НЕ должно (так мы отсеиваем то, что не должно содержаться)

        вторую условием выбираем "соответствует регулярному выражению" и указываем, что должно содержать 



    3. Прописать значения (указывайте такие значения, которые реально встречаются в нейминге кампаний у клиента. Постарайтесь учесть все варианты - на английском и на кириллице. Это поможет корректно группировать кампании, если клиент изменит написание признака). Значения прописываются либо регулярным выражением, либо обычным. Как прописывать - зависит от выбранного условия.

      Созданные категории можно удалить или поменять местами (чтобы в отчете они выводились в нужном порядке).


Если название кампании удовлетворяет нескольким указанным категориям, то она попадет в категорию, находящуюся первой в списке. Если кампания не подходит ни под одну категорию, она попадет в строку "Остальное"


Добавление категорий зависит от того, по каким признаком вы хотите группировать кампании. На скрине выше пример для группировки по признаку "Бренд" (когда хотим оценить кампании в разрезе "Брендовые кампании" и "кампании не брендовые".

Категории кампаний в отчетах

Если категория активна, то в отчете она отобразится в окне "Группировка" в конструкторе отчетов с пометкой "Пользовательская".


Данные в отчете будут выглядеть так:

В каждую категорию попадут данные, которые удовлетворяют условиям, которые мы прописали для каждой категории,
Из каждой категории можно провалиться в детализацию отчета и посмотреть, какие кампании относятся к категории:

Из кампании доступна детализация до объявлений/ключевых слов (если это кампания из Яндекса - до объявлений + ключевых слов, для кампаний из MyTarget и VK - до объявлений).

Обратите внимание, если вы строите отчет по всему трафику и выбираете показатели с разными источниками данных (например, CRM, рекламный кабинет, колл-трекинг, импортированные данные), а также показатели, которые созданы по формуле и включают в себя разные типы данных (как Бюджет факт, который состоит из импортированных расходов и расходов из рекламных кабинетов) - группировка будет работать. Но при этом какие-то данные будут разделяться на группировки, а какие-то нет.

Для показателей с импортированными данными и данными из интегрированных кабинетов группировка разложит на категории только данные, полученные из рекламных кабинетов. Импортированных данные попадут в "остальное". При этом, данные из других показателей могут разложиться на категории полностью.
