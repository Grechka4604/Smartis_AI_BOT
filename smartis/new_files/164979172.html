У нас есть проверенные решения для запуска этой механики, по которым мы работаем и по которым запуск достаточно быстрый.

Суть:

1) Для каждого клиента в CRM генериться специальная ссылка в следующем виде https://alia.moscow/?smid=05a617f2 (генерится на стороне клиента - не Смартис генерит). А именно - Хэшируются номера и джойнятся к ссылке

2) Параметр (идентификатор) smid = 05a617f2 - это закодированный номер телефона 79152823275 (в этом примере) методом хэширования.   ! Метод хэширования на стороне клиента требуемый и поддерживаемый со стороны Smartis - crc32b


3) Каждому клиенту из серой зоны (и не только можно всем) создаётся такая ссылка и направляется в рассылке/сообщении

4) При переходе клиента по такой ссылке мы считывает smid, распознаём и связываем с crm историей клиента + подтягиваем в cjm все сессии клиента, которые ранее были у него (если были)

5) Менеджер проверяет начали ли приходить посетители с smid

6) Далее Нужно по Шаблонной задаче на нашей стороне включить анализ по Smid

7) Проверить что СРМ данные связались с посетителями со smid
