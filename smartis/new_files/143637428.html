Знакомство с интерфейсом, основные разделы и их назначение


  1. Чаты и соцсети:

В этом разделе операторы работают с обращениями из чатов/мессенджеров, если такая услуга подключена у клиента. Так же, в этом разделе, настраивается интеграция с подрядчиком Blinger, с помощью которого мы предоставляем сервис чатов/мессенджеров. Ни в коем случае нельзя думать о том, что мы "просто" перепродаём технологию Blinger, нет! Наш сервис берёт часть их технологий, накладывает сверху своих технологии, в итоге получается продукт Smartis мессенджеры. 


        2. Общие настройки:

Информация о Вас - в этом разделе содержится скрипт, который необходимо устанавливать на сайте клиента. У каждого клиента скрипт индивидуальный! Для каждого клиента формируется свой уникальный token в скрипте, за счёт которого происходит обмен информацией.  

В этом разделе так же содержится информация об открытых доступах API в рамках этого клиента (кабинета).



        2.1 Пользователи

В этом разделе создаются/редактируются пользователи. Существуют три варианта пользователей: оператор - создается для работы с обращениями из чатов\мессенджеров; аналитик - создается обычно для рекламных агентств, имеет ограниченный набор прав; администратор - имеет полный доступ. Для каждого пользователя можно настраивать доступ к определенным доменам, настраивать определённый набор оповещений или отключить их вообще. Если необходимо "скинуть" пароль, то есть создать новый пароль для ранее созданного пользователя - просто вписываете новый пароль в поля "новый пароль" и "Повторите пароль". 



        2.2 IP-фильтр 

В этом разделе можно заблокировать нехорошего человека по IP-адресу. Бывают ситуации, когда какой-то человек начинает спамить (оставлять заявки на разные номера телефонов\один номер телефона) - в таком случае выясняем IP-адрес человека и заносим его в этом разделе. 



        2.3 Чёрный список 

Точно такой же принцип, как и в разделе IP-фильтра, только вносится номер телефона. 



        2.4 Работа с тегами 

В этом разделе можно создавать новые теги, которые в последствии можно использовать для заявок. 




        2.5 История

Здесь вы можете найти записи о любых изменениях настроек Smartis виджеты.



        3. Аналитика: 

Экспорт и фильтры - в этом разделе можно посмотреть все заявки, которые были в рамках этого кабинета\клиента. Можно отсортировать заявки по различным фильтрам: дате, сайту, источнику, номеру телефона посетителя, по типу заявки и т.д. Так же в этом разделе можно сделать выгрузку заявок, в правом верхнем углу находятся кнопки для создания выгрузки в определённом формате. 



в колонке ID - содержится ID заявки;

в следующей колонке отображается тип устройства;

в колонке Статус - отображается статус, тип и лог заявки. При нажатии на слова "лог заявки" открывается её лог, по которому можно понять, как обрабатывалась заявка;



в колонке Клиент (сайт) - прописывается название кабинета и название сайта;

в колонке Запись разговора - можно послушать запись разговора, скачать запись разговора, а так же раскрыть все попытки прозвона (если их было больше 1). 




в колонке Телефон содержится номер телефона посетителя;

в колонке email содержится email посетителя;

в колонке Телефон оператора содержится номер телефона оператора;

в колонке Источник содержится информация об рекламном источнике и utm метках (если таковые были). Если в колонке источник будет находится фиолетовый прямоугольник с надписью внутри - значит, эта заявка была оставлена через встроенную форму обратной связи на сайте, которую обрабатывает наш сервис 



В колонке Дата заявки содержится дата и время создания заявки. Если посетитель выбрал удобное время для перезвона - то рядом будет жирным шрифтом приписка У.В. (удобное время). Если заявка уже обработана - то ниже будет написана дата и время последней\удачной попытки прозвона. 



В колонке Действия содержится тег заявки, тип заявки (целевая/нецелевая), карточка обращения. При нажатии на слова Карточка обращения откроется окно, в котором будет содержатся более детальная информация о заявке 



Раздел сводная статистика в данный момент не используется. 


        4. После идёт список доменов, которые заведены в рамках этого кабинета/клиента. 

При нажатии на название домена - под доменом открываются три пункта:



Настройки - в этом разделе происходит настройка домена;

будет расписано в разделе 1.4 Настройки домена


Акции - в этом разделе настраиваем мобильные акции

Документация по настройке мобильных акций - https://docs.uiscom.ru/pages/viewpage.action?pageId=138906890


А/В Тесты - в этом разделе происходит настройка А/В тестов. 

Документация по настройке A/B тестов - https://docs.uiscom.ru/pages/viewpage.action?pageId=138904066


        5. Подключить еще домен

В этом разделе происходит создание нового домена.

Если домен прописан кириллицей, например домен.рф - то его нужно преобразовать в кодировку punycode, с помощью онлайн-сервиса.

Таких сервисов много, напишу сюда один из популярных - https://www.punycoder.com/ 


Домен.рф с помощью кодировки преобразовывается в xn--d1acufc.xn--p1ai. 

Именно xn--d1acufc.xn--p1ai нужно добавлять в качестве домена в личном кабинете Смартис Виджетов!!
