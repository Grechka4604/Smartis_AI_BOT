    Оцениваем первичный результат июля

Планы на 2 этап

1. Во вложении ДС на подпись и два счета на оплату, закрывающие документы (август) по первому счету вышлю позже

2. Утверждаем формат данных, по сделкам, контактам, обращениям Автоспот формат утвержден, после созвона со Стасом сложилось понимание, что выгрузка из МУВТ требует доработки (поле источник информации и нужно что-то еще), так как сейчас нет возможности понять что за источник обращения по НА, если нет id вебхуки. Утверждаем формат выгрузки, присылаю образцы, затем загружаем период за 90 дней

3. Доделываем список отчетов

4. Дорабатываем МА, чтобы учитывать не общее перетекание, а только в рамках марки, требуется доработка на нашей стороне по разделению классифайдов по маркам

5 Задача по автоматизации остальных форм становится не актуальной, если Кипер на своей стороне реализует доработку. Коротко суть в том, что Кипер должен использовать метод smAnalytics.getSessionId(); при отправке заявки и сохранять в базе наш id и при передаче данных в каждом обращении отдавать нам наш id (не главная задача)
