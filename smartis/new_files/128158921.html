Передача спам-звонков в срм
  • найти способ, который поможет исключить передачу спам-звонков в срм
  • исключить поднятие карточки на спам-звонок
  • проанализировать пустые звонки без тега спам
  • На данный момент поднятие карточки происходит по нашему событию "Входящий звонок на АТС", необходимо перенастроить поднятие карточки по событию "Исходящее плечо". Даша Рахвалова должна проверить, возможно ли с помощью "исходящего плеча" передавать необходимые параметры в карточку срм, по аналогии с параметрами "Входящий звонок на АТС". Если завязать поднятие карточки на "Исходящее плечо", то звонки не будут передаваться
  • ТИСА разработали приложение к коннектору для нашей телефонии, которое работает на "Исходящем плече", но на данный момент Левел использует приложение для Простых звонков (Манго), который работает по другому принципу, поэтому спам-звонки передаются в срм. Переход на нашу телефонию и на приложение ТИСЫ для нашей телефонии поможет эту проблему исключить. Также Армен сказал, что они смогут скорректировать данное приложение в случае, если будет повторяться передача спам-звонков в срм
  • Даша Рахвалова на событие "Записанный разговор" поставила фильтр, исключающий передачу звонков с тегом "спам", чтобы не загружать срм пустыми звонками
  • Владимир в срм видит аналогичные спамным звонкам пустые звонки, но они не содержат тег "спам". Владимир должен прислать список примеров, чтобы мы смогли выявить источник данных звонков и также исключить их передачу в срм. Есть предположение, что это звонки от абонентов из черного списка
  • На данный момент пустые карточки собираются в срм, Владимир их вручную прочищает. За день порядка 50 карточек
  • После окончательного теста FMC планируется переход на нашу телефонию. Ориентировочно - 10.08 команда Левела будет решать вопрос по срокам перехода на нашу телефонию
  • Переход на нашу телефонию потребует настройки софтфона для каждого менеджера и установки новых приложений в срм на каждом компьютере
