Команда Smartis передает задачи на реализацию команде Проектных решений и Продукту. 

Для управлением разработкой организована еженедельная приоритезация задач. 


Цель: согласовать задачи и результат по ним, которые берет в работу команда Проектных решений

Участники процесса:  Руководитель направления

Инициатор :Руководитель направления

Инфо поле: Спринт ОПР Premium, Спринт в Продукте

KPI процесса:

  • Регулярность - 1 раз в неделю по понедельникам

Результат: 

  1. Актуализированные статусы задач по Смартис
  2. Согласованы задачи и результат работы по ним, согласно статусу
  3. Разобрана обратная связь по реализации или анализу задач

Как контролируем: Актуальная вкладка в документе Спринт ОПР Premium / Спринт в продукте

Владелец процесса: Руководитель направления

Статус процесса: эксплуатация.


Описание процесса:

  1. В документе Спринт ОПР Premium или Спринт в Продукте создать новую вкладку с датой начала спринта.
  2. Актуализировать задачи, начиная с последнего этапа - Приемка. Заходим в каждую карточку, проверяем комментарий и следующий шаг. 
  3. Проверяем выполнение задач, взятых в завершенный спринт. По комментариям в карточке и/или по обратной связи от команды клиента.
    1. Задачи, переведенные в статус Закрыто удаляем из вкладки нового спринта. Удалить строку. 
    2. Там, где нужно что-то обсудить пишем в комментарии вопрос к ОПР. 
  4. Добавляем новые задачи для ОПР. Дашборд Бизнес Анализ
    1. Проверить, что нет задач Премиума/Смартиса, которые попали в Lite. Просмотреть все созданные задачи. 
    2. Проверить, что нет подвисших багов
  5. Проставляем приоритет для задач в столбце Приоритет. 10 - обязательно нужно взять, 1 - можно не закладывать ресурс.  Примерный объем спринта -  2-3 задачи для анализа, 1 задача написание для ТЗ, 1 задача для разработки.
    1. Помечаем зеленым задачи, которые хотим отдать в следующий спринт
    2. Отписываем в чат регулярной встречи о том, что спринт сформирован. 
  6. Проводим регулярную встречу с ОПР (Валя Жехова), понедельник 13:00 или с продуктов понедельник 15:00
    1. По согласованным в спринт задачу проставляем дату начала спринта.
