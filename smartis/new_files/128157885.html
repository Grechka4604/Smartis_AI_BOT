Цель - сделать единый инструмент для фиксации задач связанны с допродажами действующим клиентам. 

Требования к Проектам: 

  • тема "Допродажа. [Название компонента]
  • карточка клиента → наименование клиента, которому непосредственно будет подключен компонент; 
  • срок исполнения → предпологаемая дата завершения текущего этапа; 
  • исполнитель → сотрудник, который отвечает за сделку; 
  • сумма →  стоимость сделки
  • описание → см. ниже 

Описание должно содержать обязательно следующие атрибуты: 

  1. Компонент;  
  2. Потребность (Пейн) в разрезе конкретного клиента;
  3. ЛПР по компоненту; 
  4. План проработки; 
  5. Планируемая сумма сделки; 


Далее в комментариях при выполнении задачи прописывается: 

  • что было сделано (что стало результатом); 
  • какие следующие шаги запланированы (когда пишем следующие шаги исходим из конечной цели, они должны привести нас к выполнению поставленной задачи);
