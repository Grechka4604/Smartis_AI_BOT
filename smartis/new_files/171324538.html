Дописать: формы, лиды (что это и зачем, как настроить и тд). также есть синхроны по веб хукам - надо уточнить что именно мы забираем через веб хук и как это влияет на получение данных и достоверность.

Основная информация

MyTarget - рекламный сервис, с помощью которого клиенты размещают рекламу в соц.сетях (ВК, Одноклассники), а также на других ресурсах mail.ru. Какие форматы доступны для запуска через myTarget: Баннер — картинка, анимация или видео с текстом и кликабельной кнопкой. У нас нативная интеграция с MyTarget , которая позволяет получать и отображать в Smartis данные о кампаниях, объявлениях из рекламного кабинета.


Если у вашего клиента есть данные в старом кабинете Майтаргет, к которому нет доступа, но при этом данные клиенту нужно получать в смартис в ретро периодах, то это можно сделать через импорт данных из старого кабинета.
Что для этого нужно:

  • получить от клиенты выгрузки со всеми необходимыми показателями (расход, клики, показы)
  • создать импортируемые показатели по кликам и показам
  • загрузить данные по Майтаргет: 
    • клики - в импортируемый показатель Клики
    • показы - в импортируемый показатель Показы
    • расход - в импортируемый показатель Бюджет медиапланов (факт) в площадку Майтаргет
  • для того, чтобы старые данные отображались в отчетах, необходимо создать показатели по кликам, расходу и по показам, который включает импортированные вручную данные + данные которые поступают из Майтаргет по апи интеграции. Чтобы это сделать, создается новый показатель который строится по формуле Клики (импорт) + Клики Майтаргет (с показами и расходами аналогично)
  • эти показатели нужно добавить во все отчеты, где нужно отслеживать данные по Майтаргет.



Какие данные по MyTarget  доступны в Smartis:

  • показы
  • клики
  • расход (без НДС)
  • лиды

Расходы по MyTarget, которые мы получаем из интеграции, всегда загружаются в системную площадку MyTarget соц.сети (id 41), канал = Реклама в соц.сетях

Данные по обращениям и сессиям, которые поступили с рекламы в MyTarget также привязываются к этой площадке по умолчанию.

Важно! Нельзя удалять эту площадку и переносить данные в другую площадку, даже если клиенты очень сильно просят переименовать площадку. 

Подключение интеграции

Структура кабинетов MyTarget у клиентов может быть разная. Вот возможные варианты:

  • к одному логину в MyTarget относится несколько рекламных кабинетов
    Логин: ИвановИван@mail.ru, кабинеты доступные логину: ЖК 1 (рекламные кампании в кабинете по ЖК 1), ЖК 2 (рекламные кампании в кабинете по ЖК 2),
    ЖК 3 (рекламные кампании в кабинете по ЖК 3)

  • к одному логину в MyTarget относится 1 логин, и в нем ведутся рекламные кампании по 1 объекту/сайту
    Логин: ИвановИван@mail.ru, кабинеты доступные логину: ЖК 1, рекламные кампании в кабинете по ЖК 1

  • к одному логину в MyTarget относится 1 логин, и в нем ведутся рекламные кампании по нескольким объектам/сайтам
    Логин: ИвановИван@mail.ru, кабинеты доступные логину: "Общий", рекламные кампании в кабинете по ЖК 1, ЖК 2, ЖК 3

В любом из этих случаев мы для подключения интеграции используем логин с доступами в кабинет/кабинеты, и далее привязываем к сайту либо каждый логин из общего кабинета отдельно, либо единственный кабинет с логином к 1 сайту, либо если реклама в кабинете по нескольким объектам - 1 логин ко всем нужным сайтам + прописываем условия привязки по названиям кампаний.

Подключение интеграции

  1. Получить у клиента доступ в кабинет MyTarget, либо если подключает сам клиент не нужно. Для подключения интеграции со Smartis достаточно прав на чтение. 
  2. В разделе Интеграции в Smartis выбрать интеграцию с MyTarget в разделе "Доступные для подключения" и нажать "подключить"
  3. Откроется страница авторизации - нужно ввести свои данные и пароль к кабинету MyTarget
  4. После ввода логина и пароля к кабинету MyTarget, отобразится окно с информацией о том, к каким данным Smartis получит доступ. На этом этапе нужно выбрать, к какому кабинету предоставить доступ (если подключаете через логин, у которого доступы к нескольким кабинетам - как в примере на скрине).
  5. Если вам нужно подключить сразу все кабинеты - выбирайте "предоставить доступ к аккаунту ".....". Если подключить каждый кабинет по отдельности или какой-то 1 из них - то "предоставить доступ к аккаунту клиента". При выборе 2 варианта откроется список аккаунтов. Выбрать можно только 1 из списка.
    Если клиент планирует отправлять сегменты Smartis.CDP в MyTarget, то нужно подключать каждый логин отдельно (2 вариант)

  6. После выбора нужного логина/кабинета, нажмите "разрешить доступ". После этого интеграция подключится и отобразится в разделе "Подключенные интеграции". Если вам нужно подключить насколько отдельных логинов, то шаги 1-6 нужно выполнить столько раз, сколько логинов нужно подключить


Когда интеграция подключена, по ней начинается автоматически обновление данных. По умолчанию мы забираем данные из интеграции за за последний месяц по текущий день. Если нужно обновить больший период, воспользуйтесь ручным обновлением данных (справка тут 9. Администрирование: Мониторинг системы )

Если интеграция отключилась и по ней отображается ошибка: Ошибка авторизации - это значит, что владелец кабинета сменил пароль или логин для доступа к данным кабинета. Если интеграцию подключал менеджер Smartis, нужно получить актуальные данные для авторизации от клиента. Если подключал клиент - нужно чтобы шаги ниже выполнил клиент.

Привязка интеграции к сайту

Данные по интеграции нельзя просматривать в отчете, пока интеграция не привязана к сайтам в личном кабинете. Интеграцию нужно привязать ко всем сайтам, по которым ведется реклама из данного рекламного кабинета.
Например, у вас рекламный кабинет MyTarget, в котором ведут рекламу по ЖК #1 - значит, ее нужно привязать к сайту этого ЖК.
Если по 1 ЖК у вас несколько сайтов, но все они входят в 1 объект в Smartis, интеграцию достаточно привязать только к 1 из этих сайтов (любому).

  1. Зайти в раздел Администрирование → Проекты и сайты → Сайты и выбрать нужный сайт
  2. По клику на название сайта откроется меню для привязки интеграций. В разделе Интеграции доступные для привязки найдите нужную и нажмите "Привязать с логином". Если подключали весь кабинет - будет много логинов в списке. 
  3. Если подключили 1 логин - то в списке отобразится только он. 
  4. В выпадающем списке логинов выберите нужный и кликните на него (выбрать модно только 1 логин единовременно). После выбора логина страница обновится и привязанная интеграция отобразится в разделе Привязанные к сайту интеграции
  5. Для каждой интеграции можно настроить условия привязки - например, если вам нужно получать из интеграции для выбранного сайта только определенные кампании, вы можете прописать такие условия. Для этого кликните на иконку шестеренки на плашке интеграции. Здесь можно указать условия для привязки к сайту определенных рекламных кампаний в виде регулярного выражения. 
    Такая настройка используется, если в рекламном кабинете рекламные кампании по нескольким объектам ЖК или есть общие рекламные кампании по всему бренду клиента и их нужно просматривать в Smartis в отдельном объекте.

    Наименование кампании соответствует регулярному выражению - если в названии рекламной кампании в MyTarget указаны названия ЖК, то можно прописать условие (в виде регулярного выражения) в этом поле (учтите все возможные варианты написания - на русском и английском языке, сокращения и прочее).

    Наименование кампании не соответствует регулярному выражению - если нужно привязать все рекламные кампании, кроме определенных - заполнить в виде регулярного выражения это поле 

    Наименование формы соответствуют регулярному выражению - у каждой формы для сбора лидов тоже есть названия, если нужно отфильтровать по конкретным названиям форм, то вписываем через регулярку названия. 

  6. Если вам нужно отвязать интеграцию от сайта, кликните на иконку как на скриншоте.

Показатели MyTarget

После привязки интеграции к сайту, при выборе проекта или объекта, в которых этот сайт состоит, или сайта отдельно в конструкторе отчетов, у вас в списке показателей появятся системные показатели MyTarget. Они будут находится в категории "MyTarget" (создается тоже автоматически). Показатели системные, их нельзя изменить. Но на их основе можно создать пользовательские показатели с помощью формул (про них справка тут 1. Показатели и их настройка )

По данным можно построить отчет в группировках, которые доступны при выборе показателей. Площадка, по которой будут строиться данные - всегда будет только MyTarget даже если у вас выбран канал "Весь трафик". Если будет выбран канал не Весь трафик и НЕ Реклама в соц.сетях, то в отчете отобразится 0.

Данные MyTarget по кликам, показам и расходам невозможно посмотреть в виде детализированных данных, поэтому для просмотра отчетов по этим данным MyTarget доступны все виды визуализации кроме "Исходные данные".

Детализация доступна только по MyTarget Лиды - так как лиды это обращения. Они такие же, как данные по заявкам у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.

Детализация источников по MyTarget (проваливание)


из отчета по каналам или по площадкам по данным MyTarget доступны такие уровни детализации:

  • Категории кампаний (пользовательские группы, настраиваются тут 9. Администрирование: Конфигуратор )
  • Кампании - открывает срез по кампаниям
  • Объявления - открывает срез по объявлениям. Отчет по объявлениям можно смотреть не только по данным MyTarget, но и по данным сервисов обращений и CRM. То есть можно посмотреть статистику как отработало и сколько обращений, сделок и других конверсий поступило с каждого объявления MyTarget.


    Из площадки можно провалиться в кампании и категории кампаний. В объявления можно провалиться только кампании. Если нужен отчет по всем кампаниям MyTarget по объявлениям, воспользуйтесь группировкой "Объявления".
