Основная информация

Уведомления показывают разделы, в которых есть ошибки/алерты, на которые необходимо обратить внимание администраторам личного кабинета клиента. Они не кликабельны.

Уведомления могут видеть только суперадмины и сотрудники смартис.


В каких разделах есть уведомления:

  1. Проекты и сайты

  2. Интеграции

  3. Администрирование (показывает суммарное кол-во "уведомлений" по всем подразделам, которые находятся во вкладке "Администрирование". То есть по Проектам и сайтам, Интеграциям, Мониторингу)

  4. Импорт

  5. Мониторинг системы

В каждом ЛК отображаются уведомления только по этому ЛК


Как работают уведомления:

1. Проекты и сайты: элемент появляется и показывает кол-во сайтов в разделе "Потенциальные сайты" за последние 30 дней (если появляется новый потенциальный сайт за последние 30 дней - элемент показывает 1). При переходе в раздел проектов и сайтов элемент располагается на вкладке "Потенциальные сайты" . Элемент пропадает, если сайтов на данной вкладке нет (то есть их либо удалили, либо добавили в проект). также кол-во "уведомлений" изменяется, если сайт удалили или добавили в проект (то есть кол-во сайтов на этой вкладке уменьшилось)


2. Интеграции - элемент появляется и показывает кол-во интеграций с ошибками. Элементы исчезают после того, как проблема устранена. То есть, если пользователь обновил токен/доступы и у интеграции больше нет ошибок, то сообщение об ошибке пропадает. Также, если удалили интеграцию, по которой были ошибки - элемента не должно быть. Кол-во "уведомлений" изменяется, если кол-во интеграций с ошибкой стало меньше (было 3, осталась 1).
Уведомления не появляются:

  • если интеграция удалена
  • если интеграция архивирована
  • если у интеграции больше 1 ошибки, то уведомление будет все равно 1 - по количеству интеграций с ошибками.


3. Администрирование - элемент появляется и показывает кол-во уведомлений из других подразделов (Сумма уведомлений из Интеграций и Проектов и сайтов). Кол-во "уведомлений" изменяется в зависимости от изменения по каждому подразделу в котором эти уведомления появились.

4. Импорт - элемент появляется если в импорте произошли ошибки. Отображает кол-во импортов (ID импортов) за последние 30 дней, которые завершились с ошибкой. При клике на вкладку Импорт данных элемент отображается на вкладке История импорта.
Уведомления не отображаются по строкам импорта, для которых не доступны действия (удалить, обновить). Это связано с тем, что ошибку в таком импорте нельзя исправить - можно только загрузить данные заново (что и делают при получении ошибки).

элемент исчезает, если строк с ошибкой импорта больше нет (файл либо удалили либо перезагрузили). Кол-во "уведомлений" на элементе изменяется, если кол-во строк (id импорта) с ошибкой изменилось.



5. Мониторинг системы: элемент появляется, когда у клиента по какой-либо интеграции получаем ответ "Не получили никакую информацию о расходах из АПИ". при этом у интеграции может быть высокая достоверность.

Кол-во уведомлений = кол-во интеграций, по которым такую ошибку получаем

в Самом мониторинге уведомления отображаются:

  1. на строке сервиса (суммирует кол-во уведомлений по интеграциям с этим сервисом)
  2. на строку интеграции при проваливании

Пример: За 1 ноября по интеграции Х была сущность, по которой пришла нужная нам ошибка "Не получили никакую информацию о расходах из АПИ". Если за последующие дни не приходило данных по этой сущности, то мы отображаем баджик. Если по сущности стали приходить данные, мы не отображаем баджик. И даже если мы смотрим мониторинг за 10-15 ноября, мы всё равно будем учитывать логику, описанную выше. 

Элемент исчезает, если:

  1. интеграцию отключили

  2. интеграцию удалили

  3. по интеграции начали поступать данные из АПИ
