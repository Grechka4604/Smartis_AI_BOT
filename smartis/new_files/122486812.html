Возможности

Решение работает для облачного сервиса Microsoft Dynamics 365 и для установленных на собственных серверах (On Premise). 

  1. Поднятие карточки звонка в Microsoft Dynamics CRM при поступлении входящего звонка от клиента. 

  2. Исходящий звонок по клику из Microsoft Dynamics CRM. 

  3. Передача параметров о звонке из коллтрекинга и телефонии:

    1. Источник звонка (рекламная кампания, UTM метки)

    2. АОН звонящего,

    3. Сценарий

    4. И другие

  4. При трансфере на другого сотрудника карточка звонка всплывает у сотрудника, на кого производится трансфер. 

  5. Ссылки на аудиозаписи разговора клиента в карточке Microsoft Dynamics CRM.

Возможно доработать (реализуется на стороне ТИСА под клиента):

  1. Если звонок не пропущен, но аварийно прерван, то ставить тег “срыв звонка” в карточку контакта\звонка в Microsoft Dynamics CRM.

  2. Наличие статистики в АТС и Microsoft Dynamics CRM в разрезе пропущенных по количеству обратных вызовов и факту дозвона. (по ФИО клиента, № звонка, типу звонка, проекту).  

  3. Возможность автоматических задач на обратный звонок в Microsoft Dynamics CRM на конкретную дату и время и напоминание оператору. 

Техническая сторона реализации:

  • Взаимодействие между двумя системами осуществляется путём обмена http-запросами. Принимающей стороной является веб-сервис, развёрнутый на веб-сервере Заказчика, который является также коннектором между АТС и Microsoft Dynamics CRM. При наступлении какого-либо события в CoMagic, принимающей стороне отправляется http-уведомление, содержащее объект в json формате.   
  • Взаимодействие коннектора (серверная часть) с клиентской частью (рабочие места сотрудников) осуществляется по протоколу WebSocket, путем обмена сообщениями, при получении коннектором уведомления из CoMagic. 
  • Сопоставление звонка из АТС с карточкой в Microsoft Dynamics CRM по call_sesion_id;
