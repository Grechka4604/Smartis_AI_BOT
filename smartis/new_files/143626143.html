Кроме облачной версии Smartis, существует ещё и специальное решение для наших клиентов.

Мы устанавливаем приложение в их контуре (на их серверах), при этом приложение там развёрнуто в docker окружении, наподобие того, что используется в локальной разработке.

Ключевые моменты по работе с коробками

docker-compose.yml находится в директории `/var/smartisstat`

Само приложение рядом: `/var/smartisstat/www/my.smartis`

Попасть в контейнер с PHP: `docker-compose -f /var/smartisstat/docker-compose.yml exec php bash`

Все доступы, как обычно, либо в .env файле приложения, либо в .env файле рядом с docker-compose:

  • `/var/smartisstat/www/my.smartis/.env`
  • `/var/smartisstat/.env`

Помните, что в коробке своя сеть, свой адрес, и, зачастую, у нас нет возможности достучаться до них извне!


Для того, чтобы скачать файл из коробки к себе в компьютер необходимо поместить его в директорию public основного проекта, а затем открыть соответствующий адрес в браузере. Так, например, мы можем скачать файл по ссылке: https://my.smartis.bi/mix-manifest.json


Выгрузка данных

Пересылать данные по почте/скайпу/ftp/google диску и т.д. из контура коробки нельзя, это грозит лично вам огромным штрафом.

Единственная возможночть так делать - если доступ к личному кабинету или файлам доступна по обычному http


Для понимания:

Лиза как то себе на рабочую почту отправила экспортнутый файлик из личного кабинета - компания чуть не попала на 1 млн штрафа


Если фактически подобное дойдет до реального штрафа - начнут действовать внутренние подписанные вами всеми НДА компании, там предусмотрены штрафы сотрудников
