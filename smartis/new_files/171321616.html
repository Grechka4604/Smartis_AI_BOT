Основная информация

Раздел импорта данных нужен для того, чтобы импортировать в smartis любые значения, которые нельзя получить с помощью интеграций. 

Чаще всего импортируют:

  • Медипланы и факты 
  • Обращения из внешних сервисов 
  • CRM данные
  • Просмотры post-view

Способы импорта:

  • Автоимпорт через google doc
  • Ручной импорт
  • Импорт с помощью анализа корректности
  • Импорт excel файлом
  • Api импорт

Ниже в разделах описана более подробная информация, как импортировать данные по каждому из способов.

Интерфейс 

1) В первой вкладке "Занесение данных" мы совершаем в основном все действия по импорту всех данных. 

2. В анализе корректности мы тоже можем импортировать данные (только медиапланы), нов основном данный раздер нужен, чтобы внести вручную значения по площадкам по определенному сайту, т.е скорректировать уже импортированные данные и Проверить внесенные данные по площадкам.

3. История импорта необходимо для того, чтобы проверить кто, когда и какой файл импортировал в кабинет, открыть его, свериться тем, что было загружено. В основном используем для Api импорта.


4.Курс валют

Необходимо для того, чтобы верно расчитывать сумму в рублях для клиентов которые используют Google ads.

В файле есть новая площадка 

При сопоставлении значений из столбца "Площадка" нашли новую площадку, которую ранее не сопоставлялиВыбрать в меню сопоставления значений существующую площадку или нажать "+ Новая площадка". В таком случае создастся новая площадка с названием как в файле

Выберите столбец с уникальным идентификатором сущности в CRM

При импорте данных не нашли столбец, к котором указан id элемента в CRMПри импорте файла обязательно должен быть столбец с идентификаторами CRM сущностей. Если такого столбца нет и его невозможно добавить - то id можно сгенерировать вручную. Далее при загрузке файла укажите, что этот столбец содержит Уникальный идентификатор сущности в CRM

Не нашли ни одного значения для показателей

В файле обязательно должны быть столбцы со значениями показателей, например: Бюджет план, Кол-во целевых обращений и т.д. если в столбце не указаны значения, появится такая ошибка.

Проверить, как заполнены столбцы с данными. Нет ли столбцов, в которых данные не содержатся.

Не смогли получить дату из строки, проверьте формат даты.

Дата должна быть формата "Обычный текст". Smartis понимает форматы, которые указаны в выпадающем списке сопоставления колонок с датами, на текущий момент (могут расширяться)

Если дата в вашем файле отличается от принимаемых в смартис вариантов, необходимо ее изменить на подходящий формат. Также проверьте, что вы указали правильное значение из этого выпадающего списка (например, дата в файле указана в формате 01.2023, а вы выбрали в списке параметр YYYY-MM-DD)

Ошибки отображаются для всех типов импорта, в том числе для API импорта медиапланов (как у Sminex)

Примечания (сообщения) сервиса Импорта

Помогает понять, почему при импорте пропустили строки (не весь файл импорта загрузился). Такие ситуации могут быть и при ручном импорте, и при авто импорте (и импорте по API). Когда в истории импорта в столбце Строк передано больше, чем в строк импортировано, можно посмотреть причну в столбце Информация - по клику на кнопку "посмотреть"

  1. Медиаплан - Сообщение о том, что часть строк было пропущено, когда мы сами настраиваем пропуски каких-либо строк при сопоставлении. Это рабоатет для всех типов импорта из файла: ручной, автоимпорт из гугл таблицы и API импорт.

  2. CRM-данные — когда не получилось определить тип импортируемой сущности (например, если сущность в смартис пропала в связи с отключением интеграции коннектора, в которой этот тип сущности был).
    Что делать: выбрать при сопоставлении столбцов другую сущность (которая есть в коннекторе) и повторить импорт.

  3. Медиаплан — когда во всей строке нет значений для показателей или Вы указали пропустить необходимые столбцы с показателями. Ситуация когда вы при сопоставлении столбцов указали "пропустить столбец" либо в таблице изначально не были заполнены значения для показателей. например, как на скрине - показатель Бюджет план и Кол-во целевых в 5 строке пустые. Значит, при импорте будет уведомление что в строке 5 нет значений для показателей. При этом, если будет заполнена ячейка Бюджет план или Кол-во целевых, мы не пропустим строку и уведомление не покажем (тк есть значения для 1 ячейки в строке).

    Что делать: зависит от ситуации

    1. если пустые строки связаны с неправильным заполнением файла, то нужно скорректировать сам файл и заполнить строки

    2. если ситуация связана с тем, что в этих ячейках не должно быть данных и это ок, то проигнорировать сообщение.

Обратите внимание что сами предупреждения не являются ошибками. Они уведомляют о том, что часть строк из файла не удалось импортировать (а остальные импортированы успешно)
