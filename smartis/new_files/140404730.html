Источники данных о сегментах для Я.Аудиторий:

  1. Яндекс.Метрика
    Создать сегмент в Аудиториях можно на основе данных вашего счетчика или счетчика, к которому у вас настроен гостевой доступ на редактирование
    Обратите внимание, сегмент будет построен на основе данных за последние 90 дней    .
    Если охват составит не менее 100 анонимных идентификаторов, сегмент получит статус «Готов»
    Если доступ к счетчику был отменен или изменен на гостевой доступ на чтение, то сегмент получит статус «Не активен» и показ рекламы по нему будет остановлен.
  2. Выгрузки
    Дополнить существующий сегмент на основе загружаемых данных собственными данными нельзя. Вы можете только создать новый сегмент и заменить в настройках рекламы в Яндекс.Директе старый сегмент на новый.
  3. внешние DMP-провайдеры
    Для сегмента данного типа нельзя просмотреть статистику, создать на его основе сегмент похожих пользователей или предоставить доступ к нему другому пользователю.
  4. на основе гео-локации
    например: 55.755831,37.617673
    После добавления в сегмент строка с его адресом или координатами попадает в блок Местоположения. 
    В сегмент можно добавить не более 1 000 местоположений (не более 100 для условия Аудитория: была N дней за период)
    ИЛИ Введите или скопируйте список интересующих вас адресов или географических координат, по одному местоположению в строке. 
  5. на основе показов пикселя (аналог DCM)
  6. Похожие на готовый сегмент
    Нельзя создавать сегменты похожих пользователей на основе сегментов, к которым вам предоставили доступ другие пользователи Аудиторий.

    По умолчанию распределение пользователей по городам и типам устройств в новом сегменте будет приближенным к исходному. Например, если в исходном сегменте половина пользователей была из Москвы, а другая половина — из Санкт-Петербурга, в новом сегменте пользователи будут аналогично распределены по городам. Если не удастся найти пользователей, сохранив исходное распределение, Аудитории подберут наиболее похожих пользователей без учета городов проживания или типов устройств. Для поиска пользователей в любых городах или с любыми устройствами отключите соответствующие опции.


Сегмент в статусе «Мало данных» или «Слишком много данных» не обновляется автоматически, и его нельзя использовать для настройки рекламных кампаний.

Можно пересчитать один сегмент не более двух раз и всего можно выполнить не более 20 пересчетов за последние 24 часа. Не пересчитываются сегменты  на основе данных DMP и сегменты, к которым вам предоставили доступ другие пользователи.

Лимиты

Пользовательские лимиты интерфейса:

  • за минуту можно создать не больше 10 сегментов любого типа;
  • за час — не больше 100 сегментов любого типа;
  • за сутки — не больше 500 сегментов любого типа.
  • Если на вашем логине 1000 или больше сегментов на основе данных о геолокации и меньше 5% из них используются для нацеливания в Директе, создание геосегментов будет недоступно. Удалите неиспользуемые сегменты.

Статус готового сегмента изменился на «Мало данных»
Сегменты Аудиторий регулярно и автоматически обновляются. При этом из сегмента удаляются неактуальные анонимные идентификаторы и добавляются вновь найденные. Соответственно, охват сегмента может изменяться как в большую, так и в меньшую сторону. Сегмент получает статус «Мало данных», когда его охват менее 100 анонимных идентификаторов.

Лимиты API Яндекс Аудиторий:

  • по IP — определяет предельно допустимое количество запросов в секунду, пришедших с одного IP-адреса, и равно 30.

  • по user_login — ограничивает число запросов в сутки со стороны одного пользователя и равно 5000.

  • При отправке запроса на создание или изменение сегмента любого типа действуют ограничения:

    • за минуту — не больше 10 запросов;

    • за час — не больше 100 запросов;

    • за сутки — не больше 500 запросов.

  • При отправке запроса на создание/изменение геосегментов действуют дополнительные ограничения: если на вашем логине 1000 или больше сегментов на основе данных о геолокации и меньше 5% из них используются для нацеливания в Директе или Дисплее, отправка запроса на создание геосегментов будет недоступна.

В случае превышения квот дальнейшие действия пользователя временно блокируются. При этом API возвращает ответ с HTTP-статусом 429 Too Many Requests. В теле ответа содержится информация о типе превышенной квоты.

При превышении квоты по IP разблокировка происходит, когда число запросов к API, совершенных за последнюю секунду, становится меньше 30.

При превышении квоты по user_login разблокировка происходит автоматически в полночь по московскому времени.

Источник про лимиты: https://yandex.ru/dev/audience/doc/intro/quotas.html

API Яндекс.Аудиторий

Ссылка на документацию: https://yandex.ru/dev/audience/

API позволяет создавать разнообразные аудиторные сегменты, которые можно использовать для настройки рекламных кампаний в Яндекс.Директе и Яндекс.Дисплее. В основе API лежит технология Крипта.

С помощью API вы сможете создавать, изменять и удалять сегменты, а также управлять настройками доступа к сегментам.
