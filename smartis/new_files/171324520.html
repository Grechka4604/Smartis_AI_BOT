Основная информация

Jivosite -  сервис для коммуникаций. Он предоставляет возможности общения посетителя сайта через онлайн-чат, через обратный звонок и через мессенджеры. 

Интеграция с Jivosite работает методом webhooks, то есть на стороне Smartis в кабинете клиента генерируется URL для приема уведомлений. При таком типе интеграции данные по обращениям будут доступны с момента подключения и настройки вебхук, синхронизация исторических данных не доступна.


Для этой интеграции нет синхронов (так как интеграция через webhooks)

Статья в том числе для клиентов тут: https://smartis.bi/faq/intieghratsiia-jivosite-i-smartis-instruktsiia-po-podkliuchieniiu-i-vozmozhnosti/

Особенности работы внутри коробок

Хуки должны идти в облако поскольку коробки недоступны внешнему миру. Поэтому прописываем в самом Jivosite облачные адреса для вебхуков. Smartis облако сначала регистрирует хуки у себя (можно строить отчеты по JivoSite в облаке.

В коробке надо создать интеграцию с сервисом, она должна быть активная. Если у клиента несколько активных, то данные складываются в первую не удаленную активную интеграцию по списку.
Если в коробке есть хотя бы одна активная неудаленная интеграция по Jivosite - коробка заберет все обращения из облака автоматически раз в час синхроном ид 363 "Box: передача информации в коробки из таблицы requests по "сервисам-хукам"" App\Synchronizers\Stat\Box\BoxRequestsByWebhookServicesSync

Подключение интеграции


  1. Для подключения интеграции вам понадобится ваш личный кабинет Smartis с доступом к разделу интеграций, и доступ в личный кабинет Jivosite.
  2. Зайдите на страницу управления интеграциями Smartis в разделе администрирования: https://my.smartis.bi/admin/integrations
  3. Найдите в списке "Доступные для подключения" интеграцию с Jivosite, нажмите "Подключить"
  4. В открывшемся модальном окне введите:
    • Наименование интеграции – любое человекопонятное название, чтобы вы потом могли найти подключенную интеграцию в списке.
    • Нажмите "Сохранить" (остальные поля могут потребоваться потом, рассмотрим в шагах ниже)

  5. В том же разделе в списке "Подключенных интеграций" появится только что сохраненная интеграция. Нажмите на шестеренку:



    Смартис автоматически сгенерил адрес для приема webhook от jivosite через которые они будут общаться передавая обращения, он нам понадобится для настройки уже в кабинете jivosite
  6. Переходим в настройки интеграций личного кабинета Jivosite: https://app.jivosite.com/settings/channels





Включаем вебхуки и вставляем адрес подготовленный смартисом.

Доп. функция
Если у вас уже настроены вебхуки на другой адрес (дживосайт позволяет отправлять вебхуки только на один адрес), воспользуйтесь дополнительной возможностью смартиса для перессылки вебхук. В настройках интеграции включите опцию "Перенаправление webhooks" и вставьте в поле "URL адрес перенаправления" текущий настроенный адрес для вебхук в кабинете jivosite.

При включенной опции "Перенаправление webhooks" - когда смартис примет вебхуку от jivosite - он сразу же перенаправит ее дальше по указанному адресу со всеми входными параметрами, которые отдал сам jivosite без искажений. Например это часто необходимо если вы подключили дживосайт к вашему Б24 для приема лидов.

  • Опцию "Передача сессий Smartis" настоятельно рекомендуем оставить включенной, в этом случае в чат/заявку с сайта будет проброшен параметр smartis session id и будет произведена 100% связка со всей информацией цифровых паспортов смартиса. При выключенной опции будет производится анализ источника и сессии - только по параметрам которые знает сам jivosite
  • если перенаправление происходит не сразу (например, клиент получает от нас данные только ночью, а не регулярно) - нужно проверить доступность клиента в течение дня. Тк с этим может быть связано то, что перенаправленные данные приходят не сразу.

ВАЖНО! Лучше подключать 1 интеграцию к 1 сайту, потому что потом нельзя поделить на жк будет 1 интеграцию. 

Привязка интеграции к сайту

Чтобы использовать данные jivosite в отчетах Smartis нужно привязать ее к сайтам, и использовать пришедшие данные из jivosite в качестве показателей для построения отчетов:


  1. Для работы со статистикой необходимо выполнить привязку интеграции к сайту. Варианты для привязки будут доступны после того, как будет обработано хоть одно обращение из jivosite 
  2. Для привязки необходимо пройти в раздел "Проекты и сайты", выбрать нужный сайт и зайти в его настройки. 
  3. В доступных для привязки интеграциях необходимо выбрать интеграцию jivosite и "привязать ее с логином" к сайту.

  4. Из выпадающего списка выбрать логин, по которому приходят данные для выбранного сайта (можно выбрать только 1 логин, если к сайту нужно привязать несколько логинов - пункт 3 и 4 выполняются столько раз, сколько логинов нужно привязать.
  5. Для данного сервиса невозможно настроить привязки в рамках сайта дополнительно.


Показатели JivoSite

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. Также показатели jivosite чаты и заявки добавляются в формулу для системного показателя "Все обращения (новый)"

  1. jivosite чаты - отображает данные по чатам - онлайн чат на сайте (источник чатов определяется на основе данных рекламной кампании jivosite, по которой пришел чат)
  2. jivosite заявки - отображает данные по заявкам - заявки на обратный звонок (источник чатов определяется на основе данных рекламной кампании jivosite, по которой пришла заявка)


Фильтры для данных JivoSite

Особые фильтры для данных JivoSite, применяются только к данным из этого сервиса

  • JivoSite: Статус чата - фильтрует данные по статусу (диалог закрыт/новый диалог)

  • JivoSite: Теги - фильтрует данные по тегам
    JivoSite: event_name - #34666 - по названию события в jivosite

    • chat_updated
    • chat_finished
    • chat_accepted
    • offline_message
    • client_updated

Также к данным из сервиса применяются все системные фильтры, которые доступны в выпадающем списке при выборе показателей данного сервиса.

Привязка источника обращения JivoSite в Smartis

Когда мы получаем данные из JivoSite, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis (из JivoSite мы получаем utm метки, на основе чего и выполняются привязки по условиям площадок). О привязках по условиям к площадкам в Smartis читать тут: 2. Каналы и площадки
Если обращения из JivoSite не привязаны к площадке, то они будут попадать в площадку "Другое" в отчете по площадкам/каналам.


В СJM клиента обращения из JivoSite отображаются так:

Отображаются:

  • тип обращения (онлайн заявка, офлайн заявка и другие)
  • длительность (заполнения)
  • телефон клиента 
  • Канал и Площадка (на основании того, к какой площадке мы привязали данные обращения)
  • регион
  • город
  • utm метки - мы их также получаем из JivoSite
  • Ссылающаяся страница - страница, с которой выполнен переход на сайт

  • Страница входа - страница сайта, на которую зашел пользователь перед совершением обращения

  • Google client ID - id пользователя в Google 
  • Yandex client ID - id пользователя в Яндекс метрике 
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в JivoSite теги не передадутся.
  • история переписки (для чатов) - открывается по клику на иконку истории переписки
