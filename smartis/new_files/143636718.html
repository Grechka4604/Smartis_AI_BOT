Рис. 1 - Пример интерфейса импорта СРМ данных


Для чего нужен

С помощью данного функционала, присутствует возможность загружать xls файл с подготовленными срм данными по определённым полям, которые присутствуют в системе для конкретного клиента.

После успешной загрузки формируется шаблон для последующих ручных загрузок или внешних загрузок посредством API.

Для того, чтобы клиент мог воспользоваться загрузкой через API, необходимо подготовить шаблон (сохранённые настройки импорта), по которому будет происходить сохранение переданных значений в систему.

Нужен для того, чтобы единоразово или с какой-то периодичность импортировать срм данные, в случае если нет интеграции.

ВАЖНО! Если необходимо делать это на постоянной основе импортировать срм данные, то можно воспользоваться Автоимпортом или Api импортом 


Этапы импорта

Чтобы загружать настройки в один клик, достаточно успешно выполнить и сохранить импорт один раз.

Перед загрузкой убедитесь, что в таблице нет объединенных ячеек и формул 

Последовательность действий следующая:

  1. Подготовить xls файл, по возможности с реальными данными клиента, которые должны быть загружены в систему. Заголовки должны быть в 1 строке

    1. Пример: 

      Тип сущностиЖКПлановое количество, шт......
      Договор бронированияЮжная Битца7......
      Договор купли продажи............ 
  2. Заполняем форму.  ( Рис. 2)
    1. Выбираем из списка, нужную интеграцию.
    2.  Указываем Тип CRM события. (Можно выбрать определить автоматически).
    3. Загружаем подготовленный xls файл.
    4. Нажимаем кнопку "Загрузить данные".
  3. Соотносим указанные столбцы из xls файла с полями из системы (Рис.3)
  4. И подтверждаем сохранение и указываем наименование шаблона импорта.

Загрузка данных по визитам в офис продаж (пример)


Первым делом создаем коннектор, так будет удобнее понять, корректно ли загрузились данные.

Этап 1:

  1. Если в файле клиента нет id CRM, то его нужно сгенерировать вручную в файле с выгрузкой. Важно - чтобы id были уникальными

  2. Добавляем столбец Тип сущности CRM, везде указываем Визит (если такого столбца нет)

  3. Для загрузки нам нужны только эти столбцы:

  4. Уникальный ID

  5. Тип сущности CRM

  6. Телефон клиента

  7. Объект ЖК (если нет - добавить)

  8. Дата и время завершения встречи (в формате timestamp метки) - как пересчитать - в столбце "Формула" есть формула по пересчету

  9. Дата и время создания сущности в CRM

4. Дата и время завершения встречи. если клиент присылает файл, где дата и время в отдельных столбцах, то нужно объединить значения этих столбцов, чтобы были в таком виде: 2021-11-08 10:30:00

Далее, после того, как объединили столбцы, нужно задать столбцу Дата и время формат: Обычный текст. После этого нужно перевести получившиеся значения в timestamp, потом удалить формулу, проверить, чтобы в других столбцах тоже не было формулы, и загрузить файл в смартис.

5. Дата и время создания сущности CRM: чтобы импортировать данные в этот столбец, нужно в столбце, в котором  объединяли время и дату и она отображалась в формате 021-11-08 10:30:00 поставить формат Обычный текст и формулу объединения удалить.


Этап 2:

  1. На вкладке импорта выставляем такие значения, как указано выше.

  2. id выбираем: Общие параметры: Уникальный идентификатор сущности в CRM

  3. тип сущности: Общие параметры: Тип сущности/этап воронки/статус

  4. телефон: Контакты/Клиенты: "Все телефоны клиента через запятую"

  5. ЖК: Общие параметры: ЖК/Наименование объекта

  6. Дата и время завершения встречи: Визит: "Дата и время завершения встречи"

  7. Дата создания сущности: Общие параметры: Дата и время создания сущности в CRM



Загрузка данных по сделкам (пример)


Этап 1. Подготовка файла

  1. Если в файле клиента нет id CRM, то его нужно сгенерировать вручную в файле с выгрузкой. Важно - чтобы id были уникальными

  2. Добавляем столбец с ЖК договора (если нет)

  3. Добавляем столбец с Типом сущности CRM - Договор купли-продажи (если нет)

  4. Редактируем столбцы с датами. Их должно быть 2:

  5. Дата и время создания сущности CRM (этого столбца у клиента нет, он просто дублирует столбец с датой подписания - то есть его мы создаем сами)

  6. Дата и время подписания договора

Оба эти столбца должны быть в таком формате: 20.11.2021 00:00:00, и оба отформатированы на "Обычный текст".

Сначала надо объединить дату и время в 1 столбец, далее настроить формат отображения даты как в примере, и уже потом перевести дату и время в формат текста и удалить формулу.


Если у клиента в файле не будет времени, а будет только дата, то нужно прописать вручную время (я везде предлагаю ставить время 00:00:00 и протягивать вниз ВАЖНО: проверить, как время протянулось - чтобы не было значений 00:00:61 и т.д. - иначе сделки не загрузятся в нужные даты).

Чтобы объединить 2 столбца с данными (дата и время), надо использовать формулу объединения =E2&" "&D2 (где E2 и D2 это ячейки, которые надо объединить)



Этап 2. Загрузка в Смартис

  1. Загружаем все столбцы из файла клиента, кроме: Дата сделки (Год), Дата сделки (Квартал), Дата сделки (Индекс месяца), Дата сделки (Месяц)

  2. На вкладке импорта выставляем такие значения, как описаны выше

  3. Для столбцов выбираем такие значения:

  4. id - Общие параметры: Уникальный идентификатор сущности в CRM

  5. ФИО - Контакты/Клиенты: "Имя клиента"

  6. Телефон - Контакты/Клиенты: "Все телефоны клиента через запятую"

  7. Тип лота - Договор купли-продажи: "Количество комнат"

  8. ЖК договора - Общие параметры: ЖК/Наименование объекта

  9. Этап - Общие параметры: Тип сущности/этап воронки/статус

  10. Номер квартиры - Квартира: "Номер квартиры"

  11. Кв.м - Договор купли-продажи: "Приобретаемая площадь, м2"

  12. Стоимость ОН - Договор купли-продажи: "Бюджет"

  13. Дата сделки - Договор купли-продажи: "Дата и время подписания договора (системное)"

  14. дата создания сущности - Общие параметры: Дата и время создания сущности в CRM

  15. Комментарии - Общие параметры: Примечания/Комментарии

  16. Форма оплаты - Договор купли-продажи: "Способ оплаты"

  17. Источник - Договор купли-продажи: "Источник ДКП в CRM ур.1"

  18. Источник Рекламы - Договор купли-продажи: "Источник ДКП в CRM ур.2"

  19. МП - Договор купли-продажи: "Ответственный менеджер (ДКП)"

  20. Статус - Договор купли-продажи: "Факт оплаты"


FAQ.

  1. Что делать если прошло время, но количество импортированных строк не равняется количество полученных строк? 
    Это зависит от ситуации:
    1. бывают случаи, что клиент отправляет в одном файле 1 и тот же элемент несколько раз. Если это так, то кол-во переданных строк должно быть больше, чем кол-во импортированных строк и это нормально.
    2. Если же мы видим, что не импортировано много элементов (например, 300 элементов из 1200), то необходимо подождать час, возможно элементы еще сохраняются. Если после часа ничего не поменялось, можно повторно выполнить импорт. Если повторный импорт не помог, то стоит спросить есть ли проблемы в системе или же создать тикет.
