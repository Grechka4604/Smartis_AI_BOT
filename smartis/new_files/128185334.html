Обмен данными для Post View аналитики Smartis 

Со стороны площадки необходимо создать API метод по которому Smartis получает данные.  

Параметры ответа 

  • portal_client_id [Required] - уникальный идентификатор посетителя у медийной площадки (кука медийной площадки).
  • external_login_id String, - идентификатор клиента у медийной площадки. Можно выдавать данные только по конкретному клиенту определяя его по ACCESS_KEY. В этом случае Smartis будет опрашивать API от имени всех клиентов, которые подключили интеграцию с соответствующими ACCESS_KEY.
  • external_id String [Required], - уникальный идентификатор показа[1]

  • utm_source String,
  • utm_medium String,
  • utm_campaign String,
  • utm_content String,
  • utm_term String,

  • device_type UInt8, - 0 = ПК, 1 = планшеты, 2 = телефоны
  • view_page String, - url медийной площадки, где произошел показ
  • view_banner_link String, - url куда вел креатив, если бы посетитель на него кликнул с метками, которые настраивал клиент/РА
  • action_type String, - тип действия (show/click, ?)
  • view_place String, - место на медийной площадке где был показан креатив в терминах площадки (шапка, поиск, список ЖК, справа/слева - итд)
  • banner_type_name String, - наименование формата размещения (тгб/параллакс/шапка - итд - в терминах площадки)

  • creative_image_url String, url картинки которая показалась пользователю
  • creative_title String, заголовок текстово-графического блока если есть
  • creative_text String, более подробный текст текстово-графического блока

  • created_at DateTime, - дата и время просмотра медийного размещения пользователем YYYY-mm-dd HH:ii:ss, обязательно с точностью до секунд
