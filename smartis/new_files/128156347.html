Цель: Поддерживать задачи проектов в актуальном состоянии

Участники процесса: аккаунт-менеджер.

KPI процесса: Актуальные данные на каждый день.

Как контролируем: Проверяем результаты выполнения задач, следим за историей изменения документа

Владелец процесса: руководитель консалтинга


Правила

  1. Актуализируем все задачи проекта каждый день. Чек лист по утрам. Документ должен быть актуален до 18:30 каждого дня. 

  2. Менеджер контролирует статус каждой задачи.


1) Что такое актуален. Чек лист.

  1. Появились новые задачи/баги внутри проекта? 
    1. Создать новые тикеты в JIRA (баги/доработки)
    2. Узнать суть новых тикетов по клиенту (баги)
  2. Появились вопросы/правки по задачам? 
    1. Ответить по описанию решения
    2. Внести комментарии в тикеты в JIRA. 
  3. Отметить выполненные задачи
  4. Спланировать следующие шаги на сегодня?
    1. Все ли можешь выполнить?
      1.  Заявить о проблеме/попросить помощь у команды/согласовать перенос сроков.  
      2. Описать следующий шаг в формате - кто, что и когда


2) Регулярные встречи с Заказчиком

  1. Согласован график контактов/синхронизации с командой клиента
