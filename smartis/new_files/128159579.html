Описание решения для клиента (на доработке)

Пошаговая инструкция для реализации коллтрекинга на сайте ЦИАН (общий документ с клиентом): https://docs.google.com/document/d/1mEIooQU9l4JIOANup2oGlkwp5BQEMu37R4LBdIovcds/edit#heading=h.5ai154h1yr52

Инструкция описывает алгоритм реализации и используемые методы с примерами запросов и ответов. 

Описание релиза от Вики Фетисовой: Коллтрекинг для Cian


Критерии успешного тестирования:

https://docs.google.com/spreadsheets/d/1lLpF2lZPmojCCxB5HcNBWqBeQj90eO-DCE8Gq2kxQ20/edit?usp=sharing

  1. АПИ часть клиента
    1. Скорость ответа API p90 - 350 мс, p99 - 800мс.
    2. В АПИ нет ошибок. Доля успешных запросов - 95%
  2. АПИ часть админки
    1. Работают АПИ для блоков (создание, редактирование, удаление)
    2. Происходит переадресация
    3. Можно мониторить охват поблочно.
  3. Телефония
    1. Прозваниваем вручную, слушаем громкость голоса и шумы
    2. Смотрим по API - количество недозвонов. (С вами обговорим количество проблем)
    3. Прозвоним нашим роботом номера. Посмотрим дозвоны. (Сейчас количественную метрику не скажу, так как мало ЖК)



Возможные проблемные кейсы:

  1. Что мы делаем, если мы не выделили подменный номер телефона?
  2. Система может работать прямее, если дать возможность покупать номера без расчета количества.
