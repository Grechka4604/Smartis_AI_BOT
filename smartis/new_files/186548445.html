Основная информация

Фактически, MacroData это новый способ получения данных из MacroCRM. См. также 

MacroData подразумевает прямой доступ к удаленной БД (MySQL). Преимуществом данного подхода является полный доступ к информации о ключевых сущностях (Сделки, Заявки, Контакты и т.п.). Для каждого клиента выдается свой набор реквизитов и доступ к своему набору таблиц. Схема данных (название таблиц, полей, связи и т.п.) одинакова для всех клиентов. Документация https://docs.macrodigital.ru/manual/macrodata. Документация по схеме данных https://api.macroserver.ru/docs/macroData#/

По состоянию на декабрь 2024 г. компания Macro декларирует поэтапный перевод получения данных из MacroCRM с MacroAPI (так работает наша текущая интеграция MacroCRM) на MacroData для всех клиентов. Сроки перевода не известны. Важно! Подключение MacroData для клиента платная услуга. Для работы интеграции необходимо, чтобы клиент подключил MacroData. Важно! Для корректного матчинга персон необходимо, чтобы клиент разрешил компании Macro передачу нам персональных данных. Без этого, в таблице контактов будут замаскированы символами *** имена, телефоны, емейлы и иные ПД. Вне зависимости от наличия разрешения на передачу ПД в таблице Звонки содержится полный номер телефона контакта и его идентификатор.

При первичном внедрении клиента, работающего с MacroCRM и заинтересованного в передаче данных из MacroCRM в Smartis для комплексного анализа, предпочтительно согласовывать с ним подключение MаcroData и передачу нам ПД.

Разработка велась в рамках задачи https://app.asana.com/0/1206880245385494/1207865012078515 и ее подзадач.

Важно! По состоянию на 28 декабря 2024 г. от Артема Бреславского получена информация по MacroAPI - "Планируем отключать в ближайшее время. Держим только для вас его." См. чат MACRO & Smartis. Рабочая в Телеграм.
