Цель: Проект готов к запуску - создано описание проекта и согласованы ресурсы, необходимые для реализации.

Участники процесса: клиент, аккаунт-менеджер, руководитель консалтинга, ОПР.

KPI процесса: Скорость выполнения процесса - 5 рабочих дней. 

Как контролируем: Даты в создании документа и завершения

Владелец процесса: руководитель консалтинга



Описание процесса

  1. Создать описание проекта

    1. Создать новый тикет в JIRA

      1. Тип задачи - Проект

      2. Тема - Клиент || Суть проекта
      3. Epic Name - Клиент || Суть проекта (системное поле JIRA, это название проекта, которое будет отображаться во всех задачах проекта)
      4. Срок исполнения - дата завершения всего проекта
      5. Клиент - выбрать клиента (позволяет привязать проект к основной карточке Клиента в JIRA)
      6. Исполнитель - руководитель проекта

    2. Заполнить карточку

      Заказчик. 

      1. Плохо - Смайнекс, фин директор, маркетологи. 

      2. Хорошо - Алексей Горшков, руководитель отдела маркетинга. 

    3. Участники. 

      1. Плохо - Коля, Петя, Вася. 

      2. Хорошо - Коля - инженер, отвечает за работу АТС, Петя - маркетолог, отвечает за передачу данных в CRM, Вася - отвечает за настройки личного кабинета


    4. Предпосылки

      1. Зоны ответственности заказчика?

      2. Какую проблему хочет решить клиент?

      3. Что послужило причиной для старта проекта?

      Плохо: Сквозная аналитика сейчас не реализована, нужно сделать.
      Хорошо:  Компания занимается недвижимостью стоимостью от 1 млн долларов. В компании работает 10 департаментов (жилая, коммерческая, покупка, аренда). У каждого департамента есть план продаж, исходя из которого формируется медиаплан для маркетинга. Для каждого направления есть нормальная стоимость лида и сделки, которую нужно удерживать, чтобы обеспечить продажи. 

    5. Цели
      1. Результат, который хочет получить клиент по СМАРТ
        Плохо: реализовать СА от всех проектов
        Хорошо: Автоматизировать отчет, показывающий ассоциированные конверсии от контекстной рекламы для обоснования бюджета на 2020 год
      2. Критерии успешности/Требования - список ожидания от решения (бизнес требования, технические требования)

  2. Согласовать с руководителем

    1. Согласовать предпосылки/цель/участников проекта и план с руководителем
    2. Перевести в статус "В работе"

  3. Согласовать план работ с командой проекта

    1. Руководитель проекта готовит состав работ

      1. Шаблон для проектов Внедрения СА -

    2.  Согласовать состав работ с внутренними участниками


  4. Согласовать условия реализации

    1. Получить себестоимость реализации от команды
    2. Получить согласование от директора консалтинга

  5. Согласовать с клиентом

    1. Согласовать план и условия реализации с Заказчиком и участниками со стороны клиента

      1. Если нет - кладем в архив - ставим статус "Закрыто"

      2. Если согласен начинаем работать

    2. Заполнить графу в карточке: "Дата окончания работ (план)"
