При клике на название сервиса - переход на страницу с тех информацией из базы знаний.

Колтрекинг:

1) CoMagic - нативная интеграция (входит UIS, CM, CallGear), тянем: звонки, заявки, чаты.

2) CallTouch - нативная интеграция, тянем: звонки, заявки.

3) Roistat - нативная интеграция, тянем: звонки, заявки.

4) Alloka - нативная интеграция, тянем: звонки.


Квизы:

1) Marquiz - можем подключить напрямую и через колтрекинг (если mq оцифрован через кт). НО! При подключении через колтрекинг может теряться источник, не получаем данные о наименовании квизов.

2) JivoSite - тянем чаты и заявки

3) Flexbe - это сервис для создания квизов, лендингов, а также для сбора обратной связи с сайта компании. В Smartis доступна интеграция с сервисом для того, чтобы собирать данные по лидам, отправленные через формы flexbe. С помощью интеграции можно отслеживать заявки, их источники, ответы на квиз и переписку с клиентом из Flexbe в Smartis.

Интеграция работает через API. В Smartis получаем данные по лидам.

4) CalltoVisit - сервис бесплатного заказа такси. Сервис достаточно популярен среди застройщиков, так как офисы продаж статичны и важным этапом конверсии в воронке застройщика является визит в офис продаж. 


Какие данные получаем из CalltoVisit:


  • заказы
  • расходы на такси


5) Rrofitbase - интерактивный каталок объектов/квартир на сайте застройщика, плюс сбор бронирований в crm застройщика. 


Интеграция в Smartis с этим сервисом работает для получения заявок, которые пользователи отправляют через инструменты profitbase (из сервисы мы получаем заявки).


6) CallKeeper – это веб-сервис обратного звонка, который помогает автоматически следить за посетителями сайта, предлагать звонок и соединять потенциальных клиентов с менеджерами.С помощью интеграции можно отслеживать заявки на обратный звонок и их источники в Smartis.


Интеграция работает через API. В Smartis получаем данные по обращениям.


7) Импорт Битрикс24– это искусственный сервис, в  который мы можем импортировать обращения из внешних сервисов. 


Например, обращения из старого коллтрекинга/квиза/чата с которым нет интеграции или настроить автоматический импорт на постоянной основе с сервисом с которым нет интеграции. 


CRM

Натив:

1) АМО CRM - система CRM для контроля продаж. В Smartis с этой CRM нативная интеграция (работает по API и подключается без участия разработчиков). Из АМО мы получаем всю информацию о клиентах - сделки, их статусы, контактные данные и другие параметры, которые для каждого покупателя заполняются в CRM.

2) Битрикс 24 - система CRM для контроля продаж. В Smartis с этой CRM нативная интеграция (работает по API и подключается без участия разработчиков). Из Битрикс мы получаем всю информацию о клиентах - сделки, их статусы, контактные данные и другие параметры, которые для каждого покупателя заполняются в CRM.

Данные из Битрикс 24 не содержат информацию о том, с какого источника (рекламного, органического) "пришел" клиент. На основе информации о рекламном источнике предыдущих касаний клиента с рекламой (просмотры медийки, сессии, обращения) строится аналитика по CRM данным в разрезе каналов, площадок, кампаний и т.д.

3) Новая (с 2022 года) отраслевая СРМ для недвижимости. Они объединяют в себе управление продажами, коммуникациями, шахматкой, документооборот. Маркетинговая аналитика у них не развивается - мы помогаем закрыть эту задачу в партнерстве.


В Smartis с этой CRM нативная интеграция (работает по API и подключается без участия разработчиков). Из МакроCRM мы получаем всю информацию о клиентах - сделки, их статусы, контактные данные и другие параметры, которые для каждого покупателя заполняются в CRM.


4) Microsoft Dynamics — СRM и ERP системы для планирования ресурсов предприятия и управления взаимоотношениями с клиентами, разработанных компанией Microsoft.


Мы считаем интеграцию полу кастомной, потому что у каждого клиента свои справочники и часть справочников может быть интегрирована, а  часть справочник необходимо будет вручную разработчику доделывать. Так же обычно клиенты не дают прямой доступ к crm в целях безопасности и приходится выбирать один из кастомных методов. Редко когда подключаем такую нативную интеграцию. 


Цель интеграции – забрать из первоисточника данных (CRM) о продвижении клиентов необходимую и достаточную информацию, чтобы связать маркетинговую историю с CRM историей продвижения конкретных клиентов, а также проводить дополнительную бизнес- аналитику согласно запрошенным требованиям в рамках ТЗ заказчика.

Маркетинговые касания:


Как правило клиенты фиксируют в CRM больше касаний с клиентами чем отслеживают маркетинговыми сервисами (прямые звонки в колцентр, самоходы прямо в офис продаж, прямой проброс в crm от сервиса к которому больше нет доступа, импорты данных, базы телемаркетинга, брокерские активности, самодельные статические колтрекинги). Чтобы смартис нормально разложил срм данные по источникам - нужно такие касания иметь в качестве маркетинговых (в requests).
