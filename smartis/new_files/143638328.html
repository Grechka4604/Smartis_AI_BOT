На сайтах с базами недвижимости можно использовать такую функцию, как сквозные звонки.

Сайты с базами недвижимости называются классифайды.


Обычно, классифайды выглядят так: 



На одной странице очень много проектов сразу. У каждого проекта есть кнопка заказа обратного звонка - зеленый кружочек, при нажатии на который появляется окно виджета обратного звонка. 

При этом, у каждого проекта - свой контакт-центр, и нужно правильно распределять звонки. То есть, если посетитель жмякнул на кнопку обратного звонка проекта ONLY - звонок должен идти на номер контакт-центра ONLY и т.д.

Как раз функция "Сквозные звонки" поможет это сделать. 


Подключение сквозных звонков.

  1. Разместить скрипт SmartCallBack на всех страницах сайта.
  2. Далее провести разметку на сайте:

На любой странице сайта, в местах, где необходимо разместить кнопку вызова виджета с определенными параметрами, необходимо разместить блок:

 

<div class="scb_thcall_block" callback_id=«ID_строки» callback_type=«ТИП_кнопки»></div>

 

По параметру callbackid = «IDстроки» система найдет требуемый набор настроек из файла автосинхронизации.

По параметру callback_type система сможет понять, какой вид кнопки вызова отобразить. На данный момент система имеет несколько видов кнопок, но Вы всегда можете отрисовать собственные. Если клиент не хочет использовать наши кнопки – он может не использовать callback_type.


       3. Формирование файла

 

Необходимо сформировать и заполнить https://disk.yandex.ru/i/dyheT_T67N6NjQ

В строке файла можно задать:

  • номера телефонов клиента - с кем будет соединен посетитель;
  • рабочее время call-центра клиента. В нерабочее время посетителю будет предложен выбор удобного времени для звонка;
  • Дата начала показа / дата завершения показа;
  • Заголовок;
  • Текст на кнопке;
  • Текст подписи

 

Клиенту файл необходимо разместить на сайте, например, https://www.test.ru/smartcallbacktgb.xls

После этого ссылку на файл необходимо прислать аккаунт-менеджеру, который в свою очередь передают эту ссылку разработчику.

Система автоматически будет забирать файл, расположенный по http и обновляет все настройки. Данные с файла синхронизируются раз в 10 минут.

Порядок колонок не важен, главное сохранить название колонок (не менять их).

Обязательные поля это:

 ID блока,

Режим работы call-центра,

Телефон(ы) call-центра (через запятую),

Дата начала и Дата окончания.
