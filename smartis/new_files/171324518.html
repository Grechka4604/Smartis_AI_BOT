Основная информация

Marquiz - сервис квизов, который используется для сбора лидов. Это конструктор маркетинговых квизов (опросников), после прохождения которых посетителя просят оставить контактные данные. Пользователь заполняет анкету-квиз и таким образом создается обращение-запрос к застройщику.

Данные из квизов можно получать 2 способами:

  • если квизы оцифровываются в колл-трекинге клиента - то мы можем получать информацию об оставленных таким образом обращениях через интеграцию с колл-трекингом
  • можно подключить интеграцию с Marquiz напрямую в Smartis.

Несмотря на то, что можно получать данные квизов через колл-трекинг, мы рекомендуем подключать интеграцию с Marquiz в Smartis. Вот чем это полезно:

  • коллтрекинг теряет источник (вместо рекламного источника присылает источник марквиз, потому что в доменах они по-другому указать иногда не могут)
  • при интеграции с Marquiz мы получаем данные о наименовании квизов - для аналитики эффективности сервиса. В данных колл-трекинга такая информация будет недоступна.

При использовании одновременно интеграции Marquiz и оцифровке обращений из Marquiz через колл-трекинг, технически проблем не возникнет. Однако, стоит учитывать:

  • заявки будут дублироваться
  • может размываться вес канала (например, придет клиент обратился по Marquiz с контекста. И далее совершил сессию переходом из ВК. Если мы оцифровываем касания через Marquiz и КТ, то вес контекста в линейной модели атрибуции будет 0,66 а вес ВК 0,33. Однако если мы оцифруем Marquiz только через прямую интеграцию, то вес контекста составит 0,5 и ВК 0,5 соответственно)

Для Marquiz в Smartis есть уточнение геолокации - сам сервис определяет ГЕО клиента с точностью около 50%. Благодаря алгоритмам Smartis мы умеем обогащать информацию о клиентах и повышаем точность определения геолокации обращения до 80%+. Это позволяет застройщикам искать целевую аудиторию вне своего домашнего региона.

Интеграция с Marquiz работает методом webhooks, то есть на стороне Smartis в кабинете клиента генерируется URL для приема уведомлений. При таком типе интеграции данные по обращениям будут доступны с момента подключения и настройки вебхук, синхронизация исторических данных не доступна.

Для этой интеграции нет синхронов (так как интеграция через webhooks)

Особенности работы внутри коробок

Хуки должны идти в облако поскольку коробки недоступны внешнему миру. Поэтому прописываем в самом марквизе облачные адреса для вебхуков. Smartis облако сначала регистрирует хуки у себя (можно строить отчеты по марквизам в облаке.
В коробке надо создать интеграцию с сервисом, она должна быть активная. Если у клиента несколько активных, то данные складываются в первую не удаленную активную интеграцию по списку.
Если в коробке есть хотя бы одна активная не удаленная интеграция по марквизу - коробка заберет все обращения из облака автоматически раз в час синхроном ид 363 "Box: передача информации в коробки из таблицы requests по "сервисам-хукам"" App\Synchronizers\Stat\Box\BoxRequestsByWebhookServicesSync

Подключение интеграции

  1. Для того, чтобы сгенерировать url для приема хук необходимо в списке интеграций Smartis выбрать интеграцию Marquiz

  2. Далее задать имя (может быть произвольным), которое поможет идентифицировать интеграцию в списке, URL будет сгенерирован автоматически 

  3. Далее, данный адрес необходимо указать в кабинете Marquiz в настройках интеграций. На указанный URL будет отправлен POST-запрос. Протестировать получение заявок можно при помощи: http://webhook.site

    Для добавления адреса куда будут приходить хуки, вам необходимо:

    - Зайти в редактирование квиза из личного кабинета;
    - Зайти во вкладку "Интеграции" и выбрать "Webhooks";




    - Добавить хук и вписать адрес, на который будут отправляться хуки


  4. После подключения интеграции она отобразится в разделе "Подключенные интеграции"
    Далее обращения будут поступать и отображаться в кабинет Смартис. 


ВАЖНО! Рекомендуем подключать 1 интеграцию на все квизы и уже через выпадающие списки делать привязки, чтобы не плодить множество интеграций с одним квизом. 

Привязка интеграции к сайту

  1. Для работы со статистикой необходимо выполнить привязку интеграции к необходимому сайту. Варианты для привязки кроме "все данные всех интеграций сервиса" будут доступны после того, как будет обработано хоть одно обращение из марквиза (генерим из форм марквиза)
  2. Для привязки необходимо пройти в раздел "Проекты и сайты", выбрать нужный сайт и зайти в его настройки. 
  3. В доступных для привязки интеграциях необходимо выбрать интеграцию Marquiz и "привязать ее с логином" к сайту.

  4. Из выпадающего списка выбрать логин, по которому приходят квизы для выбранного сайта (можно выбрать только 1 логин, если к сайту нужно привязать несколько логинов - пункт 2 и 3 выполняются столько раз, сколько логинов нужно привязать, если нужно привязать все логины интеграции - выбрать в списке "Все данные всех интеграций"). После выбора логина интеграция привяжется к сайту. 
  6. Если нужно изменить логин, по которому получаете данные из Marquiz, можно в уже привязанной интеграции выбрать другой логин и подтвердить изменения.
  7. Иногда данные по разным проектам могут находится внутри одного сайта (логина) Marquiz. Например, когда у клиента зонтичный бренд, а каждый ЖК - отдельная страница или раздел внутри одного сайта. В таком случае мы должны перейти в настройку привязкок в привязанной к сайту в Smartis интеграции. 


Пример: в логине "Застройщик" есть информация о квизах по ЖК1, ЖК2 и Бренду. Нам нужно сделать так, чтобы к сайту ЖК1 были привязаны только те квизы, которые поступают по этому ЖК.
Как это можно сделать:

  1. Страница обращения соответствует регулярному выражению - подходит в случае, когда квизы по данному ЖК поступают на определенный сайт.  Url страницы обращения указываем в виде регулярного выражения. 
  2. utm_campaign соответствует регулярному выражению - если нужная метка указывается в utm_campaign квиза, то можно использовать этот способ. С помощью регулярного выражения нужно указать все возможные значения меток.
  3. Наименование логина интеграции соответствует регулярному выражению - если к сайту привязаны "все данные всех интеграций сервиса", и вам нужно выбрать сразу несколько логинов с определенным выражением в названии логинов, можно использовать этот способ. Такой способ применим если вам нужно привязать все данные из нескольких логинов.


    Эти условия работают как сужающие. То есть, если вы указали например определенные логины и прописали условия по utm_campaign - то к сайту привяжутся только те данные, которые соответствуют 2 этим условиям (квизы из выбранных логинов, у которых в метке utm_campaign есть прописанное вами условие). 

Показатели

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. Также показатель Marquiz анкеты добавляется в формулу для системного показателя "Все обращения (новый)"

  1. Marquiz анкеты - отображает данные по квизам (источник квиза определяется на основе данных рекламной кампании Marquiz, по которой пришел квиз)
  2. Marquiz анкеты (по моделям атр.) - отображает данные по квизам, которые строятся по моделям атрибуции- то есть источник квиза определяется по-разному в зависимости от выбранной в отчете модели атрибуции. Про модели читать тут: 4. Модели Атрибуции

Детализация данных Marquiz

Для данных Marquiz доступна детализация (исходные данные). Они такие же, как данные по заявкам у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.

Общая справка по детализации данных сервисов обращений и квизов:
3. База обращений
6. Исходные данные (Детализация, Сырые данные)


Основные свойства, передаваемые в обращениях Marquiz:

  • Данные о свойствах обращения из Marquiz: внешний ID, сайт, дата создания, запись разговора (ответы в квизе), теги
  • Данные о клиенте: номер телефона, email, город, регион, устройство и т.д.;
  • Данные о рекламном источнике обращения: utm_метки, gclid, yclid, страница входа, страница обращения, ссылающаяся страница


Для Marquiz в детализации доступны также такие наборы данных (столбцы) которые можно просматривать и при необходимости фильтровать:


  • Marquiz: Наименование Квиза
  • Marquiz: ID Квиза
  • Marquiz: IP заполнителя
  • Marquiz: Результаты опроса
  • Marquiz: Имя заполнителя


Для добавления фильтра по указанным выше параметрам Marquiz обратитесь в тех.поддержку Smartis.

Привязка источника обращения Marquiz в Smartis


Когда мы получаем данные из Marquiz, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis (из Marquiz мы получаем utm метки, на основе чего и выполняются привязки по условиям площадок). О привязках по условиям к площадкам в Smartis читать тут: 2. Каналы и площадки
Если обращения из Marquiz не привязаны к площадке, то они будут попадать в площадку "Другое" в отчете по площадкам/каналам.

В СJM клиента обращения из Marquiz отображаются так:

Отображаются:

  • тип обращения (всегда webhook)
  • длительность (заполнения)
  • телефон клиента 
  • Канал и Площадка (на основании того, к какой площадке мы привязали данные обращения)
  • регион
  • город
  • utm метки - мы их также получаем из Marquiz

  • Страница входа - страница сайта, на которую зашел пользователь перед совершением обращения

  • Google client ID - id пользователя в Google 
  • Yandex client ID - id пользователя в Яндекс метрике 
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в Marquiz теги не передадутся.
  • история переписки (ответы на вопросы) - открывается по клику на иконку истории переписки


Достоверность Marquiz

Достоверность по сервису можно обновлять по кнопке - как в остальных интеграциях. Доступно обновление достоверности и достоверности+данных.

Отображается в разделе Мониторинг системы. О его работе можно почитать тут: 9. Администрирование: Мониторинг системы
Достоверность Marquiz показывает, сколько webhooks мы получили.

Особенности:

  • по сервису нет синхронов, тк интеграция через webhooks
  • ЕСЛИ кабинет марквиза не оплачен, то марквиз хуки по заявкам не отправляет.
    Агентства этого могут не знать и будут нас мучать вопросами почему нет тестовых заявок
