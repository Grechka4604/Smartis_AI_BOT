Основная информация

CallKeeper – это веб-сервис обратного звонка, который помогает автоматически следить за посетителями сайта, предлагать звонок и соединять потенциальных клиентов с менеджерами.С помощью интеграции можно отслеживать заявки на обратный звонок и их источники в Smartis.

Интеграция работает через API. В Smartis получаем данные по обращениям.

Подключение интеграции

  1. Запросить доступы у клиента для подключения интеграции CallKeeper (также клиент может подключить сам, не передавая доступы вам). Нужен API-ключ из CallKeeper. API-ключ можно получить в кабинете CallKeeper  на вкладке Профиль. 
  2. В Smartis перейти в раздел Администрирование → Интеграции и в разделе "Доступные для подключения" выбрать интеграцию CallKeeper


  3. Нажать "Подключить" и заполнить данные для подключения. В поле формы API-ключ из CallKeeper ввести  Api-ключ полученный в сервисе. 

  4. После подключения интеграция отобразится в разделе "Подключенные интеграции" (Администрирование → Интеграции).После подключения в клиенте интеграции с сервисом запускается первичная синхронизация данных.


Данные по интеграции после подключения автоматически забираем за 30 дней в прошлое, с момента подключения интеграции. 

Привязка интеграции к сайту

Пока интеграция не привязана к сайту, данные по ней невозможно просматривать в отчете по данному сайту в Smartis


  1. Зайти в раздел Администрирование → Проекты и сайты → Сайты и найти в списке нужный сайт. 
  2. Кликнуть на название сайта - откроется меню настройки привязок интеграций к сайту. 
  3. В разделе "Интеграции доступные для привязки" найти интеграцию CallKeeper и нажать "Привязать с логином".

  4. Из выпадающего списка выбрать логин, по которому приходят лиды для выбранного сайта (можно выбрать только 1 логин), если к сайту нужно привязать несколько логинов - пункт 3 и 4 выполняются столько раз, сколько логинов нужно привязать. Можно также выбрать "Все данные этой интеграции", если нужно привязать все логины сразу. После выбора логина интеграция привяжется к сайту. 
  5. Если нужно изменить логин, по которому получаете данные из CallKeeper, можно в уже привязанной интеграции выбрать другой логин и подтвердить изменения.

  6. Если выбран логин "Все данные этой интеграции", можно дополнительно настроить условия привязки - выбрать сразу несколько логинов, прописав регулярные условия. Это позволит не подключать большое кол-во интеграций к сайту и не следить за тем, что при появлении новых логинов для этого сайта нужно будет привязывать новые интеграции.


Отчеты по данным CallKeeper

Показатели

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. 


  1. CallKeeper обращения- отображает данные по заявкам (источник заявки определяется на основе данных рекламной кампании CallKeeper)
  2. CallKeeper обращения (по моделям атр.) - отображает данные по заявкам, которые строятся по моделям атрибуции- то есть источник заявки определяется по-разному в зависимости от выбранной в отчете модели атрибуции. Про модели читать тут: 4. Модели Атрибуции


Фильтры для данных CallKeeper


Применяются все системные фильтры, которые доступны в выпадающем списке при выборе показателей данного сервиса.

Также есть фильтры, которые применимы только к данным CallKeeper:


  • CallKeeper: Статус обращения - позволяет фильтровать по принятым/пропущенным

  • CallKeeper: Тип заявки - позволяет фильтровать по типу (заявка, api, ВК и другие - у разных клиентов могут отличаться).

Детализация данных CallKeeper

Для данных CallKeeper доступна детализация (исходные данные). Они такие же, как данные по заявкам у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.

Общая справка по детализации данных сервисов обращений:

Привязка источника обращения CallKeeper в Smartis

Когда мы получаем данные из CallKeeper, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis (из CallKeeper мы получаем utm метки, на основе чего и выполняются привязки по условиям площадок). О привязках по условиям к площадкам в Smartis читать тут: 2. Каналы и площадки

Если обращения из CallKeeper не привязаны к площадке, то они будут попадать в площадку "Другое" в отчете по площадкам/каналам.

В СJM клиента обращения из CallKeeper отображаются так:

Отображаются:

  • тип обращения 
  • телефон клиента и оператора
  • Канал и Площадка (на основании того, к какой площадке мы привязали данные обращения)
  • utm метки - мы их также получаем из CallKeeper
  • IP адрес
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в CallKeeper теги не передадутся.


Достоверность CallKeeper

По данному сервису достоверность не отображается.
