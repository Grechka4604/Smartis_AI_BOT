ПАРТНЕРСТВА - есть (появились ключевые партнеры, которые помогают нам развиваться)

Ключевые партнеры

- ЕРЗ - канал продаж.
- ЦИАН - партнер по развитию продукта; доверил нам эксклюзивное право обрабатывать данные по просмотрам медийной рекламы на портале, что позволило создать уникальное на рынке решение "Postview-аналитика портала ЦИАН"
- MGCom - партнер по развитию продукта postview + потенциально канал продаж (наиболее значимое performance-агентство в России).
- Profitbase - маркетинговый и "идеологический" партнер.
- Trace IT - потенциальный партнер нового типа (интегратор). Валидируем на нем модель работы с партнерами такого типа (нашим коллегам из Profitbase, для примера, интеграторы приносят >50% продаж).

Система продаж

Наша модель продаж - Solution Selling. Ключевые компоненты:
- Продажа как проект (ведение сделки = управление проектом).
- Взаимодействие со всей командой на стороне клиента (модель "4 уровня ценности")
- Квалификация клиента.
- Демо решения.


Наш метод продаж - это экспертные продажи.
- Эффективно продавать в нашем целевом сегменте может только эксперт в нашей предметной области. Поэтому наш продавец - это эксперт.
- Три уровня экспертности.

  1. Экспертность в продукте (проблемы-решения, кейсы).
  2. Экспертность в рынке (тренды, челленджи, цифры)
  3. Экспертность в компании-клиенте (инфополе в компании и вокруг нее).

- Наш инструмент "прогрева" рынка - это "маркетинговые касания".


Поддержка и развитие клиентов

Ключевые принципы

- Не врем - всегда выполняем свои обязательства
- Придерживаемся идеологии Customer Success - фокус на ценности нашего продукта для клиента и развитии клиента через повышение ценности.
- Наши клиенты - не компании, а люди.
- Каждый клиент - это проект (или набор проектов)
- Помним, что клиент постоянно дрейфует к уходу (и предотвращаем это).

Модель развития клиента - ключевые компоненты

1. Модель развития клиента через цикл повышения ценности

2. Профиль клиента (профиль клиентской базы)
- Фиксируем и постоянно актуализируем свои знания о клиенте.

3. Карта здоровья клиента

4. Процесс ведения проектов

5. Риски ухода клиента, которыми надо управлять
