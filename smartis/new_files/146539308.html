При организации рабочего пространства необходимо соблюдать следующие правила. 

  1. Под каждого клиента в Asane создаётся "Портфель"
  2. В рамках каждого клиента создаётся проект
    - "Основной проект" - это проект для операционных задач/текущих задач в него НЕ ПРЕДОСТАВЛЯЕМ доступ клиенту; 
  3. Все проекты по клиенту добавляются в портфель клиента; 
  4. При необходимости под клиентом можно создавать дополнительные проекты, в который по согласованию с руководителем можно предоставить доступ представителю клиента. 
    В каком случае может понадобиться дополнительный проект: 
    - если с клиентом решается задача, которая не может быть выполнена типовыми средствами и для её решения требуется привлекать дополнительных ресурсов. 
    По сути мы выносим набор задач если у нас возникает с клиентом договорённость о старте отдельного проекта. 
    Проект — это временное деятельность, направленная на создание уникального продукта, услуги или результата. 
    Проекты ограничены по времени. У них есть чётко очерченные начало и конец.

Правила наименования портфелей и проектов: 

  1. Портфель - это по сути папка в котором собираются проекты по одному клиенту. 
    При создании портфеля используем следующее наименование SMA или SCB/<Наименование клиента>
  2. Проекты - могу присутствовать в нескольких портфелях одновременно если это необходимо. 
    При создании проекта используем следующее наименование SMA или SCB/<Наименование клиента>/ Основной проект
