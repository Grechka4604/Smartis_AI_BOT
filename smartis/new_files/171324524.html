Основная информация

CalltoVisit - сервис бесплатного заказа такси. Сервис достаточно популярен среди застройщиков, так как офисы продаж статичны и важным этапом конверсии в воронке застройщика является визит в офис продаж. 

Какие данные получаем из CalltoVisit:

  • заказы
  • расходы на такси

Обновление данных

Название: CallToVisit: синхронизация данных по заказам
ID: 48
Статус: Активен
Запуск: Раз в день
Условия запуска: Запуск раз в сутки в 1 час


Подключение интеграции

  1. Запросить доступы у клиента для подключения интеграции CallToVisit (также клиент может подключить сам, не передавая доступы вам). Нужен логин и API ключ из CallToVisit. API-ключ однозначно определяет для какого объекта будет создан заказ на такси. API ключ можно найти на странице в CallToVisit
  2. В Smartis перейти в раздел Администрирование → Интеграции и в разделе "Доступные для подключения" выбрать интеграцию CallToVisit


  3. Нажать "Подключить" и заполнить данные для подключения. В поле формы API-ключ ввести  Api-токен полученный в сервисе
  4. После подключения интеграция отобразится в разделе "Подключенные интеграции" (Администрирование → Интеграции).После подключения в клиенте интеграции с сервисом CallToVisit запускается первичная синхронизация данных

Данные по интеграции после подключения автоматически забираем за 30 дней в прошлое, с момента подключения интеграции. Если нужно обновить другой период - воспользуйтесь ручным обновлением  тут: 9. Администрирование: Мониторинг системы

Привязка интеграции к сайту

Пока интеграция не привязана к сайту, данные по ней невозможно просматривать в отчете по данному сайту в Smartis

  1. Зайти в раздел Администрирование → Проекты и сайты → Сайты и найти в списке нужный сайт. 
  2. Кликнуть на название сайта - откроется меню настройки привязок интеграций к сайту. 
  3. В разделе "Интеграции доступные для привязки" найти интеграцию CallToVisit и нажать "Привязать с логином".
  4. Из выпадающего списка выбрать логин, по которому приходят заказы для выбранного сайта (можно выбрать только 1 логин, если к сайту нужно привязать несколько логинов - пункт 3 и 4 выполняются столько раз, сколько логинов нужно привязать. После выбора логина интеграция привяжется к сайту. 

  5. Если нужно изменить логин, по которому получаете данные из CallToVisit, можно в уже привязанной интеграции выбрать другой логин и подтвердить изменения.
  6. Для каждой привязанной к сайту интеграции нельзя дополнительно настроить условия привязки. При подключении по сайту будут поступать все данные из выбранного логина


Отчеты по данным CallToVisit

Показатели

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. Также показатель Кол-во заказов CallToVisit добавляется в формулу для системного показателя "Все обращения (новый)"

  1. Кол-во заказов CallToVisit - отображает количество заказов такси через форму на сайте
  2. Расход на такси CallToVisit - отображает сумму расхода по заказам такси


Фильтры для CallToVisit


Для данных CallToVisit доступны системные и особые фильтры (особые применяются только к данным CallToVisit):


  • CalltoVisit: Статус заказа - фильтрует по статусу заказа (выполнен, запланирован, на модерации и т.д.)

    Также для данных из этого сервиса доступны все фильтры, которые появляются в выпадающем списке при выборе показателей с данными CallToVisit

Детализация данных CallToVisit

Основные свойства, передаваемые в обращениях CallToVisit:

Привязка источника обращения CallToVisit в Smartis

Когда мы получаем данные из CallToVisit, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis. Определение источника обращения происходит так:

  • Если застрощик (его менеджеры) сами вручную вызывают такси по заявке - мы получаем источник предыдущего обращения из сервиса коллтрекинга и присваиваем заявке CallToVisit такой источник.
  • Если CallToVisit используется как виджет на сайте - то мы получим информацию о utm метках и определим рекламный источник на основе этих данных. Привязка к площадке выполнится автоматически по прописанным к ней условиям Smartis (параметрам из utm)

О том, как работают привязки обращения по рекламным кампаниям колл-трекинга к площадкам в Smartis читать тут: 2. Каналы и площадки

В СJM клиента обращения из CallToVisit отображаются так:

Отображаются:

  • тип обращения (как создана заявка) 
  • статус (запланирован и тд)
  • телефон клиента 
  • Канал и Площадка (на основании того, к какой площадке мы привязали предыдущие обращения из колл-трекинга)
  • Регион
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в CallToVisit теги не передадутся.
