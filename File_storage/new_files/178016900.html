Основная информация

Callibri https://callibri.ru/ - основанная функция сервиса это коллтрекинг, так же имеются сбор заявок из различных сервисов, онлайн чаты и сквозная аналитика.

В Smartis можно подключить интеграцию с Callibri в качестве сервиса обращений (то есть для получения из Callibri заявок, звонков). Интеграция нативная, работает автоматически.

Какие данные получаем из Callibri:

  • звонки
  • заявки
  • чаты

Данные из Callibri содержат информацию о том, с какого источника (рекламного, органического) "пришел" клиент (совершил обращение). На основе информации о рекламном источнике обращения строится аналитика по CRM данным в разрезе каналов, площадок, кампаний и т.д.
То есть, на основании данных об источниках обращений из Callibri в Smartis работает атрибуция CRM данных (в зависимости от выбранной Модели Атрибуции)

Подключение интеграции

  1. Для того, чтобы подключить интеграцию с Callibri, необходимо от имени авторизованного пользователя, с доступом уровня Аналитика и Настройка зайти в кабинет Callibri. Далее получить ваш уникальный токен. Для этого зайдите в настройки своей учетной записи (имя в левом верхнем углу личного кабинета) и нажмите кнопку Получить доступ к API:



    После этого Callibri отправит инструкции и методы на email-адрес, указанный в аккаунте Callibri.

    Клиенту не обязательно выдавать нам доступ к кабинету Callibri, создавать пользователя для нас. Он может передать нам API-токен для интеграции. 
    НО! Доступ к кабинету облегчит получение информации для сверки и отладки, когда это может потребоваться.


  2. Далее для подключения интеграции в кабинете Callibri необходимо пройти в раздел Администрирование/Интеграции и среди доступных для подключения выбрать Callibri

  3. Для настройки интеграции необходимо задать: 

    1. Наименование интеграции - может быть любым, но лучше указывать название сайта, к которому относится интеграция. 
    2. API токен - забираем из письма от Callibri (передается в зашифрованном виде)
  4. После подключения интеграции она переместится в раздел "Подключенные интеграции"

  5. Если в интеграции изменится логин и/или пароль для доступа, то в разделе интеграций у интеграции Callibri будет отображаться надпись "Есть ошибки", при клике на шестеренку с настройками будет отображаться текст ошибки:

    "Callibri: Доступ запрещен". Это значит, что интеграцию нужно переподключить с актуальными авторизационными данными.

Привязка интеграции к сайту

Для того, чтобы данные по обращениям отобразились в статистике проекта в Smartis, необходимо привязать интеграцию к нужному проекту. Данные мы всегда привязываем к базовой единице - сайту.

  1. Для привязки необходимо пройти в раздел "Проекты и сайты", выбрать нужный сайт и зайти в его настройки. 
  2. В доступных для привязки интеграциях необходимо выбрать интеграцию Callibri и "привязать ее с логином" к сайту. Логин в данном случае - это сайт в Callibri, внутри которого настраивается подмена и рекламные кампании.   

  3. Из выпадающего списка выбрать логин, по которому приходят обращения для выбранного сайта (можно выбрать только 1 логин, если к сайту нужно привязать несколько логинов - пункт 2 и 3 выполняются столько раз, сколько логинов нужно привязать, если нужно привязать все логины интеграции - выбрать в списке "Все данные этой интеграции"). После выбора логина интеграция привяжется к сайту. 

  4. Если нужно изменить логин, по которому получаете данные из Callibri, можно в уже привязанной интеграции выбрать другой логин и подтвердить изменения.
  5. Иногда данные по разным проектам могут находится внутри одного сайта (логина) Callibri. Например, когда у клиента зонтичный бренд, а каждый ЖК - отдельная страница или раздел внутри одного сайта. В таком случае мы должны перейти в настройку привязкок в привязанной к сайту в Smartis интеграции. 


    Пример: в логине "Застройщик" есть информация о звонках по ЖК1, ЖК2 и Бренду. Нам нужно сделать так, чтобы к сайту ЖК1 были привязаны только те обращения, которые поступают по этому ЖК.
    Как это можно сделать:
    1. Если в названии рекламной кампании в Callibri указаны названия ЖК, то можно прописать условие (в виде регулярного выражения) в поле Название РК соответствует регулярному выражению (учтите все возможные варианты написания - на русском и английском языке, сокращения и прочее)
    2. Выбрать конкретные рекламные кампании Callibri в поле "Выбор конкретных кампаний" (с этим пунктом будьте осторожны, если у клиента добавятся новые рекламные кампании - условия привязки нужно будет актуализировать.  Поэтому вариант из пункта "а" более надежный
    3. Посадочная страница соответствует регулярному выражению - подходит в случае, когда в названии кампании не указано название ЖК, но обращения по данному ЖК поступают на определенный сайт.  Url страницы обращения также указываем в виде регулярного выражения. 

    4. Метка utm_campaign соответствует регулярному выражению - если нужная метка указывается в utm_campaign (и например не всегда указывается в названии кампании) то можно использовать этот способ. С помощью регулярного выражения нужно указать все возможные значения меток.



Показатели


При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. Также показатели Callibri звонки, Callibri заявки добавляются в формулу для системного показателя "Все обращения (новый)"


  1. Callibri звонки - отображает данные по звонкам (источник звонка определяется на основе данных рекламной кампании Callibri, по которой пришел звонок)
  2. Callibri заявки - отображает данные по заявкам (источник заявки определяется на основе данных рекламной кампании Callibri, по которой пришла заявка)
  3. Callibri чаты - отображает данные по чатам (источник заявки определяется на основе данных рекламной кампании Callibri, по которой пришел чат)
  4. Callibri звонки (по моделям атр.) - отображает данные по звонкам, которые строятся по моделям атрибуции- то есть источник звонка определяется по-разному в зависимости от выбранной в отчете модели атрибуции. Про модели читать тут: 4. Модели Атрибуции
  5. Callibri заявки (по моделям атр.) - отображает данные по заявкам, которые строятся по моделям атрибуции- то есть источник заявки определяется по-разному в зависимости от выбранной в отчете модели атрибуции. Про модели читать тут: 4. Модели Атрибуции
  6. Callibri чаты (по моделям атр.) - отображает данные по чатам, которые строятся по моделям атрибуции- то есть источник чата определяется по-разному в зависимости от выбранной в отчете модели атрибуции. Про модели читать тут: 4. Модели Атрибуции

Детализация данных Callibri


Для данных Callibri доступна детализация (исходные данные). Они такие же, как у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.


Основные свойства, передаваемые в обращениях Callibri:


  • Данные о свойствах обращения из Callibri: внешний ID, сайт, дата создания, запись разговора, длительность разговора, теги, номер обращения в сервисе (первичность), качество обращения в сервисе и т.д.; 
  • Данные о клиенте: номер телефона, email, город, регион, устройство и т.д.;
  • Данные о рекламном источнике обращения: РК трекинга, параметры сессии, utm_метки, gclid, yclid, страница входа, страница обращения.

Привязка источника обращения Callibri в Smartis


Когда мы получаем данные из Callibri, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis (из Callibri мы получаем названия рекламных кампаний и utm метки, на основе чего и выполняются привязки).

О том, как привязать обращения по рекламным кампаниям Callibri к площадкам в Smartis читать тут: 2. Каналы и площадки

Если обращения из Callibri не привязаны (автоматически по названию кампании или вручную), то они будут попадать в площадку "Другую рекламу" в отчете по площадкам/каналам.

В СJM клиента обращения из Callibri отображаются так:

Отображаются:

  • тип обращения (звонки, чаты или заявки) 
  • статус (принятый/пропущенный)
  • телефон клиента 
  • Регион
  • Канал и Площадка (на основании того, к какой площадке мы привязали данные обращения)
  • РК треккинга - это название кампании в Callibri. 
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в Callibri теги не передадутся.
  • страница входа
  • Yandex client ID
  • Google client id
