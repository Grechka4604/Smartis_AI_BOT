Организационный процесс:

  1. Поставить задачу аналитику не менее чем за неделю до завершения. Или составить с ним график, когда ему собирать данные.
  2. Проверить корректность данных в ТЗ
  3. Договориться о сборе комментариев с CS и Repa. Сообщить им даты когда понадобиться их помощь
  4. Получение данных, сбор письма. Проверка работы дизайнеров.
    1. Соответствие фирменному стилю
    2. Корректность данных
  5. Собрать комментарии с помощью CS и через наших партнеров. Написать вопросы, которые нужно задать.
    1. При необходимости согласовать комментарии
  6. Проверить сегментацию базы для рассылки и провести апдейт данных по необходимости
  7. Занести в рассылку в календарь рассылок комеджик
  8. Собрать финальную версию с комментариями.
  9. Согласовать с Димой К
  10. Отправить черновую версию
  11. Отправить чистовую версию по базе клиентов
  12. Отправить чистовую версию по базе сотрудников
  13. Через 3 дня собрать отчетность 
  14. Публикация на Repa

Процесс создания Road Map:

  1. Собирать лучшие идеи предоставления аналитики.
    1. Перечислить источники где брать материалы для идей. Pinterest и т.п
  2. Проконсультироваться с узкими специалистами
    1. Список консультантов или подрядчиков под ключ
  3. Собирать раз в 2 месяца, фидбэк у 5 клиентов кто получает рассылки
    1. Составить список вопросов
  4. Добавить новые идеи в Road map email дайджеста
    1. Определиться как будет выглядеть Road Map
      1. Структура письма
      2. Ключевые цифры аналитики которые мы хотим показать
      3. Варианты визуализации информации взять из задач №1 и №2
        1. Заказ шаблона визуализации, при необходимости
