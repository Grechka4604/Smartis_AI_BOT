Цель: предоставление комфортных условий сотрудничества для клиента и получение официального подтверждения сроков оплаты у клиента.

Участники процесса: аккаунт-менеджер, GH, руководитель направления, ДОУ

Инициатор: клиент

Инфо поле: Jira, файл с отсрочкой платежа, Infopin

KPI процесса: 

KPI 1 - фиксация всех клиентов с невозможностью оплаты в срок прописанный в договоре ООО НОВОСИСТЕМ. 

Как контролируем:  ежемесячно при подготовке отчета по ДЗ опираемся на перечень клиентов с отсрочкой платежа.

Результат: результатом выполнения процесса будет отсутствие дебиторской задолженности по итогам отчётного периода

Владелец процесса: руководитель направления

Статус процесса:  работает. 


Краткое описание процесса.

Согласно договору клиенты обязуются производить оплату по закрывающим документам в теч. 21 дня следующим месяцем за отчетным.

В случае, если у клиенты есть весомые причины по которым он не успевает произвести оплату в указанный срок необходимо с ним заключить Доп. Соглашение на пролонгацию даты оплаты.

Для запуска процесса необходимо:

  1. Аккакнт- менеджер создает заявку в Infopin с подробным описанием причины необходимости отсрочки
  2. Направляет заявку на согласование руководителю направления
  3. После согласования заявка перевешивается на отдел ДОУ
  4. После формирования ДС отделом ДОУ клиенту направляется ДС на подписание и производится обмен оригиналами подписанных с 2х сторон ( клиент и НОВОСИСТЕМ)
  5. После получения оригинала ДС ответственный ( инициатор заявки) вносит клиента и период отсрочки в перечень клиентов с отсрочкой.
