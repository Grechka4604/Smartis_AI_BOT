Процесс перезаключение контрактов.

Согласование SLA по ключевым операциям

Определили ключевые пункты, которые будут блокировать нас по скорости решения задачи по перезаключения контракта. По данным ключевым пунктам определили SLA:

  1. Добавление карточки в инфопине - 2 дня (от момента обращения, до момента появления готовой карточки в инфопине);
  2. Активация оболочки - в день обращения (при наличии в карточке клиента подписанного договора с стороны клиента, как минимум скана);

Дополнительно проговорили формат обучения сотрудников.

Формат обучения предполагает конкретный разбор процесс с сотрудником + наставничество/помощь при переводе первого клиента.

По шагам это будет выглядеть следующим образом:

  1. Куратор назначает встречу с PM
  2. Куратор рассказывает информацию по процессу, контрактам, где находиться информация и т.д..
  3. На примере конкретного клиента PM предоставляет куратору обратную связь как он будет действовать при переводе клиента, куратор корректирует при необходимости.
  4. Фиксируем дедлайн перевода.
  5. Предоставляем информационную поддержку в процессе перевода.
  6. Фиксируем факт перевода.
  7. Анализируем причины отклонения, при необходимости вносим изменения в процесс
