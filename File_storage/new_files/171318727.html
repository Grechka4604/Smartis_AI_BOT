Задача функционала:

  1. Внести вручную значения по площадкам по определенному сайту

  2. Проверить внесенные данные по площадкам

Для чего на практике используется функционал:

  • Добавить сумму Агентской комиссии к площадкам, расходы по которым получаем по API из интеграций (Директ, ВК, Майтаргет)
  • Внести данные по расходам или по другим показателям по площадке вручную
  • исправить точечно ранее импортированные данные (например, чтобы не перезагружать весь файл, можно исправить значение по 1 площадке)

Как работает функционал:

Зайти в Импорт данных → Анализ корректности

Открывается вкладка Анализ корректности

  • анализируемый проект - всегда нужно выбирать только сайт (при выборе Объекта или Проекта анализ корректности не будет работать). Это связано с тем, что данные всегда привязаны к сайту - как к наименьшей "единице" в иерархии Smartis.
  • Плановый показатель - выбираем импортируемый показатель (с иконкой экселя рядом с названием). Если выбрать показатель, созданный по формуле, то изменить его данные не получится.
  • Группировать результат: можно по дням и по месяцам.
    Группировка по дням используется для внесения или редактирования импортированных данных по дням (например, если нужно исправить внесенный план бюджета за какие-то определенные числа месяца).
    Группировка по месяцам используется для внесения или редактирования импортированных данных по месяцам (если нужно исправить или внести данные сразу за 1 или несколько месяцев).
    Данные, которые внесли сразу за 1 месяц будут разнесены по дням равномерно.

    При группировке по месяцам можно выбрать несколько месяцев или 1 месяц - данные отобразятся как на скрине (1 столбец = 1 месяц)


    При группировке по дням данные отображаются так: 1 столбец = 1 день (в заголовке столбца указано число месяца)


  • Период анализа: можно выбрать любой. Если выбираете группировку по дням - кол-во дней в Периоде можно выбрать любое. Если группировка по месяцам - то вносите данные с выбранным периодом в 1 календарный месяц или более. Если вы выберете период например 1.10-27.10 и группировку по месяцам, то данные внесутся не в весь октбярь, а в 1-27 октября и в рамках этого перпиода распределятся автоматически равномерно.

Занесение и корректировка данных

Если ранее по сайту в выбранном никогда не вносили данные импортом, то в анализе корректности значения по нему не отобразтся.

Что можно сделать в этом случае, чтобы данные отобразились:

  • выбрать бОльший период (например, если данные в мае не вносили, то построить таблице за март-май). В таком случае если данные вносили в марте или апреле, таблица отобразится, в мае везде будет 0
  • на вкладке "занесение данных" вручную или из файла загрузить данные в площадку или площадки за нужный период (Инструкция  )

Когда данные в анализе корректности за нужный период есть, их можно редактировать или внести информацию по незаполненным площадкам.
Чтобы внести или исправить внесенные данные, нужно кликнуть на числовое значение в столбце напротив площадки 1 раз - появится окно редактирования. нужно ввести нужное значение и нажать Enter

После того, как нажали Enter данные сохранятся и появится сообщение "Сохранено". если оно не появилось - данные не сохранились.

Рядом с некоторыми строками могут отображаться восклицательные знаки - они подсвечивают аномалии - большие расхождения в данных по сравнению с прошлыми периодами, в которых есть данные. Например, если у вас в прошлом месяце был внесен расход 1 000 000 руб., а в этом - 0, то подсветится аномалия. Убрать этот знак никак нельзя (конечно если только вы не введете данные, которые будут примерно такими же как в прошлом периоде)

также есть возможность добавить комментарий к внесенному значению - например, указать что вы его скорректировали.

После того, как вы внесли нужные изменения в таблицу и каждое изменение сохранилось, не нужно дополнительно что-то сохранять - можно закрыть раздел и перейти в конструктор отчетов - если нужно еще раз убедиться, что в отчете при построении нужного показателя изменения отразились.
