ШагиОжидаемый результат
1
  1. Добавить фильтр через кнопку "+"
  2. Выбрать из выпадающего списка фильтр "Источники в Smartis: Площадка"
  1. Появляется поиск по фильтрам с выпадающим списком фильтров
  2. После выбора фильтра появится окно выбора условия фильтрации и список площадок
  3. При выборе условия "Включает" - в выпадающем списке площадок можно выбрать несколько площадок
  4. Отчет с фильтром успешно строится, показываются только выбранные площадки (удобно в группировке по площадкам)
  5. Рядом с выбранным фильтром указано, к каким показателям он применится (для фильтра площадок - должен применяться ко всем показателям)
2
  1. Добавить фильтр
  2. Выбрать условие
  3. Ввести в строке невалидное значение
Отчет не строится, но ошибку не выдает
3
  1. Выбрать фильтр "Источники в Smartis: Наименование кампании"
  2. В фильтрации добавить условие "соответствует/не соответствует регулярному выражению"
  3. построить отчет по кампаниям сначала с пустым фильтром (отчет должен построиться и фильтр не применяться)
  4. Выбрать среди наименований кампаний из отчета какое то имя и ввести в фильтр регулярку с ним, например для Кампании "Брендовые запросы ФСК" => ".*бренд.*"
  5. Построить отчет - в нем должны остаться только кампании содержащие введенное регулярное выражение
  6. Сменить условие на Не соответствует регулярному выражению - из отчета должны исчезнуть кампании содержащие введенное значение
  1. В параметре фильтрации можно ввести значения вручную
  2. отчеты строятся
  3. кампании в отчете соответствуют введенным параметрам фильтра
4
  1. выбрать в показателях Все обращения (новый)
  2. Построить в группировке День с типом визуализации "Исходные данные"
  3. В отчете, в столбце "тег" найти какие теги есть у звонков
  4. в фильтрах найти "Наименование тега", ввести значение тега из обращений, построить отчет
  5. выбрать другой фильтр (например у фск) CoMagic: Теги и (например у 1944) CallTouch: Теги
  6. выбрать любое значение, построить отчет
  1. отчет с фильтрацией по наименованию тега строится корректно
  2. в сырых данных в колонке тег - везде присутствует фильтруемое имя
  3. в выпадающем списке значений фильтра CoMagic: Теги ИЛИ CallTouch: Теги - есть теги
  4. отчет с фильтром CoMagic: Теги ИЛИ CallTouch: Теги - строится без ошибок (проверять отчета на соответствие фильтру не надо, это не тривиально)
5
  1. Выбрать в показателях "Все обращения (новый)"
  2. Выбрать группировку "День"
  3. Выбрать какую-нибудь неделю
  4. Добавить фильтр "Календарь: Особые настройки периода" и выбрать "Сместить на 2 недели в прошлое"
  5. Построить отчет.
  6. В новой вкладке построить такой же отчет, но в выборе дат выбрать на 2 недели раньше
Результаты в двух вкладках должны совпадать
6
  1. Выбрать показатель "Деление показателей с фильтром "Особый период"" (ЮСИ)
  2. Выбрать показатели составляющие формулу показателя выше
  3. Построить отчет
Число по выбранному показателю равна частному по составным показателям (результат деления)
7
  1. Зайти в клиента ЮСИ
  2. Построить отчет по показателям "Динамика общие лиды (мес)", "Кол-во сделок" и "Кол-во сделок с фильтром сместить на прошедший месяц"
  3. Выбрать 3 последних месяца
  4. Выбрать группировку Месяц
  1. Каждое значение "Динамика общие лиды (мес)" должна равняться "кол-во сделок"/"Кол-во сделок с фильтром сместить на прошедший месяц" (должна соответствовать формуле описанной в настройках показателя)
  2. Значение "Кол-во сделок с фильтром сместить на прошедший месяц" = значению показателя "кол-во сделок" за предыдущий месяц
8
  1. Создать отчет за определенный период
  2. Выбрать сущность CRM (например, "кол-во сделок")
  3. Выбрать одну из сделок в построившемся отчете и посмотреть дату закрытия сделки
  4. Добавить условия фильтра в отчете: "Дата отображения в отчете" Равно "Дата закрытия"
  5. Выбрать период куда входит дата закрытия сделки
  6. Заново построить отчет
Фильтр меняет отображение в отчете по периоду: в отчете договор будет отображаться по дате его закрытия. С таким фильтром в выбранном периоде будут показаны те договоры, у которых дата закрытия находится в выбранном периоде. Если какой-то конкретный договор не попал в отчет - проверить, какая у него дата закрытия - скорее всего, она в другом периоде
9
  1. Выбрать id одной сделки и построить по нему отчет: Поиск по внешнему id - Равно - id
  2. Выбрать период (например, текущий месяц)
  3. Построить отчет
В отчете отобразится данная сделка
10
  1. Добавить фильтр: Дата фильтрации периода - Равно - Дата визита
  2. Выбрать другой период (например, другой месяц)
  3. Построить отчет
Сделка теперь не попадает в отчет, тк у нее дата визита не в выбранном периоде
11
  1. Выбрать показатель обращений
  2. Добавить фильтр, условие - сумма значений поля - параметр фильтрации "Длительность разговора"
  3. Построить отчет
В отчете отображается длительность разговора (сумма по всем обращениям)
12
  1.  Выбрать показатель "Сделки"
  2. Добавить фильтр "логика подсчета значения" - сумма значений поля - "сумма сделки"
  3. Построить отчет
В отчете отобразилась сумма всех сделок за каждый день
13
  1.  Выбрать показатель "Сделки"
  2. Добавить фильтр "логика подсчета значения" - количество уникальных значений поля - "CRM element person ID"
  3. Построить отчет
В отчете отобразится сколько клиентов купили недвижимость в выбранном месяце
14
  1. Выбрать показатели в отчете
  2. Выбрать фильтры, выделенные в этом списке
  • Строится отчет по выбранным фильтрам
  • Фильтры не обнуляют отчет
  • Результат отчета соответствует указанным в фильтре параметрам
15
  1. Выбрать показатель"Сделки"
  2. Выбрать фильтр Когорта Person
  3. Выбрать условие "равно" - так как нужно найти сделки, перед которыми были обращения с медийной рекламы
  4. В качестве параметра фильтрации указать обращения
  5. Период: указать период, за который происходили обращения. Например, месяц
  6. Добавить фильтр по каналу "Медийная реклама"
Результаты отчета строятся
  1. по признакам: совершили обращение 
  2. по времени - в указанный месяц
    Видим поведение этой когорты до сегодняшнего дня - поэтому и отчет выстроен до сегодняшнего дня, а не заканчивается в указанный месяц.
16
  1. Выбрать любую сущность в показателях (например, Встречи)
  2. Добавить фильтр "Поле связанной сущности"
  3. Выбрать поле связанной сущности, по которому нужно фильтровать (Встречи → Сотрудники)
  4. Выбрать условие фильтрации (например, "равно") 
  5. Указать параметр фильтрации: выбрали конкретного менеджера
Нажимаем "построить отчет" - и получаем визиты, которые проводил выбранный ответственный. Фильтр применяется и фильтрует данные по заданной логике
17
  1. Выбрать показатель ( Целевые обращения, или например: встречи, договоры)
  2. Выбрать фильтр: Когорта персон
  3. Выбрать условие фильтра: нет последующего события → (например, кол-во лидов, все обращения или посетители smartis)
  4. Построить отчет
  5. Открыть детализацию

В отчете должны отображаться только звонки, после которых не было лида

18
  1. Выбрать показатель любую сущность (например, договоры)
  2. Выбрать фильтр Когорта персон → Есть последующее событие → Все обращения
  3. Построить отчет
  4. Открыть детализацию
Попадут в отчет данные: персоны, у которых есть договор-купли продажи в указанном периоде и звонки в любом периоде с начала указанного периода до сегодняшнего дня и обязательно после договора купли-продажи. То есть мы смотрим персон, у которых есть договор купли-продажи в феврале, и после договора есть звонки с февраля по сегодняшний день. В отчет добавятся договоры купли-продажи, зафиксированные в указанный период, после которых есть звонки до сегодняшнего дня.
19
  1. Выбрать случайные 5+ фильтров AmoCRM
  2. Выбрать проект "включает" и хотя бы 2 проекта
Отчет строится корректно и все фильтры применяются
