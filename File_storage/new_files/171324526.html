Основная информация

Rrofitbase - интерактивный каталог объектов/квартир на сайте застройщика, плюс сбор бронирований в crm застройщика. 

Интеграция в Smartis с этим сервисом работает для получения заявок, которые пользователи отправляют через инструменты profitbase (из сервисы мы получаем заявки).

Интеграция подключается через API-токен, но если из сервиса нужно получать только данные о заявках - нужно также передать в profitbase данные для подключения через webhooks. 


Подключение интеграции

  1. Запросить доступы у клиента для подключения интеграции profitbase (также клиент может подключить сам, не передавая доступы вам). Нужно название аккаунта в profitbase, поддомен аккаунта в profitbase, API токен из profitbase. API-токен Токен можно получить у вашего менеджера profitbase или в личном кабинете сервиса.
  2. В Smartis перейти в раздел Администрирование → Интеграции и в разделе "Доступные для подключения" выбрать интеграцию profitbase

  3. Нажать "Подключить" и заполнить данные для подключения. В поле формы API токен из profitbase ввести  Api-токен полученный в сервисе. 


  4. Если нужно получать обращения из profitbase (через webhook), то далее нужно скопировать url для приема webhook, который Смартис автоматически сгенерировал. Он нам понадобится для настройки уже в кабинете profitbase. Для настройки в кабинете profitbase зайдите в настройки виджета Смарт-каталога и во вкладке Заявки, Webhook по новым заявкам нажмите Добавить webhook.  Укажите URL вашего скрипта и выберите объекты, по которым будет отрабатывать вебхук.

    Формат:

    { "source" : "http://web.profitbase.ru", // ссылка, откуда была отправлена заявка "property" : { // данные о квартире "id" : // id квартиры "number" : 10 // номер "rooms_amount" : 2, // кол-во комнат "price" : 2500000, // цена "area" : 56.5, // площадь "house" : "ЖК Пример IV секция", // ЖК, дом "house_id" : 10 // id дома }, "customer" : { // данные клиента "name" : "Анатолий", // имя "phone" : "+7 900 1234567", // телефон "email" : "example@example.com" // email }, "comment" : "Хочу купить", // комментарий в заявке "tracking" : { // трекинговые данные, могут быть пустые "roistat" : "123456", } }

    Справка в Profitbase: https://help.profitbase.ru/article/34388


  5. После подключения интеграция отобразится в разделе "Подключенные интеграции" (Администрирование → Интеграции).


Привязка интеграции к сайту


Пока интеграция не привязана к сайту, данные по ней невозможно просматривать в отчете по данному сайту в Smartis


  1. Зайти в раздел Администрирование → Проекты и сайты → Сайты и найти в списке нужный сайт. 
  2. Кликнуть на название сайта - откроется меню настройки привязок интеграций к сайту. 
  3. В разделе "Интеграции доступные для привязки" найти интеграцию profitbase и нажать "Привязать с логином".

  4. Из выпадающего списка выбрать логин, по которому приходят заявки для выбранного сайта (можно выбрать только 1 логин), если к сайту нужно привязать несколько логинов - пункт 3 и 4 выполняются столько раз, сколько логинов нужно привязать. После выбора логина интеграция привяжется к сайту. 
  5. Если нужно изменить логин, по которому получаете данные из profitbase, можно в уже привязанной интеграции выбрать другой логин и подтвердить изменения.

  6. Для привязанной к сайту интеграции нельзя дополнительно настроить условия привязки. 



Отчеты по данным Profitbase

Показатели

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. 


  1. Profitbase заявки - отображает данные по заявкам (источник определяется на основе данных рекламной кампании Profitbase, по которой пришла заявка)


Фильтры для данных Profitbase

Применяются все системные фильтры, которые доступны в выпадающем списке при выборе показателей данного сервиса.

Также есть отдельный системный фильтр для данных из этого сервиса:


  • Profitbase: тип обращений - позволяет отфильтровать заявки по типу (webhook и другие)

Детализация данных profitbase

Для данных profitbase доступна детализация (исходные данные). Они такие же, как данные по заявкам у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.


Общая справка по детализации данных сервисов обращений и заявок:
3. База обращений
6. Исходные данные (Детализация, Сырые данные)



Основные свойства, передаваемые в заявках profitbase:

  • Данные о свойствах обращения из profitbase: внешний ID, сайт, дата создания,  теги
  • Данные о клиенте: номер телефона, регион, и т.д.;
  • Данные о рекламном источнике обращения: utm_метки, страница входа


Привязка источника обращения profitbase в Smartis


Когда мы получаем данные из profitbase, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis (из profitbase мы получаем utm метки, на основе чего и выполняются привязки по условиям площадок). О привязках по условиям к площадкам в Smartis читать тут: 2. Каналы и площадки

Если обращения из profitbase не привязаны к площадке, то они будут попадать в площадку "Другое" в отчете по площадкам/каналам.


В СJM клиента обращения из profitbase отображаются так:


Отображаются:


  • тип обращения 
  • телефон клиента 
  • Канал и Площадка (на основании того, к какой площадке мы привязали данные обращения)
  • utm метки - мы их также получаем из profitbase
  • Страница входа - страница сайта, на которую зашел пользователь перед совершением обращения
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в Marquiz теги не передадутся.


Достоверность profitbase

Достоверность по сервису не обращается, тк интеграция через webhook.
