Основное описание

Это классифайд, специализируется на недвижимости. На этой площадке есть возможность размещать медийную/охватную рекламу. У нас нативная интеграция с сервисом по API.

Интеграция Novostroy.ru и Smartis позволяет:

  • получать в Smartis данные о просмотрах медийных размещений на данной площадке

  • матчить их с CJM клиента.

id сервиса в Smartis - 63.

По всем сервисам Postview работает 1 синхрон: описан тут 2. Post-view

Как подключить

  1. Нам нужно запросить у Novostroy.ru. токен для конкретного клиента (То есть они выделяют нам доступ под каждого конкретного клиента). Токен запрашиваем у площадки мы, в чате телеграм. Присылают его в течение 1-2 дней. В Smartis данные по просмотрам с этой площадки поместяся в тот ЛК, в котором будет создана интеграция с этим токеном. 
  2. Токен нужно прописать в поле API-токен при подключении интеграции. (Чтобы подключить, переходим в Администрирование → Интеграции → Доступные для подключения и выбираем плашку Novostroy.ru. По клику на кнопку "Подключить" откроется меню для ввода названия интеграции и API-токена).


  3. Нажать "Подключить" - после чего интеграция переместится в раздел "Подключенные интеграции"

    После того, как создана новая интеграция, начинают генерироваться планы для скачивание показов с начала прошлого месяца. 

    Важно:     У Novostroy.ru нет обязательств поддерживать все исторические данные - они хранят их только за последние 30 дней. Поэтому исторические данные за период более 30 дней в прошлое восстановить по этому сервису невозможно. Проверка работы интеграции должна происходить по данным с момента подключения интеграции.

После того, как интеграция подключена, ее не нужно привязывать к сайтам.


Как работает интеграция:

  • происходит процедура cookie-матчинга (как в интеграциях cian, novostroy-m)
  • мы определяем, что в клиенте подключена интеграция с площадкой → любая сессия на сайтах этого личного кабинета клиента подгружает невидимый пиксель/картинку, который производит матчинг (сопоставляет идентификатор наш - pid, и идентификатор Novostroy.ru)

Что важно делать для проверки работы интеграции:

  • динамическая разметка колл-тркинга по площадке novostroy-m
  • важно, чтобы большая часть онлайн трафика была размечена динамически (чтобы больше пользователей имели связку телефон+cookie)
    Это обеспечит максимальный матчинг данных и корректную Postview аналитику.
  • в конце каждого месяца проводить сверку полных прошедших дней (чтобы убедиться, что все форматы медийки охвачены, данные сходятся)

Особенности работы в коробке:

  • интеграция изначально настраивается в облаке
  • после того, как данные по интеграции появились в облаке, коробка забирает эти данные по дате изменения
  • поскольку данных у Novostroy.ru немного (обычно меньше 1млн просмотров - 100-200 тысяч), передача данных из облака в коробку происходит достаточно быстро.

Отчеты по данным Novostroy.ru 

По просмотрам не доступна детализация (нельзя просмотреть исходные данные), но каждый просмотр фиксируется в CJM клиента.

Для каждого просмотра отображается:

  • дата
  • канал и площадка (всегда канал=Медийная реклама, площадка = novostroy.ru)

Все данные о просмотрах по площадке доступны в показателе "Просмотры медийных размещений". Если у клиента подключено несколько разных сервисов Postview, то выделить просмотры по площадке можно с помощью фильтра "Источники в Smartis: Площадка" равно "novostroy.ru"

Фильтры для postview данных

Доступны все фильтры, которые отображаются в выпадающем списке при выборе показателя "Просмотры медийных размещений". Ниже список особых фильтров для данного показателя:

  • Медийка: Идентификатор пользователя в рамках площадки
  • Медийка: Место показа на площадке
  • Медийка: Наименование типа баннера
  • Медийка: Сервис Postview
  • Медийка: Страница, где произошел показ
  • Медийка: Страница, на которую ссылался баннер/видео показа
  • Медийка: Тип баннера
  • Медийка: Тип события

  • В фильтре "Person: Когорта персон" при выборе показателей по сессиям, обращениям и CRM данным, есть условие фильтрации для данной площадки:
    • Увидели медийную рекламу на novostroy.ru за выбранный период
    • Увидели медийную рекламу на novostroy.ru за выбранный период (Перекрыть площадку на novostroy.ru)
      Этот фильтр позволяет просматривать события, которые произошли у когорты пользователей, которые, например, просматривали рекламу на данной площадке

С помощью этих фильтров можно создавать отчеты со статистикой о просмотрах по данным критериям. Например, по месту показа баннера на площадке.


Группировки для postview данных

Доступны все группировки, которые отображаются в выпадающем списке при выборе показателя "Просмотры медийных размещений". Ниже список особых группировок для данного показателя:

  • Устройство - группирует по типу устройства, с которого совершен просмотр
  • Медийка: Тип баннера 
  • Медийка: Место показа баннера - группирует по месту показа баннера на площадке
  • Объявления - позволяет строить отчеты по объявлениям с отображением картинки. (группировка применится не только к данным просмотров, но и к обращениям и CRM данным)
  • Ключевые слова/форматы медийки - так как это медийная площадка, работает группировка именно по признаку "Формат". Как и отчет по объявлениям, позволяет построить сквозной отчет в такой группировке - применится не только к данным просмотров, но и к обращениям и CRM данным.



Достоверность по novostroy.ru

Для каждого клиента отображется достоверность по отдельной интеграции с данным сервисом (то есть у всех клиентов будут разные данные в достоверности).
Достоверность и данные по сервису нельзя обновить вручную. Также обновление достоверности в облаке возможно только за предыдущие 5 дней (и происходит только автоматически). При подключении интеграции достоверность будет доступна только за последние 5 дней. Сами ретро данные сервис хранит 30 дней.

Отдельного синхрона для данного сервиса нет, так как Postview данные забирает 1 общий синхрон по всем Postview сервисам.


Novostroy.ru и Novostroy.su

Novostroy.ru - недвижимость в Москве и Московской области

Novostroy.su - недвижимость в Санкт-Петербурге и Ленинградской области

Интеграция с Novostroy настраивается одна.
При построении отчета с данными Post-view необходимо учитывать обе площадки (Novostroy.ru и Novostroy.su) независимо от локации недвижимости, если данные присутствуют на обеих площадках.
