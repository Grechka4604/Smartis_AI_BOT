1. Название, по которому будет происходить поиск решения (кратко):
 + теги, по которым можно будет искать куда-то впихнуть  
Пример: Перетекание трафика между ЖК 

2. Задачи — какие задачи клиента решает:
 

  • Что необходимо клиенту (запрос)?  
    Пример: Посмотреть как распределяются лиды с рекламных кампаний по ЖК1, уходят ли они в ЖК2 и ЖК3.  

  • Зачем это нужно клиенту, в каком процессе это участвует?  
    Пример: на основе данных о перетекании принять решение о перераспределении бюджетов между ЖК, медиапланирование 

3. Техническое решение:
Четкий алгоритм построения отчета/сбора показателя/сбора дашборда и тп.  
+ скриншоты 
+ готовые примеры у других клиентов (id клиентов + название дашборда/отчета) 
+ ссылки на доки существующие (при наличии таковых) 

Пример: документация по перетеканию уже есть https://docs.uiscom.ru/pages/viewpage.action?pageId=177997237 
 

4. Описание готового решения 
 
1) Описание решения для PM-а и клиента 

2) Презентация результатов + полезность



Описание готового решения на примере готового решения в формате "дашборда":  
 
1) на какие вопросы необходимо ответить в подпункте 1:  

  1. Из чего состоит дашборд, какие виджеты в него входят (общая картинка)

  2. Какой виджет за что отвечает? Какие данные он содержит? 
    (возможно) какие уникальные показатели используются (если они только в этом дашборде и были созданы под него), как у них логика построения

  3. Как пользоваться дашбордом? (условно: смотрим в 1 виджет, здесь есть данные по этому), далее есть виджет 2, в нем вот эта информация есть 

2) на какие вопросы необходимо ответить в подпункте 2: 

  1. "полезность" — преимущества данного решения, которые можно озвучить клиенту 

  2. короткое заключение по готовому решению, отвечая на вопрос "какую основную задачу оно помогло решить нашему клиенту?"
