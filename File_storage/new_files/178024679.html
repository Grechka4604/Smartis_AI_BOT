Подготовка внутренних документов (приказов/доверенностей/официальных писем и т.п.)


Потребность в подготовке документов (приказов/доверенностей/официальных писем и т. п.) ) может возникнуть в процессе осуществления хозяйственной деятельности.


1. Постановка задачи (зона ответственности инициатора)

  • инициатор формирует задачу в Jira на специалиста по документообороту (Smartis Документооборот (SMA))

Тип задачи: задача

Тема: подготовка документа.

Описание: наименование и дата документа, содержание документа*

* 1) для подготовки доверенности обязательно указываются: организация (в которой доверенное лицо будет представлять интересы), ФИО доверенного лица и перечень полномочий, которыми его необходимо наделить, срок действия доверенности (разовая или на длительный период)

2) для подготовки приказа обязательно указываются: основание издания приказа, содержание приказа, ФИО работника, на которое будет возложено исполнение приказа и иные дополнительные сведения В задачу может быть вложен уже подготовленный инициатором проект приказа

Для подготовки приказа по проведению мероприятия/акции для клиентов указываются: цель проведения мероприятия/акции, дата и время (с ____ час. по ____ час.) проведение мероприятия, место проведения мероприятия (адрес) и лицо ответственное за заключение договоров с подрядчиками/поставщиками, организацию мероприятия и подготовку отчета о проведенном мероприятии

!!!! Учитывая сквозную регистрацию в соответствующих журналах задача на подготовку документов НЕ МОЖЕТ быть поставлена в Jira прошедшими датами

3) для подготовки иных документов: перечень указываемой информации устанавливается индивидуально по согласованию с ведущим юристом

2. Подготовка документа (зона ответственности ведущего юриста и специалиста по документообороту)

  • специалист по документообороту берет задачу в работу (переводит задачу в статус «Подготовка документа»), тэгает ведущего юриста

  • ведущий юрист осуществляет подготовку проекта документа, вкладывает проект документа в задачу и тэгает специалиста по документообороту

  • специалист по документообороту регистрирует документ в соответствующем журнале регистрации, вносит в проект документа данные из соответствующего журнала регистрации (номер приказа/доверенности), вкладывает в задачу документ с внесенными данными

3. Подписание документа (зона ответственности специалиста по документообороту)

  • специалист по документообороту организует подписание документа (аналогично процессу → Подписание (генеральным директором) документов в офисе г. Москва


4. Закрытие задачи (зона ответственности специалиста по документообороту)

  • после получения подписанного документа специалист по документообороту формирует отчет в задаче, тэгает инициатора с комментарием «Задача выполнена», размещает скан-образ документа в электронном архиве, закрывает задачу
