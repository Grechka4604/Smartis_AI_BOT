Обновление про коробки:

С июня 2024 Антифрод работает и в коробках. Подключить на данный момент можно через разработку (тикет). ВАЖНО - предупредить клиента, что коробка будет запрашивать облачное API скоринг по номеру телефона (без каких либо дополнительных деталей), чтобы учесть при скоринге как знания о поведении клиента внутри коробки, так и знания о клиенте в облачной базе Smartis.


  1. Заполните Ключевые показатели (важно заполнить следующие показатели):

    1. Все обращения

    2. Целевые обращения

    3. Состоявшиеся первичные встречи

    4. Бронирования

    5. Успешные сделки

    6. Сумма по успешным сделкам

    7. Маркетинговый бюджет

    8. Обращения для скоринга Антифрод - в этом поле указать показатель целевых обращений, по которому нужно считать и отображать % фрода. Этот показатель будет добавлен в автоматический дашборд и % фрода рассчитан будет на его данных. При этом, автоматически все равно будет выполняться скоринг по всем обращениям клиента, а не только по целевым.

      Если у вашего клиента какой-то из этих показателей не используется (не существует), то создайте пустой показатель с типом Импорт и добавьте его в ключевые показатели. Это ни на что не повлияет, зато позволит создать дашборд. Далее можно будет удалить из виджетов дашборда ненужные показатели, и из ключевых показателей убрать "пустышку".Обратите внимание, что если в строке Обращения для скоринга Антифрод будет указан показатель на CRM данных, то автоматически скоринг антифродом не будет выполняться. Чтобы он выполнялся, нужно обратиться в консультацию по шаблону и создать регулярную задачу на скоринг данных клиента. Периодичность скоринга желательно установить не чаще 2 раз в неделю.
      Ссылка на задачу: https://app.asana.com/0/1206876306252490/1206877365352404/f
  2. Только после заполнения ключевых показателей в разделе CDP "Готовые решения" нажимаем "подключить" на плашке Антифрода
  3. После этого решение запускается, скоринг всех обращений выполняем автоматически с текущего дня + 90 дней в прошлое (в случае если вам нужно скорить еще и срм данные - см пункт 1 примечание)
  4. Создается автоматический дэшборд. Если вы забыли настроить ключевые показатели (или настроили не все указанные в пункте 1) до создания дашборда, то у вас дашборд не создастся. При этом интерфейсно запустить повторно его создание нельзя. Для этого используйте шаблонную задачу в консультации - https://app.asana.com/0/1206876306252490/1206877364947878/f


Скоринг  (вместе с периодом в прошлом) после подключения занимает 1-2 дня

Анализ данных выполняется 426 синхроном. Он запускается в час ночи и работает около 1 часа, обновляя данные за последние 30 дней.

Анализуем не только АДЕКВАТНЫЕ персоны, но и СЕРЫЕ (иными словами все персоны кроме подозрительных и неадекватных). Анализируем историю обращений исключая клиентов-лидгенов. Для клиентов-лидгенов - в API выдаем детализацию скоринга.

Поддержка и приостановка решения

  1. После завершения скоринга можно проводить оценку площадок руководствуясь картой: https://www.mindmeister.com/app/map/2685158583

  2. Запуск Антифрода - вместе с клиентом разбираем отчет после анализа данных за ретро-период, анализируем текущий сплит:

    1. есть ли сейчас площадки, которые ранее были с фродом?

    2. если да, было ли продвижение по воронке продаж с этих площадок?

    3. наличие сделок и ДРР по площадкам, которые ранее замечены во фроде.

    4. если сейчас нет площадок, которые были ранее замечены во фроде - смотрим данные за ретро период (чтобы клиент больше не подключал фродовые площадки)

      3. В конце 1-го месяца:

    1. анализируем долю фрода по площадкам за месяц, если есть площадки с высокой долей фрода - рекомендуем клиенту их оптимизировать или отключить

    2. если площадок с высокой долей фрода нет по итогам первого месяца, то можно уточнить, планирует ли клиент подключать новые СРА площадки в следующем месяце ИЛИ значительно увеличивать KPI площадкам, которые активные сейчас. Если да, то использование антифрода для аналитики площадок целесообразно.

    3. Если нет, то можем заморозить антифрод до момента, когда клиент в следующий раз будет расширять медиасплит по кол-ву площадок или по бюджету.

    4. В конце каждого последующего месяца использования необходимо подводить итог с клиентом:

    1. есть ли фродовые площадки

    2. какие решения принимали на отчете Антифрод по площадкам

    3. какой эффект дали эти решения


    Основания для заморозки Антифрода:

    1. сейчас фродовых площадок нет

    2. увеличение KPI для площадок СРА не планируется

    3. подключение новых площадок СРА не планируется

клиент НЕ использует данные детализации для проверки на фрод отдельных обращений


Чтобы перевести решение в режим "заморозки", нужно:

  1. В разделе готовых решений на плашке Антифрода кликнуть "Поставить на паузу"


  2. Статус решения изменится на "на паузе" и данные скоринг антифродом больше не будут доступны в кабинете клиента



    3. Что делаем в фоновом режиме, если решение у клиента заморожено:

    смотрим на общую долю фрода по всем обращениям (при отключении делаем замер доли обращений, у которых % фрода 75 и более и наблюдаем как меняется эта доля далее). Это можно настроить графиком в графане. Если общая доля фрода будет расти, обязательно говорим клиенту об этом. В таком случае мы не скажем, по какой площадке произошли изменения, в сможем дать сигнал о том, что антифрод снова нужен.
     Посмотреть % фрода в фоновом режиме можно в графане в отчете: http://grafana.smartis.bi/d/PX2p8pc4z/antifraud?orgId=1&from=1690837200000&to=1693429199000

Как использовать отчет по Антифроду

  1. Отчет по окупаемости площадок
    - фильтруем по убыванию показатель "Доля фрода 75-100%" - этот показатель указывает, какой % целевых обращений с высокой вероятностью фрода (более 75%). Получаем топ площадок по фродовости.
    - обращаем внимание на площадки, по которым доля фрода более 50-60 - это значит, что по площадке более половины целевых обращений - подозрительные
    - чтобы принять решение об отключении площадки, недостаточно посмотреть только на долю фрода - надо посмотреть, есть ли по площадке визиты, брони, сделки
    - если по площадке много фрода, но при этом есть сделки - надо оценить ДРР! Если ДРР меньше 100, это значит, что площадка окупается - и ее не надо отключать. Если ДРР больше 100 - площадка не окупается, расход на нее больше чем доход, площадка сливает деньги. Если ДРР = 0, значит сделок нет, площадка не окупается и ее можно отключать.
    - можно оценивать также наличие визитов и броней по площадке, но на визиты опасно смотреть - фрод может совершать визиты! брони тоже может совершать фрод, в случае если у клиента брони не платные. Если брони платные - то на этот показатель тоже можно ориентироваться при принятии решений (есть платные брони - скорее всего не надо отключать площадку).

2. Градация фрода:

- отчет показывает, сколько обращений с разным % фрода есть по площадке, с помощью отчета можно увидеть площадки с лучшим кач-вом траффика (показатель фрод 0-5%, 5-50%, 51-74%)

3. Динамика всплесков фрода по дням:

- этот отчет лучше использовать с фильтром по конкретной площадке

- с помощью отчета можно посмотреть, есть ли всплески фрода по площадке. Например, есть площадка с высоким % фрода - вы можете отфильтровать отчет по ней и посмтреть, как фрод распределялся в течение месяца. Часто бывает так, что фродовые площадки приводят много фрода в определенный день (обычно в конце месяца, но может быть и в любой день). Главный смысл: если по площадке обычно 1-2 обращения в день, а в какой то из дней приходит значительно больше целевых звонков, и при этом они с высоким % фрода - это очень подозрительно

Пример отчета по фродовой площадке: в 1 день пришло целых 10 обращений, целевых, и с высоким % фрода - это крайне подозрительно

4. Фрод по каналам: отчет показывает, сколько обращений с высоким % фрода приходит с каждого канала, в сравнении с общим объемом целевых обращений

5. Фрод по базам недвижимости, лидгенам и программатику

в этом отчете можно посмотреть, с каких триггерных площадок (триггерные - которые чаще всего фродядт) приходит больше всего подозрительных обращений
