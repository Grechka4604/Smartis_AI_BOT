Smartis.CDP - стек продуктов, которые используются для активации данных клиента (активация = использование знаний о клиентской базе для оптимизации расходов на рекламу и для улучшения качества обслуживания клиентов).


Что такое CDP-платформа

CDP (Customer Data Platform) - платформа по управлению данными клиента. В рамках кабинета Smartis у наших клиентов хранятся большие объемы данных, например, информация по сделкам, звонкам, визитам на сайт и тд. Клиент может эти данные активировать (то есть использовать в своих рекламных активностях) с помощью нашей платформы. Например, собрать данные о клиентах со сделками и создать аудиторию похожих на них - чтобы найти более целевую аудиторию в рекламных источниках. CDP не связана только с сегментацией пользователей, по сути это инструмент, который помогает также получить информацию о своих клиентах. Эту информацию можно использовать для улучшения знания об аудитории, повышению качества трафика подрядчиков, улучшению клиентского опыта на всех этапах воронки.

Какие варианты использования платформы доступны сейчас

1.Smartis.CDP Конструктор сегментов

в рамках этого решения клиент получает доступ к конструктору сегментов и может сам создавать любое количество сегментов по разным признакам персон, и далее использовать полученные сегменты в произвольных механиках в рекламных кабинетах (на данный момент это Яндекс Директ и MyTarget).

Функционал решения:

- создание сегментов

- отправка сегментов в Директ, Майтаргет и ВК
- обновление сегментов ежедневно с учетом выбранного периода

- микроконверсии (для создания сегментов мы можем разметить до 10 микроконверсий на сайте)

На каких данных можно собирать сегменты




Зачем это решение и кому мы его предлагаем:

  1. Это решение не для активных продаж - оно нужно, чтобы закрыть конкретные задачи клиента, связанные с созданием сегментов. То есть, мы предлагаем этот решение, когда клиенты хотят выполнить нечто подобное: 
    - собрать аудитории своих пользователей, отправить их в рекламные кабинеты с целью оптимизации рекламы/таргетинга    - сегментировать свою базу клиентов, смотреть отчет в разрезе сегментов
  2. Для этого решения мы не создаем отчетность автоматически, если клиенту необходимо в смартис отслеживать результат работы сегментов в отчете - просим соблюдать регламент разметки (добавлять в utm_campaign ID сегментов, в названии РК добавлять "cdp")
  3. В рамках этого решения мы не гарантируем результат связанный с изменением каких-либо метрик, тк сегменты создаются такие, какие хочет клиент. А значит, отвечать за результат мы не можем (мы просто гарантируем что сегменты соберутся и отправятся в кабинеты, в рамках наших технических возможностей)
  4. С созданием и отправкой сегментов клиенты должны работать самостоятельно. Но подключение интеграций и самого CDP - задача менеджера.


В рамках этого сценария необходимо провести обучение для клиента, чтобы показать как собирать сегменты и отправлять их во внешние сервисы. Работа РМ заключается в проведении обучения и консультации клиента при необходимости, а также в подключении всех необходимых интеграций для работы решения (консультация не равно собирать и отправлять сегменты за клиента!)

Пример: 

клиент может создать узконаправленный сегмент "персоны, которые провели на сайте больше 3 минут, но не оставили обращение" и использовать его для ретаргетинга. То есть, при использовании конструктора сегменты могут быть любыми в рамках наших возможностей и использоваться на усмотрение клиента в любых доступных механиках. 

Важно, что конструктор сегментов с использованием CRM данных подходит в основном для клиентов, которые используют/планируют использовать сквозную аналитику. Это связано с тем, что без наличия сквозной аналитики интеграция с CRM усложнит и сделает более дорогим процесс внедрения решения.
Однако, если клиенту не нужны данные CRM для сборки сегментов, то мы можем ему подключить конструктор для сборки сегментов на других типах данных (см. таблицу ниже)

2. Антифрод  

Это решение предоставляет функционал скоринга обращений на процент вероятности фрода, то есть помогает определить, с какой вероятностью каждое обращение может быть фродовым. Также, имея данные по каждому обращению, мы выстраиваем аналитику по площадкам для определения качества трафика по каждой рекламной площадке.

  1. В рамках этого решения мы НЕ предоставляем сегменты, решение используется для всех обращений в кабинете клиента - то есть, по всем площадкам клиента.

Минусация в performance каналах - это решение позволяет исключать из показа в рамках рекламных кампаний нецелевые сегменты аудиторий (минусовать). 


Метрики, которые этот сценарий помогает улучшить: снижаем ДРР, повышаем ROMI, снижаем стоимость закупки трафика (СРС, стоимость обращения). Косвенно - улучшаем конверсию по воронке продаж.

Какие сегменты доступны в рамках решения: 

- спам

- фрод

- купившие (за последние 365 дней)

- забронировавшие (за последние 365 дней) 

- отказники (за последние 90 дней). 


В каких каналах может использоваться: пока только в Я.Директ (скоро будем дорабатывать для подключения и в Mytarget). 

Q&A
- Можно ли использовать другие сегменты для минусации? - нет, в рамках готового решения такой возможности нет
- Можно ли использовать не все сегменты из списка в рамках готового решения? - да, можно
- Можно ли использовать эти сегменты не для исключения (не для минусации) - нет, в рамках готового решения мы автоматически добавляем выбранные сегменты в выбранные рекламные кампании с корректировкой "-100%" (то есть исключаем их)
- Можем ли создать сегменты за другой период (не за 365/90 дней) - в рамках готового решения не создаем
- Можно сделать сегменты с разбивкой по ЖК или по классам недвижимости? - нет, эти сегменты собираются по всему ЛК Smartis.
- Как вы собираете сегменты Спам и Фрод? Что это за сегменты (подробнее по ссылке: )
- А это работает? У вас есть доказательства? - да, вот ссылка на кейс с ГК Эталон https://adindex.ru/publication/283620/2022/08/24/306239.phtml

- Цена решения зависит от кол-ва рекламных кабинетов/аг-в, которые это используют/ количества ЖК? - нет, если это ОЧЕНЬ крупный клиент, мы уточним цену на этапе формирования КП в индивидуальном порядке

- Нужно ли аг-вам что-то менять в разметке, ставить метки CDP? - нет, мы добавим пометку о том, что применена минусация, к названию рекламной кампании в Smartis (в рекламном кабинете этого не будет)

- Сколько дней занимает внедрение? - 1-2 дня, с момента предоставления нужных доступов к рекламным кабинетам

- К каким рекламным кампаниям лучше применять минусацию? - мы рекомендуем ко всем, если в каких-то кампаниях решение не будет давать ожидаемый результат, то можно скорректировать список кампаний, к которым мы подключили минусацию.

Подробная инструкция для подключения минусации: https://docs.uiscom.ru/pages/viewpage.action?pageId=161846522




LAL - поиск похожих аудиторий в performance каналах - это решение позволяет создавать Look-alike аудитории в интерфейсе Smartis на базе выбранных сегментов, передавать эти аудитории в Яндекс Директ и добавлять в рекламные кампании.


Метрики, которые этот сценарий помогает улучшить: увеличение количества целевых обращений, увеличение количества встреч/бронирований/сделок, снижение ДРР, увеличение ROMI

Какие сегменты доступны в рамках решения: 
- Совершили целевые обращения
- Совершили состоявшиеся первичные встречи
- Совершили бронирования
- Совершили успешные сделки

В каких каналах может использоваться:  в Я.Директ и MyTarget.
