Цель-стандартизировать заполнение статусов по проектам внедрения для более точного понимая:

  • укладываемся ли мы в сроки внедрения;
  • есть ли блокеры на этапах с нашей стороны;
  • есть ли запросы к руководителям и разработке;
  • есть ли блокеры на стороне клиента. 

Участники процесса: Project-менеджеры внедрения, Руководитель отдела внедрения, Игорь, Миша.

Инициатор: Руководитель отдела внедрения.

Инфо поле: Asana, Teams.


Процесс заполнения статуса.

У каждого проекта в asana должны стоять сроки старта и прогнозируемого окончания внедрения. 

Задачи в asana должны актуализироваться, чтобы понять ход проекта.

Есть 3 статуса в Asana:

По плану- заполняется, когда у проекта нет никаких блокеров, успеваем внедрить проект до указанного срока;

Отстает - есть блокеры, обозначить на чьей они стороне, не успеваем внедрить до указанного срока;

Под угрозой - есть риск отказала от сервиса, клиент не понимает ценности/сменилась команда.
