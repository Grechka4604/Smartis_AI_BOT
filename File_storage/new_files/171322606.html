Что такое группировка.

Группировка - признак, по которому будут выделены уникальные строки в отчете. Чтобы выбрать группировку, нужно либо выбрать ее из выпадающего списка, либо ввести ее название в поле "Группировка".

Группировки позволяют делать отчеты в срезе по какому-либо свойству данных.

Основные виды группировок:

- по времени (день, неделя, месяц, год и тд)
- по источнику трафика (канал, площадка, кампания, utm метки, объявления и прочие)
- по ГЕО (Регион, город)
- по объектам и их признакам (Объект, Класс недвижимости)
- по соц.дем призакам (Пол клиента, возраст клиента)
- по признакам Персоны (Person: Person ID, Person: Хронотип и тд)
- по признакам помещения (Комнатность, Этаж)

Также можно создавать пользовательские группировки (они появятся только в личном кабинете клиента, для которого были настроены) по полям CRM и по группам и категориям рекламных кампаний . Подробнее про группировки тут: (добавить ссылку на конфигуратор)
В списке группировок напротив каждой группировки указано, какая она - системная или пользовательская.


Не все группировки доступны для любого показателя: например, показатель "Звонки" или "Посетители Smartis" нельзя построить в группировке "Ответственный менеджер", тк у этих данных нет признака "Ответственный менеджер".

Также и с пользовательскими группировками, которые созданы для группировки только определенных данных (CRM или данных из Яндекс Директ). 

Также, группировка "Объекты" доступна только для проектов, которые состоят из нескольких объектов: нельзя построить в группировке "Объекты" данные по ЖК Ромашка, но можно - по проекту "Все ЖК" или "ЖК Москва" и прочим проектам, которые включает несколько объектов.

Если группировка не отображается в списке для выбранного показателя в отчете, то либо она недоступна для этого показателя, либо она не связана с выбранной метрикой (если такая связь нужна или должна появиться, необходимо обратиться в тех поддержку Smartis). Аналогично работает в случае, когда выбрано несколько показателей, но для какого-то из них группировка недоступна - она не будет вообще отображаться в списке группировок.


Для того, чтобы построить отчет, нужно обязательно выбрать группировку.

Например, у обращений есть свойство "utm_source" - выбрав группировку по utm_source и показатель обращений - вы получите отчет в котором каждая строка = уникальное значение utm_source в данных соответствующих фильтрам.

Общие нюансы:

  1. Системные базовые группировки (groupings.is_system = 1 AND groupings.client_id = 0) - они показываются для любого показателя и для любого клиента, мы закладываем их логику в код по умолчанию для любого нового показателя.
    Примеры: День, Месяц, Канал, Площадка, Кампания, Объявление, Объекты, utm_source
  2. Системные группировки (groupings.client_id = 0) - они показываются для любого клиента, но только если есть связь со всеми выбранными показателями. 
    Примеры: Статус обращения, Ответственный менеджер, Страница входа (посадочная страница)
  3. Пользовательские группировки (groupings.client_id > 0) - показываются только для конкретного клиента, сделаны вручную пользователем В конфигураторе или кастомной разработкой под клиента
    Примеры: Дилерский центр_Марка, Источник лида в CRM, Брокер, Комнат, Банк ипотеки

Аккаунт рекламной системы - Позволяет сгруппировать данные по наименованию рекламного кабинета, из которого приходят данные (это название аккаунта, отображаем именно названия кабинетов, а не почту через которую подключена интеграция). 

Объединяет в себе все кампании относящиеся к аккаунту Яндекса/ВК/Майтаргет или Facebook. Если вы попробуете построить отчет в этой группировке по площадкам, название логинов которых нам не передается (все площадки кроме Яндекса/ВК/Майтаргет или Facebook), то в отчете ничего не отобразится.
Если у вас подключена интеграция в смартис через "общий" аккаунт, у которого есть доступы к нескольким аккаунтам внутри рекламного кабинета, то в отчете будут названия из рекламного кабинета по всем логинам. 

Группировка применима к данным таких типов:

  • данные из рекламных кабинетов (клики, показы, расход, лиды)
  • посетители Смартис (сессии)
  • все обращения (звонки, заявки, чаты, анкеты, квизы и прочие)
  • CRM данные

Под капотом собираем именно данные по кампаниям, и уже от них ищем связку с логином рекламного кабинета. Цифры которые вы видите в итоге в отчете по рк, должны быть и в итоге данных этой группировки. 

Если к аккаунту нет доступа - в отчете отобразятся архивные данные по нему, которые мы получили ДО момента потери доступа.

Если аккаунт отключен - в отчете отобразятся архивные данные по нему, которые мы получили ДО момента отключения.

Игнорируем только УДАЛЕННЫЕ аккаунты - если аккаунт удален, то данные по нему в отчете не отобразятся.

Группа объявлений - Позволяет сгруппировать данные по наименованию группы объявлений. Объединяет в себе все объявления, относящиеся к определенным группам в рамках кабинета Яндекса/ВК/Майтаргет или Facebook. 
Применима только для площадок Яндекс/ВК/Майтаргет или Facebook.

Группировка применима к данным таких типов:

  • данные из рекламных кабинетов (клики, показы, расход, лиды)
  • посетители Смартис (сессии)
  • все обращения (звонки, заявки, чаты, анкеты, квизы и прочие)
  • CRM данные

Цифры итога, которые вы видите в итоге в отчете по объявлениям, должны быть и в итоге данных этой группировки. Кроме случаев, когда объявление не входит ни в какую группу (если такое возможно для конкретного сервиса).
