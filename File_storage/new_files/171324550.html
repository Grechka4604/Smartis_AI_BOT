Основная информация

Это медийный портал: агрегирует в себе католог застройщиков, каталог квартир, отзывы и сервис консультирования. (топ-4 медийная площадка на рынке недвижимости по медийным бюджетам и охватам). На этой площадке есть возможность размещать медийную/охватную рекламу. У нас нативная интеграция с сервисом.

Интеграция Novostroy-m и Smartis позволяет:

  • получать в Smartis данные о просмотрах медийных размещений на данной площадке

  • матчить их с CJM клиента.

id сервиса в Smartis - 88.

По всем сервисам Postview работает 1 синхрон: описан тут 2. Post-view

Как работает интеграция:

Те медийные креативы (баннеры), который застройщик хочет разместить на площадке, Novostroy-m  размечает своим пикселем и собирает данные о просмотрах у себя. На нашей стороне:

  • Получаем данные: ежедневно с 3:00 ночи до 9:00 утра синхронизируем данные за предыдущий день (по сути, получаем просто сырые  данные по показам медийки)
  • Делаем cookie-матчинг: с момента включения интеграции в конкретном клиенте происходит cookie-матчинг данных просмотров с площадки и данных в Smartis. Это происходит за счет того что после включения интеграции у клиента на сайте размещается пиксель cookie-матчинга с Novostroy-m (невидимая картинка 1х1 пиксель, загружается в защищенном слое нашего кода, сопоставляет id из площадки и наш pid). Это позволяет определить, к какой персоне в Smartis относится показ рекламы и встроить его в CJM клиента.
  • Определяем, какому клиенту принадлежат данные о показах: когда мы скачиваем данные о показах на площадке, мы получаем их не в рамках 1 клиента, а вообще по всем размеченным данным, которые передает Novostroy-m (как и ЦИАН). Поэтому при получении данных мы на своей стороне определяем, какому клиенту эти данные принадлежат. Мы знаем из данных площадке только на какую страницу вел баннер, и по этой ссылке мы ищем среди всех ЛК клиентов Smartis сайты и определяем, к какому сайту эта ссылка относится. Показ попадет к клиенту, у которого в ЛК зарегистрирован этот сайт.

    Важно:     всегда регистрировать в Smartis все новые сайты клиента, так как если баннеры будут вести на эти сайты, данные потеряются (если сайт у нас не будет зарегистрирован)

  • полноты cookie-матчинга можно ожидать только с момента включения интеграции у клиента. До этого момента могут быть пересечения, если мы знали куку, но это скорее исключение.
  • для матчинга важна динамика колл-трекинга (то есть чтобы у нас была связка cookie + телефон клиента). Важно чтобы максимально онлайн трафик клиента был покрыт динамическими номерами колл-трекинга (и конечно сама площадка Novostroy-m)


Post-view данные могут быть даже у тех клиентов, интеграция у которых не подключена. Для тех клиентов у которых в личных кабинетах Smartis уже заведены сайты - мы даже без включенной интеграции сохраняем события post-view. И только с момента включения интеграций начинаем их применять при матчинге конверсий


Как подключить

Перед подключением клиента нам напишет либо площадка в чате в телеграм с просьбой согласовать подключение интеграции, либо нам самим нужно написать площадке в этом чате. Если нам написала площадка - нужно связаться с клиентом и подтвердить подключение письменно (так как устанавливается пиксель на их сайт).

  1. В разделе Администрирование → Интеграции → Доступные для подключения и выбираем Novostroy-m. По клику на кнопку "Подключить" откроется меню для ввода названия интеграции. Больше ничего вводить не нужно.



  2. Нажать "Подключить" - после чего интеграция переместится в раздел "Подключенные интеграции"


После того, как интеграция подключена, ее не нужно привязывать к сайтам.


Взаимодействие с площадкой:

Площадка заинтересована видеть эффект от нашего партнерства и оценивать работу своего портала, поэтому может обращаться к нам с просьбой подготовить отчеты.

  • по запросу от площадки мы подводим итог по клиенту с и без поствью 
  • согласовывает его с клиентом (письменный), фиксируем согласование скриншотом (чтоб было видно кто и когда согласовал)
  • передаем отчет площадке для ознакомления (согласованную с клиентом)

Особенности работы в коробке:

  • интеграция изначально настраивается в облаке
  • после того, как данные по интеграции появились в облаке, коробка забирает эти данные по дате изменения/обновления


Отчеты по данным Novostroy-m

По просмотрам не доступна детализация (нельзя просмотреть исходные данные), но каждый просмотр фиксируется в CJM клиента.

Для каждого просмотра отображается:

  • дата
  • канал и площадка (всегда канал=Медийная реклама, площадка = novostroy-m)

Все данные о просмотрах по площадке доступны в показателе "Просмотры медийных размещений". Если у клиента подключено несколько разных сервисов Postview, то выделить просмотры по площадке можно с помощью фильтра "Источники в Smartis: Площадка" равно "novostroy-m"

Фильтры для postview данных

Доступны все фильтры, которые отображаются в выпадающем списке при выборе показателя "Просмотры медийных размещений". Ниже список особых фильтров для данного показателя:

  • Медийка: Идентификатор пользователя в рамках площадки
  • Медийка: Место показа на площадке
  • Медийка: Наименование типа баннера
  • Медийка: Сервис Postview
  • Медийка: Страница, где произошел показ
  • Медийка: Страница, на которую ссылался баннер/видео показа
  • Медийка: Тип баннера
  • Медийка: Тип события

  • В фильтре "Person: Когорта персон" при выборе показателей по сессиям, обращениям и CRM данным, есть условие фильтрации для данной площадки:
    • Увидели медийную рекламу на novostroy-m за выбранный период
    • Увидели медийную рекламу на novostroy-m за выбранный период (Перекрыть площадку на novostroy-m)
      Этот фильтр позволяет просматривать события, которые произошли у когорты пользователей, которые, например, просматривали рекламу на данной площадке

С помощью этих фильтров можно создавать отчеты со статистикой о просмотрах по данным критериям. Например, по месту показа баннера на площадке.

Группировки для postview данных

Доступны все группировки, которые отображаются в выпадающем списке при выборе показателя "Просмотры медийных размещений". Ниже список особых группировок для данного показателя:

  • Устройство - группирует по типу устройства, с которого совершен просмотр
  • Медийка: Тип баннера 
  • Медийка: Место показа баннера - группирует по месту показа баннера на площадке
  • Объявления - позволяет строить отчеты по объявлениям с отображением картинки. (группировка применится не только к данным просмотров, но и к обращениям и CRM данным)
  • Ключевые слова/форматы медийки - так как это медийная площадка, работает группировка именно по признаку "Формат". Как и отчет по объявлениям, позволяет построить сквозной отчет в такой группировке - применится не только к данным просмотров, но и к обращениям и CRM данным.



Достоверность по novostroy-m

На данный момент достоверность по сервису не отображается.
