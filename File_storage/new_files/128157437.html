Как пользоваться этим шаблоном.

Данный шаблон используется как часть процесса развертывания важного продуктового релиза.

Цель процесса: быстрое и полное распространение информации о новом продуктовом релизе.

Результат выполнения процесса:
I. Создан набор материалов для поддержки релиза (УТП, описание, презентация и т.п.)

II. Материалы доведены до сотрудников, проведено обучение по релизу.

III. Разработан и запущен маркетинговый план продвижения релиза.

Описание процесса:

  1. PM (Product Owner) уведомляет PMM (Product Marketing Manager) о предстоящем выпуске важного продуктового релиза.
  2. PMM проводит интервью PM и заполняет документ с описанием продуктового релиза (по этому шаблону).
  3. PMM организует производство маркетинговых материалов.
  4. PMM организует обучение сотрудников.
  5. PMM запускает реализацию маркетингового плана продвижения релиза.
  6. PMM проводит сбор информации о результатах продвижения релиза.
  7. PMM проводит встречу с PM для обсуждения результатов продвижения релиза.

ДЛЯ 
(целевой сегмент, полное описание)

Гипотеза - для всех компаний, у которых более 5% пропущенных звонков за неделю.

  1. Для руководителей отделов продаж и маркетологов, у которых стоит задача привлекать больше лидов (сегменты - недвижимость B, авто B, медицина B, интернет-магазины B).

  2. Для маркетологов, которые следят за качеством обработки лидов и заботятся, чтобы не пропустить ни одного (сегмент медицина B1).

  3. Для автодилеров, у которых есть штрафы от автопредставительства за пропущенные звонки (сегмент авто А1)



КОТОРАЯ 
(Wants, Needs, Pains, Fears)

Pains

  • Менеджеры (операторы) пропускают звонки, из-за этого теряются лиды и я не получаю свою премию.
  • Меня, как дилера, штрафует представительство автобренда, если теряются звонки (сегмент авто А1).

Business tasks

  • Улучшить качество обслуживания клиентов в кол-центре - не терять звонки;
  • Контроль работы менеджеров по продажам с лидами.

РЕШЕНИЕ

Сервис автоматического перезвона по пропущенным звонкам, который :

  • перезванивает несколько раз (если с первого раза не получается),
  • учитывает рабочее время,
  • планирует дозвоны (по настроенным интервалам),
  • ограничивает нагрузку на колл-центр,
  • фильтрует нецелевые звонки (по времени ожидания).

ПОСКОЛЬКУ

(в отличие от... [конкурентов])

В отличие от того, когда обрабатывают пропущенные вручную:

  • увеличивается скорость их обработки за счет автоматизации рутинных операций по поиску пропущенных и неудачным попыткам набора;
  • видно общую картину обработки пропущенных звонков.

В отличие от других аналогичных сервисов:

  • Mango Office автоперезвон не имеет настройки по времени и логике перезвона, и сделан на уровне Переадресации, поэтому, если звонок не дошел до переадресации, то перезвона не будет в принципе.
  • Наш автоперезвон собирает все звонки, настройки позволяют указать, как именно нужно перезвонить (?) когда в компании имеется несколько групп по-разному обрабатывающих звонки.
