Один из базовых и основных разделов Smartis. Позволяет создавать отчетность, которую далее можно сохранить в качестве виджета в дашборд или просматривать в самом конструкторе.

Перейти в раздел можно из бокового меню или по ссылке https://my.smartis.bi/reports/designer


Интерфейс Конструктора отчетов

  1. Проект - поле для выбора сайта, объекта или проекта, по которому нужно отобразить данные. Объект - чаще всего ЖК Дилерский центр, Проект - группа ЖК или Дилерских центров (например, сгруппированная по региону или по классу недвижимости или по рекламному агентству)
    Для того, чтобы построить отчет, поле "Проект" должно быть обязательно заполнено
    В данном поле можно выбирать как 1 проект, так и несколько значений в разных комбинациях: например, выбрать несколько объектов, несколько проектов или несколько сайтов (также можно выбирать сайт + объект, объект + проект и тд.) Данная логика работает как в конструкторе отчетов, так и в дашбордах и базе обращений. Выбранные проекты отмечаются галочкой. 

    Если у пользователя нет прав доступа к какому-то из выбранных объектов/проектов/сайтов, то отчет по ссылке у него не построится.


    Если никакой проект не выбран, то всегда указывается проект по умолчанию (поле "проект" не бывает пустым по умолчанию). 

    Если клиенту нужно выбирать только 1 проект, то возле окна выбора проекта укажите "равно", а если несколько то укажите "включает".




  2. Показатели - поле для выбора показателей, которые нужно отобразить в отчете. 
    Для того, чтобы построить отчет, поле "Показатели" должно быть обязательно заполнено

  3. Канал - поле для выбора канала, по которому нужна отчетность. Это выпадающий список, в котором можно выбрать 1 или несколько значений. Если нужно посмотреть отчет по всему трафику, выбираем "Весь трафик". В выпадающем списке каналы разделены на Онлайн и Офлайн, также есть особая группа: "Весь трафик" и "Другое"
    - "Весь трафик" - совокупность всех каналов
    - Другое - трафик, который не прикреплен ни к одному из существующих каналов
    Канал - это ресурс, из которого приходит трафик. Могут быть платные (рекламные каналы: Контекст, Медийная реклама, Соц.Сети, Базы недвижимости и Классифайды и тд.) и бесплатные (органика, самоходы и прочие). Канал обычно состоит из площадок или других более мелких единиц (источников). Кроме деления на платные/бесплатные, часто каналы разделяют на онлайн и оффлайн. Онлайн - вся реклама в интернете, Офлайн - реклама не в интернете (ТВ, радио, наружная реклама и тд).Для того, чтобы построить отчет, поле "Канал" должно быть обязательно заполнено

  4. Площадка - поле для выбора площадки (источника) трафика, по которому нужно посмотреть отчетность. Площадка это составной элемент канала (канал состоит из площадок).
    Поле выбора площадки не работает, если поле Канал не заполнено.  Также выбор площадки не доступен, если в поле "Канал" указано "Весь трафик".

  5. Группировка - признак, по которому будут выделены уникальные строки в отчете


    Основные виды группировок:
    - по времени (день, неделя, месяц, год и тд)
    - по источнику трафика (канал, площадка, кампания, utm метки, объявления и прочие)
    - по ГЕО (Регион, город)
    - по объектам и их признакам (Объект, Класс недвижимости)
    - по соц.дем призакам (Пол клиента, возраст клиента)
    - по признакам Персоны (Person: Person ID, Person: Хронотип и тд)
    - по признакам помещения (Комнатность, Этаж)
    Также можно создавать пользовательские группировки. Подробнее про группировки тут: 

    Не все группировки доступны для любого показателя:     например, показатель "Звонки" или "Посетители Smartis" нельзя построить в группировке "Ответственный менеджер", тк у этих данных нет признака "Ответственный менеджер". Также и с пользовательскими группировками, которые созданы для группировки только определенных данных (CRM или данных из Яндекс Директ).
    Также, группировка "Объекты" доступна только для проектов, которые состоят из нескольких объектов: нельзя построить в группировке "Объекты" данные по ЖК Ромашка, но можно - по проекту "Все ЖК" или "ЖК Москва" и прочим проектам, которые включает несколько объектов.

    Для того, чтобы построить отчет, нужно обязательно выбрать группировку.

  6. Модель атрибуции - позволяет указать модель атрибуции, в которой необходимо представить данные в отчете.


    В окне настроек модели атрибуции необходимо выбрать 3 параметра:
    - модель атрибуции (последнее касание, первое касание, линейное распределение и тд) - это способ, как именно будем определять источник 
    - период атрибуции (30,90,180,365 дней или бесконечность) - время, за которое учитываем касания при определении источника
    - чекбокс "Включая прямые касания" - если поставить галочку, то при работе с атрибуцией будут учитываться прямые переходы клиента (и канал "Прямой трафик"). Если галочку не ставить - не будут.
    Чаще всего модель атрибуции определяет клиент в зависимости от своей логики учета, принятой в бизнесе.

    Модель атрибуции не имеет значения для следующих данных:
    - посещения сайта (так сессия всегда содержит информацию о своем источнике)
    - данные из рекламных кабинетов (Яндекс Директ, ВК, MyTarget)
    - Если выбрана группировка по времени (день, месяц, год, неделя и другие) и канал "Весь трафик" (тк в этом случае нам не нужно определять вес источников трафика и мы выбрали "Весь трафик")
    - данные из сервисов обращений, кроме показателей "по моделям атрибуции"*

    *Показатели "по моделям атрибуции" - это системные показатели, которые создаются автоматически для каждого сервиса обращений, с которым есть интеграция у Smartis. При выборе таких показателей, данные из сервиса обращений будут подчиняться логике выбранной модели атрибуции, несомтря на то, что они сами содержат информацию об источнике.
    Такая возможность предусмотрена для того, чтобы учитывать касания клиента до совершения им обращения (просмотр медийных размещений, посещение сайта).


    Для того, чтобы построить отчет, нужно обязательно выбрать модель атрибуции.


  7. Период - выбор периода, за который нужно отобразить данные в отчете


    Как выбирать период:
    - на календаре (можно выбрать первый и последний день периода)
    - кликом на название месяца (чтобы выбрать месяц целиком)
    - ввести дату начала и дату окончания периода в окне под календарем (год, месяц, день)

    Чтобы переключить календарь по месяцам, нужно воспользоваться стрелками (справа и слева от календаря)
    Чтобы выбрать другой год, можно также воспользоваться стрелками (справа и слева от календаря) или кликнуть на номер года в верхней строке

    Особенности работы периода для группировок: неделя, месяц и год
    Неделя:     если на календаре вы выбрали любой период (даже 1 день), и при этом указали группировку "Неделя", то в отчете отобразятся данные за полную календарную неделю/недели, указанную/указанные в выбранном на календаре периоде. 
    Выбран период 19 октября - 1 ноября 2023 года  на календаре, группировка Неделя, в отчете данные по полным календарным неделям: 16.10.2023 - 22.10.2023 и 23.10.2023 - 29.10.2023. так как 1 ноября - не последний день календарной недели, данные по недели, в которую входит 1 ноября, не отображаются


    Месяц:     если на календаре вы выбрали любой период (даже 1 день), и при этом указали группировку "Месяц", то в отчете отобразятся данные за весь месяц или месяцы, указанный/указанные в выбранном на календаре периоде. Если месяц закончен - данные будут по последнее число месяца, если не закончен - по сегодняшний день месяца.
    Выбран период 19 октября - 1 ноября 2023 года  на календаре, группировка Месяц, в отчете данные за весь октябрь и весь ноябрь (тк период на календаре находится в этих месяцах). 


    Год:     если на календаре вы выбрали любой период (даже 1 день), и при этом указали группировку "Год", то в отчете отобразятся данные за весь год, указанный в выбранном на календаре периоде. Если год не закончен, данные будут по сегодняшнее число. Если год закончен - по 31 декабря.
    Выбран 1 день  (1 ноября 2023) на календаре, группировка Год, в отчете данные за 2023 год (тк период на календаре находится в 2023 году)


    При переключении XRM пользователя между клиентами будет автоматически выбран последний использованный период.
    После того, как выбран период, нажимаем "Применить" для сохранения настроек.

  8. Сравнение периодов
    Если нужно в отчете сделать сравнение данных в разных периодах, нужно кликнуть на иконку "капли" (как на скрине)

    после клика на эту иконку откроется календарь, на котором нужно выбрать период, с которым хотим сравнивать:
    На скрине для сравнения выбрана дата 1.11.2022 - то есть, в отчете отобразятся данные за 1 ноября 2023 года в сравнении с данными за 1 ноября 2022 года.


    При этом результат в отчете будет выглядеть так:

    Как читать результат:
    1 ноября 2023 года (основной период) было 397 Активностей, это на 128 активностей больше (+128), чем было 1 ноября 2022 года. Данные, которые указаны в периоде сравнения (зеленое поле в правом верхнем углу) всегда будут отображаться цветным шрифтом рядом с данными, которые находятся в основном периоде и представлять собой величину, на которую данные в основном периоде отличаются от величины в периоде сравнения.

    Если в периоде сравнения выбран период, в котором нет данных по показателю, то данных о сравнении не будет.


    Если вы на основном календаре выбрали 2 месяца, а в периоде сравнения - 3, то сравнение будет только по первому и второму из трех месяцев. Это связано с тем, что на основном календаре выбрано 2 месяца.
    Например, на календаре выбрали август и сентябрь, а период для сравнения - март-май. В отчете будут данные о сравнении по месяцам августа с мартом и сентября с апрелем. Май будет сравниваться как бы с нулевым значением.



  9. Фильтры - Основной задачей фильтра является исключение или включение определенных данных по их прямым свойствам. Для разных показателей могут быть доступны разные фильтры (так как у разных данных могут быть разные свойства). Фактически фильтр применяется к выбранным показателям (к которым его возможно применить в связи с особенностью их данных).


    Чтобы добавить фильтр к отчету, нужно нажать "+" (как на скрине)


    В выпадающем списке можно ввести название фильтра либо выбрать, прокрутив вниз.

    После выбора фильтра появится окно выбора условия фильтрации: чаще всего это равно, не равно, содержит, не содержит, соответствует регулярному выражению, не соответствует регулярному выражению, пустое значение, не пустое значение.

    Каждое из условий указывает на определенную логику фильтрации
    Равно - полное соответствие
    Не равно - полное не соответствие
    Содержит - в искомых данных есть часть, которую указали в фильтре
    Не содержит - в искомых данных нет части, которую указали в фильтре
    Соответствует регулярному выражению - соответствует условиям, прописанным регулярным выражением
    Не соответствует регулярному выражению - не соответствует условиям, прописанным регулярным выражением
    Пустое значение - выбранный параметр не заполнен в данных
    Не Пустое значение - выбранный параметр заполнен в данных


    Для числовых полей могут быть условия больше, меньше, равно, не равно
    После того, как указываем условие фильтрации, нужно добавить параметр: например, как на скриншоте - email
    в зависимости от условия фильтрации это будет текст или регулярное выражение.

    Рядом с выбранным фильтром указано, к каким показателям он применится: на скриншоте это Активности.
    Это поле нужно, чтобы пользователь понимал, что к каким-то показателям из отчета фильтр невозможно применить.


    В отчете можно использовать несколько фильтров одновременно, чтобы добавить еще один фильтр, нужно снова нажать "+" Фильтр и задать условия фильтрации.

    При построении отчетов с большим количеством строк в отчет будет выводиться ТОП 250 строк по каждому показателю. Поэтому возможен такой эффект:
    - Если в 1 показателе строка попадает в топ 250, а в 2 показателе эта же строка не попадает в ТОП 250, то на фронте показывается просто 0.
    Пример на скрине: объявление попало в топ 250 по кликам, а в топ 250 по показам не попало. В связи с чем по кликам есть данные, а по показам нет.

3. Кольцевая диаграмма
с помощью такого вида удобно смотреть данные о долях (на скрине - доля активностей с каждого канала).
Во временных группировках (день, неделя и тд) такой вид не применяется (будет пустое поле отчета), если выбран 1 показатель. 

Если во временной группировке выбрать больше 1 показателя, у которых одинаковая единица измерения, то диаграмма отобразит соотношение по долям между выбранными показателями. На скрине - выбрано 2 показателя с одинаковой единицей измерения (Активности и Интересы). Диаграмма показывает как между собой соотносится кол-во Активностей и Интересов.




В группировках не по времени, если выбран 1 или несколько показателей, то диаграмма отобразит доли признака, который указан в группировке, для каждого показателя отдельно.

Пример с несколькими показателями: отображается 2 диаграммы, для каждого показателя отдельная.

Для группировок по источнику трафика доступна детализация (проваливание в данные более низкого "уровня" - группировки). Например, при группировке Каналы можно посомтреть данные по площадкам. Чтобы это сделать, нужно кликнуть на сектор или название канала на сноске: после клика открылась кольцевая диаграмма по площадкам (скрин справа). 

Далее можно аналогичным образом провалиться в группировку по кампаниям (тк она идет ниже по иерархии после площадок). в примере диаграмма, перестроенная по кампаниям из площадки MyTarget.

Чтобы вернуться на предыдущий уровень детализации, нажмите "Назад"

Для любого уровня детализации в такой диаграмме доступны функции скачивания и печати - для этого нужно кликнуть на иконку (см скрин ниже) и выбрать действие.



Можно скачать изображение в любом из доступных форматов или распечатать его. Также можно скачать данные в CSV. В таком случае скачается таблица с числовыми значениями по каждому сектору (не в процентах). Пример скачанной таблице по отчету со скрина выше: в столбце Category - список кампаний, по которым были договоры бронирования с контекстной рекламы. В столбце Series - кол-во договоров бронирования с каждой кампании контекстной рекламы.


4. Карта
Доступна только для группировок по гео (Город и Регион). С другими группировками не будет работать (будет пустое поле отчета). 
Чтобы выбрать такое отображение, кликаем иконку карты
При наведении курсора на карту, будет отображаться значение показателя в выбранном регионе. Регионы, где больше всего данных, на карте темнее.


5. Таблица

Если в отчете выбрано несколько показателей:

  3. Поделиться отчетом
    Можно отправить ссылку на отчет двумя способами:
    1. Отчет с построением - можно отправить только пользователям Smartis, у которых достаточно прав для просмотра данного отчета. У пользователя отчет будет построен заново
    2. Снимок отчета - отправить ссылку на изображение отчета любому пользователю, даже если у него нет доступа в Smartis. Ссылка будет работать в течение 30 дней.     После того, как вы сохранили отчет, он появится в списке виджетов во всех дашбордах и его можно будет добавить в любой дашборд.

Дополнительные действия с отчетом

  1. Форматирование - иконка "палитра"
    Позволяет "окрасить" столбцы отчета 
    Применяется только к отчетам "Таблица" и таблице в "Комплексном отчете"

При клике на иконку "палитра" добавляется 1 условие форматирования. Условий можно добавить несколько.

Выбираем столбец - столбец в отчете для которого нужно задать форматирование.
Далее выбираем способ форматирования: как на скриншоте

  1. Градиент от min к max - добавит к значениям в отчете градиент, максимальные значения будут выделены ярче (подходит, чтобы выделить самые большие значения, когда больше значит лучше - например, подсветить канал с самым большим кол-вом сделок)
  2. Градиент от min к max (негативный)- добавит к значениям в отчете градиент, максимальные значения будут выделены ярче (подходит, чтобы выделить самые большие значения, когда большие значения - это плохо. Например, дорогая стоимость обращения)
  3. Скрыть столбец - позволяет скрыть из отчеты выбранный столбец
  4. Скрыть строку - позволяет скрыть из отчета из выбранного столбца строку или строки , значения которой соответствуют заданному условию.
    Доступные условия: 
    • =
    • !=
    • >
    • <
    • >=
    • <=
    К условию нужно указать значение. Например, таким образом можно скрыть строки столбца, у которых значение = 0. В примере мы скрыли из столбца "Стоимость договора" все строки, у которых значение = 0.

  5. Условное форматирование - позволяет раскрасить столбец и его значения по заданным условиям. Доступные типы условия:
    • Заданное значение - позволяет раскрасить конкретные значения из столбца. В примере мы раскрасили значения столбца Стоимость договора, которые = 0.

    • Ячейки другой колонки - позволяет построчно сравнить значения выбранного столбца со значениями из любого другого столбца таблицы по заданному условию. В примере мы сравниваем значения столбца Договоры купли-продажи со столбцом Договоры бронирования и раскрашиваем такие значения столбца Договоры купли-продажи, которые больше (>) значений в аналогичной строке в столбце Договоры бронирования.

    • Min значение столбца - подсвечивает значения выбранного столбца, которые соответствуют условию (больше, меньше, равно и прочие) по сравнению с минимальным значением другого столбца
      В примере: в столбце Договоры бронирования минимальное значение - 1. в Договорах купли продажи все значение больше 1, поэтому выделен весь столбец.


    • Max значение столбца - подсвечивает значения выбранного столбца, которые соответствуют условию (больше, меньше, равно и прочие) по сравнению с максимальным значением другого столбца


    • Avg значение столбца - подсвечивает значения выбранного столбца, которые соответствуют условию (больше, меньше, равно и прочие) по сравнению с средним арифметическим значением другого столбца

    Доступные условия: 
    • =
    • !=
    • >
    • <
    • >=
    • <=
    также можно выбрать любой цвет для раскраски столбца и его значений: после выбора цвета нажать "ОК"
     

    Форматирование таблицы сохранится, если вы сохраните сам отчет. Если вы перестроите отчет, то форматирование нужно будет настраивать заново.
    Можно применить форматирование ко всем столбцам таблицы, также может быть одновременно несколько способов форматирования одного столбца - например, может быит примененна раскраска и при этом быть скрыта строка из отчета.

Настройка отображения данных в таблице

  1. Показывать по ...
    Выбор кол-ва строк, которы отображаются на 1 странице таблицы
  2. Поиск по таблице
    Можно ввести тест, по которому выполнится поиск по строкам таблицы и в таблице останутся только строки, которые соответствуют условию поиска
    Итоговое значение при этом пересчитываться не будет! (отобразится общий итог, полученный при построении отчета)

  3. Сортировка данных в столбце
    С помощью иконки (выделено на скрине) можно сортировать данные в таблице по столбцу. Чтобы применить сортировку, нужно кликнуть на иконку возле названия показателя


    Сортировка по возрастанию значений столбца - когда стрелка указывает вверх, таблица сортируется по выбранному столбцу, при этом наибольшие значения выбранного столбца располагаются вверху.


    Сортировка по убыванию значений столбца - когда стрелка указывает вниз, таблица сортируется по выбранному столбцу, при этом наименьшие значения выбранного столбца располагаются вверху. Чтобы сортировать по убыванию, нужно кликнуть еще раз на иконку. 


    Чтобы отменить сортировку, нужно кликнуть на иконку еще раз.
    Одновременно к таблице можно применить сортировку только по 1 столбцу. 
  4. Вложенность отчета
    При использовании группировок по источнику обращения (Канал, площадка, категории кампании, кампания и тд) доступно "проваливание" в отчете. Оно работает в логике иерархии источников и позволяет не перестраивая отчет загрузить данные в более детальной группировке (Например, при группировке каналы - посомтреть площадки, кампании и тд). При просмотре площадки - кампании и группировки по utm. Важно, что вложенность всегда работает от "общего к частному" - то есть из группировки по площадкам нельзя провалиться в "Каналы".

    Чтобы посмотреть, какие уровни вложенности доступны, необходимо навести курсор на строку в таблице- появится иконка стрелки вниз:

    Далее нужно кликнуть на иконку и выбрать нужную детализацию

    Мы выбрали детализацию "Площадки" и в отчете появилось отображение по площадкам, которые содержит канал "Лидогенерация"

    далее мы можем также углубится, и выбрать еще уровень детализации для строки с названием площадки: аналогично наводим курсор на строку с названием площадки и кликаем - появилась новая детализация - Кампании. Выберем ее

    Появилась строка с кампанией. По клику нас релку возле названия кампании можно также посмотреть следующий уровень детализации: объявления, utm, ключевые слова

    Не у всех рекламных источников можно посмотреть статистику по объявлениям и ключевым словам - у Лидогенерации таких данных нет.

    Пример проваливания до объявлений (у Контекстной рекламы) - у лидгена нет данных по объявлениям.

    Если для строки недоступно проваливание, то по клику на стрелку не откроется окно с выбором детализации данных.

    Проваливание можно применить к нескольким строкам в отчете, но настраивать его нужно для каждой строки отдельно. Также проваливание не сохранится если вы скачаете, поделитесь отчетом или сохраните отчет. В скачанном, отправленном или сохраненном отчете будет отображаться таблица в первоначальном виде.
    Проваливание также можно применять в таблицах в дашбордах.
    Чтобы скрыть строки в детализации, нужно нажать на иконку напротив строки
