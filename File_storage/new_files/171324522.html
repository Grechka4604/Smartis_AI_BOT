Основной проект

Flexbe – это сервис для создания квизов, лендингов, а также для сбора обратной связи с сайта компании. В Smartis доступна интеграция с сервисом для того, чтобы собирать данные по лидам, отправленные через формы flexbe. С помощью интеграции можно отслеживать заявки, их источники, ответы на квиз и переписку с клиентом из Flexbe в Smartis.

Интеграция работает через API. В Smartis получаем данные по лидам.

Подключение интеграции

  1. Запросить доступы у клиента для подключения интеграции flexbe (также клиент может подключить сам, не передавая доступы вам). Нужен логин, url страницы сайта и API токен из flexbe. API-токен Токен можно получить у вашего менеджера Flexbe по запросу: Просьба предоставить API токен для интеграции с Flexbe
  2. В Smartis перейти в раздел Администрирование → Интеграции и в разделе "Доступные для подключения" выбрать интеграцию flexbe
  3. Нажать "Подключить" и заполнить данные для подключения. В поле формы API токен из flexbe ввести  Api-токен полученный в сервисе. Для каждого сайта подключается отдельная интеграция (так как для подключения указывается только 1 url страницы сайта, с которой нужно получать данные)
    1. Откуда брать URL:
      1. Переходим на главную страницу в кабинете Flexbe.
      2. Проваливаемся в интересующий нас сайт и переходим раздел настройки → домены.
      3. Копируем основной домен.

  4. После подключения интеграция отобразится в разделе "Подключенные интеграции" (Администрирование → Интеграции).После подключения в клиенте интеграции с сервисом flexbe запускается первичная синхронизация данных.
    Если лиды нужно получать по нескольким сайтам, пункты 2 и 3 выполняются столько раз, сколько сайтов нужно подключить


    Данные по интеграции после подключения автоматически забираем за 30 дней в прошлое, с момента подключения интеграции. Если нужно обновить другой период - воспользуйтесь ручным обновлением  тут: 9. Администрирование: Мониторинг системы

Привязка интеграции к сайту

Пока интеграция не привязана к сайту, данные по ней невозможно просматривать в отчете по данному сайту в Smartis

  1. Зайти в раздел Администрирование → Проекты и сайты → Сайты и найти в списке нужный сайт. 
  2. Кликнуть на название сайта - откроется меню настройки привязок интеграций к сайту. 
  3. В разделе "Интеграции доступные для привязки" найти интеграцию flexbe и нажать "Привязать с логином".

  4. Из выпадающего списка выбрать логин, по которому приходят лиды для выбранного сайта (можно выбрать только 1 логин), если к сайту нужно привязать несколько логинов - пункт 3 и 4 выполняются столько раз, сколько логинов нужно привязать. Можно также выбрать "Все данные этой интеграции", если нужно привязать все логины сразу. После выбора логина интеграция привяжется к сайту. 
  5. Если нужно изменить логин, по которому получаете данные из Flexbe, можно в уже привязанной интеграции выбрать другой логин и подтвердить изменения.

  6. Для каждой привязанной к сайту интеграции можно дополнительно настроить условия привязки. 

    Пример: в логине "Застройщик" есть информация о лидах по ЖК1, ЖК2 и Бренду. Нам нужно сделать так, чтобы к сайту ЖК1 были привязаны только те лиды, которые поступают по этому ЖК.
    Как это можно сделать:

    1. Наименование логина интеграции соответствует регулярному выражению - если к сайту привязаны "все данные всех интеграций сервиса", и вам нужно выбрать сразу несколько логинов с определенным выражением в названии логинов, можно использовать этот способ. Такой способ применим если вам нужно привязать все данные из нескольких логинов.
    2. Страница обращения соответствует регулярному выражению - подходит в случае, когда лиды по данному ЖК поступают на определенный сайт.  Url страницы обращения указываем в виде регулярного выражения. 
    3. utm_campaign соответствует регулярному выражению - если нужная метка указывается в utm_campaign лида, то можно использовать этот способ. С помощью регулярного выражения нужно указать все возможные значения меток.

      Эти условия работают как сужающие. То есть, если вы указали например определенные логины и прописали условия по utm_campaign - то к сайту привяжутся только те данные, которые соответствуют 2 этим условиям (лиды из выбранных логинов, у которых в метке utm_campaign есть прописанное вами условие). 



Показатели

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. 




  1. лиды Flexbe - отображает данные по лидам (источник лида определяется на основе данных рекламной кампании Flexbe, по которой пришел квиз)
  2. лиды Flexbe (по моделям атр.) - отображает данные по лидам, которые строятся по моделям атрибуции- то есть источник лида определяется по-разному в зависимости от выбранной в отчете модели атрибуции. Про модели читать тут: 4. Модели Атрибуции

Детализация данных Flexbe


Для данных Flexbe доступна детализация (исходные данные). Они такие же, как данные по заявкам у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.


Общая справка по детализации данных сервисов обращений и квизов:
3. База обращений
6. Исходные данные (Детализация, Сырые данные)

Основные свойства, передаваемые в лидах Flexbe:

  • Данные о свойствах обращения из Flexbe: внешний ID, сайт, дата создания, запись разговора (ответы в квизе), теги
  • Данные о клиенте: номер телефона, регион, IP адрес и т.д.;
  • Данные о рекламном источнике обращения: utm_метки, gclid, yclid, страница входа, ссылающаяся страница

Для Flexbe в детализации доступны также такие наборы данных (столбцы) которые можно просматривать и при необходимости фильтровать:

  • Flexbe: page.name

  • Flexbe: ip_address
  • Flexbe: ya_client_id
  • Flexbe: utm_source
  • Flexbe: utm_campaign
  • Flexbe: utm_medium
  • Flexbe: first_page
  • Flexbe: request_page
  • Flexbe: conversation_number_in_service
  • Flexbe: conversation_number

Привязка источника обращения Flexbe в Smartis


Когда мы получаем данные из Flexbe, они изначально не относятся ни к каким рекламным источникам - привязка происходит в Smartis (из Flexbe мы получаем utm метки, на основе чего и выполняются привязки по условиям площадок). О привязках по условиям к площадкам в Smartis читать тут: 2. Каналы и площадки

Если обращения из Flexbe не привязаны к площадке, то они будут попадать в площадку "Другое" в отчете по площадкам/каналам.


В СJM клиента обращения из Flexbe отображаются так:


Отображаются:


  • тип обращения 
  • телефон клиента 
  • Канал и Площадка (на основании того, к какой площадке мы привязали данные обращения)
  • регион
  • utm метки - мы их также получаем из Flexbe
  • Ссылающаяся страница - страница, с которой пользователь перешел на сайт

  • Страница входа - страница сайта, на которую зашел пользователь перед совершением обращения

  • Google client ID - id пользователя в Google 
  • Yandex client ID - id пользователя в Яндекс метрике 
  • IP адрес
  • При наличии тегов - отобразятся и теги. Теги можно проставить и в Smartis, но в таком случае в Flexbe теги не передадутся.
  • история переписки (ответы на вопросы) - открывается по клику на иконку истории переписки
