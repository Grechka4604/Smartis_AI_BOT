Интеграция не используется, нет клиентов, кому это было бы полезно.

Основная информация

Ссылка на документацию для клиента на сайте Smartis https://smartis.bi/faq/intieghratsiia-adfox-i-smartis-instruktsii-po-podkliuchieniiu-i-ispolzovaniiu/

ADFOX - это бесплатный онлайн-сервис по управлению рекламой на сайте для издателей. Он помогает владельцам сайтов, которые размещают у себя рекламу (медийные площадки) - делать это эффективно и через интерфейс с кучей функционала (права, статистика, структура рекламы, архивы и т.д.). Через этот сервис управляют рекламой уже более 1000 медийных площадок, среди которых rbc.ru, cian.ru, tass.ru и т.д.

Официальная промо-страничка: https://adfox.yandex.com/promo

Чтобы понять эффективность медийной рекламы - нужно ее измерить. В Adfox есть только агрегированная статистика - можно узнать сколько показов/кликов по дням было по кампаниям и объявлениям. Но узнать сколько продаж или лидов было благодаря медийной рекламе? Как она взаимодействовала с контекстной рекламой? Приходят ли клиенты через неделю к вам на сайт, если не кликали по баннерам? - становится возможным только вместе со Smartis.

Что делает интеграция Smartis с Adfox?

Благодаря интеграции, каждый показ баннера или видео размещенный через Adfox - будет зафиксирован в виде сырых данных. При этом будет идентифицирован каждый пользователь внутри Smartis. Благодаря этому - любая дальнейшая конверсия: заказ на сайте, звонок, лид из другой рекламной системы (например ВК или MyTarget), встреча в офисе продаж, или продажа - будет связана с показом конкретных медийных баннеров или видео роликов.

Это позволяет:

  1. Честно измерить эффективность медийной площадки, чтобы медиапланировать на данных
  2. Сравнить медийные площадки между собой в одинаковых методиках
  3. Понять какие баннеры/креативы/видео-ролики эффективны, а какие нет
  4. Понимать динамику - новые настройки помогли или нет? (за счет оценки по когортам)

  5. Более детально настраивать рекламу по сегментам целевой аудитории и понимать эффективность каждой кампании

    По всем сервисам Postview работает 1 синхрон: описан тут 2. Post-view

Как подключить?

  • Заходим в личный кабинет Smartis (облачная версия*) в раздел интеграций - https://my.smartis.bi/admin/integrations. *Для on-premise версий смотрите ниже отдельные комментарии.

  • Находим в списке интеграцию "Postview ADFOX" и нажимаем подключить:

  • Введите название интеграции - любое, главное, чтобы вы понимали с какой площадкой вы в этой интеграции интегрировались

  • Получение API токена:

    • У вас должен быть доступ в Adfox. Если вы рекламодатель - то владелец площадки обычно выделяет на ваш Яндекс аккаунт выделенный доступ к вашим кампаниям. Если еще не выделялся - запросите его у вашего менеджера, через которого вы размещаетесь на этой медийной площадке

    • Авторизовавшись в Яндексе под этим доступом - зайдите на страницу: https://developer.tech.yandex.ru/

    • Подключите API "ADFOX"
    • После того как подключите апи - вас перебросит на страничку где как раз можно взять API токен:

      • Выберите к какой рекламной площадке Smartis относится интегрируемая площадка в выпадающем списке. В отчетах по каналам/площадкам данные по показам из adfox будут попадать именно в эту площадку

      • Нажмите "Подключить"

    В течение 10-30 мин (в зависимости от количества объявлений в вашем кабинете Adfox) - все объявления будут размечены с помощью Smartis pixels и с этого момента postview данные начнут собираться внутри Smartis.

    Если в уже подключенном кабинете Adfox будут появляться новые объявления - они будут размечены пикселями Smartis каждый день с 21 до 22 часов.

Чтобы все работало, не меняйте автоматически установленное смартисом значение параметра Ссылка на промерочный пиксель (trackingURL в API) и URL перехода (user1 в API). Статистика внутри самого Adfox будет поступать вне зависимости от значения этих параметров.

Несколько одновременных промерочных пикселей можно добавить только через интерфейс ЛК, через API это невозможно, поэтому одновременно два аналитических сервиса для postview аналитики на одни и теже объявления использовать нельзя

Не переживайте, что значение параметра URL перехода заменилось на ссылку пикселя Smartis - если вы перейдете по этой ссылке - то попадете как раз по тому адресу, который был настроен изначально, со всеми метками


Как пользоваться?

  1. В конструкторе отчетов Smartis (https://my.smartis.bi/reports/designer) данные о показах и кликах доступны в системном показателе "Просмотры медийных размещений"
    1. Для отчетов с этим показателем доступна фильтрация по свойствам показов, группировки по основным свойствам. С помощью этого функционала, вы можете добавить в сквозные отчеты показатели о просмотрах, кликах
    2. Лиды, CRM-данные при выборе моделей атрибуции в которых есть "с учетом postview" будут учитывать вес всех показов, сессий, обращений до конверсии в соответствии с моделью (подробнее разбирали в статье).
    3. Для более сложных аналитических задач - есть специальный когортный фильтр "Person: когорта персон". Благодаря этому фильтру вы можете получить сколько с конкретной когорты просмотров было лидов, сделок, переходов на сайт и т.д.
    4. В Карточках клиентах (CJM) отображаются просмотры конкретным пользователем конкретных объявлений
    5. В отчете в группировке по объявлениям, если это объявление графическое - будет отображаться креатив

    Для on-premise (коробочных) версий Smartis

    Функционал postview анализа для коробочных версий также доступен, но непосредственное подключение кабинета Adfox нужно в первую очередь производить в облачном кабинете Smartis (у каждого on-premise клиента есть облачная оболочка для работы технологий, которые могут работать только через облако smartis - web-аналитика, postview, fingerprint и т.д.).

    После того как в облаке подключили Adfox, разметились объявления и появилась статистика по просмотрам - коробка будет подтягивать данные из облака Smartis. Чтобы данные postview в коробке начали влиять на анализ по моделям атрибуций в конверсиях - необходимо включить интеграцию-пустышку с Adfox.
