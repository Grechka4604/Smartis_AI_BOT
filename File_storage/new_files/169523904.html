Данный регламент описывает правила проведения встреч по статусам, какую информацию необходимо подготовить. 


  1. Встреча по подведение статусов проводится 1 раз в неделю, каждый вторник с 10 до 10:45 
  2. Для проведения встречи необходимо предварительно (в пятницу/понедельник) актуализировать статус клиента
  3. Голосом на встречи проговариваем показатели по клиенту: 
    - модуль достоверности за неделю
    - текущий процент матчинга
    - за прошедшую неделю были ли зафиксированы баги по клиенту
    - баги (текущий статус, решена, не решена)
  4. Формат актуализации статуса: 
    - цель с клиентом, согласованная на текущий отчётный период; 
    - что сделано за предыдущую неделю (ключевой тезис - перечень задач, которые выносятся в данный блок должны вести к выполнению поставленной цели, мы не добавляем всё подряд); 
    - что блокирует; 
  5. Доклад по клиенту выполняет ответственный PM. 
  6. На встрече мы проходимся по текущим статусам, стараясь глубоко не укапывать в каждого клиента. Если понимаем, что для разбора ситуации по клиенту требуется больше времени, то назначаем отдельную встречу.
