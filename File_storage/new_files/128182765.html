    Тарификация: 
  •  Виджет - лидогенератор  - 5 000 рублей 

    Состав решения: 


    1. Конверсионные виджеты захвата контактов для сайта. 


    • Включают в себя следующие комбинации форм сбора контактов : телефон/email/имя. 


    • Наши UX-дизайнеры могут разработать под сайт уникальный дизайн виджетов, а разработчики встроят их в SВ для использования всех возможностей платформы. Некоторые примеры дизайнов, которые мы делали для наших клиентов - в небольшой презентации.


    • Алгоритм нашего виджета учитывает все значимые параметры поведения посетителя на сайте и с каждым новым клиентом эффективнее выполняет свою задачу распознавания целевых посетителей. Таким образом, наш виджет адаптируется к каждому сайту индивидуально. Чем больше статистики, тем выше КПД.


    2. Настройка сегментов


    • Виджет показывается в нужный момент. Важно не только выявить заинтересованного посетителя, но и предложить ему звонок в подходящий момент - чтобы, с одной стороны, не отвлекать его от изучения сайта, а с другой, упредить потерю интереса своевременной обратной связью. Наша команда исследует новые критерии оценки вовлечённости посетителей.


    • Есть возможность создавать собственные сегменты пользователей/групп пользователей для настройки сценариев показа виджета. На данный момент в качестве условий создания сегмента учитывается : пол/гео/время на сайте/поведение клиента на сайте/посещение определенных разделов сайта/ клики на элементы/ попытки уйти с сайта 

    3. Аукцион текстов 


    • Мы загружаем в систему тексты, при необходимости группируем их по разным категориям, к примеру: Преимущества, Ипотека, Акции, Отделка и т.п. После чего система запускает в тестовом режиме отображение данных текстов и собирает статистику по итогу их показов.
    • Тексты показываются последовательно, в равномерном распределении, т.е. число показов к подведению итогов по всем текстам будет одинаково, как только по каждому тексту набирается 1000 показов система снимает статистику по конверсионности и отдает предпочтение наиболее конверсионному варианту. Таким образом мы полагаемся не на человеческий фактор, какой посыл будет лучше, а используем действительно самый конверсионный текст.


    4. Интеграция с контекстной рекламой


    • Smartis анализирует запросы клиентов, по которым они попадают на сайт девелопера. После чего подстраивается под них, предлагая посетителям то, зачем они пришли. 
    • Мы подключаем интеграцию с рекламных кабинетов Яндекс.Директ или Google Ads, собирая в систему все рекламные кампании. Можем автоматически подтянуть в виджет текст из объявления или прописать его под каждую кампанию — при переходе на сайт клиент увидит в виджете то, что совпадает с его запросом.
    • Пользователь ищет в поисковике варианты квартир с конкретными вводными данными, например: «квартира на берегу реки» или «квартира с видом на парк». При переходе на сайт застройщика появляется виджет с текстом под его запрос: «Панорамные виды на Павелецкую набережную! Закажите обратный звонок и мы подберем для вас лучший вариант».

    11. Работа с Потенциальным клиентом 


    • Человек вводит в виджет номер телефона, но по какой-либо причине не нажимает кнопку «Позвонить». Мы следим, если в течение получаса (это период можно изменять) пользователь так и не оставляет заявку, то мы заносим его номер в лк с пометкой "Потенциальный клиент» 
    • Есть несколько вариантов, как можно этот функционал использовать:
    • Не прозванивать такие заявки, а только использовать для рассылок, настроек таргета и т.п.
    • Отправлять уведомления при создании такой заявки с пометкой "потенциальный клиент", чтобы операторы по особому скрипту их обрабатывали
    • Выделить человека, который через наш ЛК будет запускать на прозвон такие заявки, при этом ему будет доступна карточка со всей историей этого человека и он сможет подготовиться к звонку.

    16. Работа с Потенциальным клиентом 


    • Человек вводит в виджет номер телефона, но по какой-либо причине не нажимает кнопку «Позвонить». Мы следим, если в течение получаса (это период можно изменять) пользователь так и не оставляет заявку, то мы заносим его номер в лк с пометкой "Потенциальный клиент» 
    • Есть несколько вариантов, как можно этот функционал использовать:
    • Не прозванивать такие заявки, а только использовать для рассылок, настроек таргета и т.п.
    • Отправлять уведомления при создании такой заявки с пометкой "потенциальный клиент", чтобы операторы по особому скрипту их обрабатывали
    • Выделить человека, который через наш ЛК будет запускать на прозвон такие заявки, при этом ему будет доступна карточка со всей историей этого человека и он сможет подготовиться к звонку.
