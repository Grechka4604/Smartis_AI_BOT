ИНИЦИАЦИЯ ПРОЕКТА

Бизнес-идея (боль) проекта

На текущий момент в Smartis процесс заключения договора с новым клиентом неудобен и болезненен для всех участвующих в нем сторон: и для продавца Smartis, и для юридической службы Новосистем, и для клиента.

  • Заключение договора происходит долго и зачастую требует многочисленных итераций согласования и внесения правок в договор.
  • Часто возникают нарекания к процессу у всех участвующих в нем сторон.

С учетом этого было принято реш­­­­ение создать новый процесс на основе старого, пересмотрев и актуализировав все входящие в него компоненты:

  • Непосредственно сам процесс.
  • Тарифные планы Smartis (их архитектура и параметры).
  • Договор и все его составляющие.
  • Внутренние регламенты, связанные с процессом.
  • Клиентские регламенты, связанные с процессом (связанные с договором документы, которые отправляем клиенту).

Цели проекта

Что команда должна получить по завершении проекта?

1. Эффективная* модель взаимодействия с клиентом на этапе заключения договора

2. Юридическая договорная конструкция, соответствующая бизнес-модели заключения договора с клиентом, допродажи и сопровождения клиентов

3. Исключить Новосистем как агента по взаимодействию с клиентами

Задачи проекта

Действие, которое необходимо совершить для достижения вашей цели

На первую часть проекта/2023Q3

1. Описать и согласовать со всеми стейкхолдерами новую модель заключения договора с клиентом Smartis (новый процесс заключения договора).

2. Разработать новый договор (полный комплект документов для заключения договора с клиентом).

3. Разработать и валидировать набор гипотез по ускорению/упрощению процесса заключения договора.

На вторую часть проекта/2023Q4

1. Внедрить новый договор (запустить работу с новым договором и связанным с ним процессами)

2. Внедрить изменения в процесс с учетом валидированных гипотез.

Продукт проекта

Инструмент для достижения цели проекта

Работающий процесс заключения договора с клиентом (алгоритм действий для всех участников процесса)
Выгода для компании
  1. Высвобождение времени сотрудников (продавцов). Это время можно использовать для каких-то других задач, ведущих в получению прибыли.
  2. Быстрый апсейл (быстрая допродажа новых продуктов действующим клиентам).
  3. Перестать терять продавцов из-за демотивации при работе с процессом заключения договора с клиентами.
  4. Качество сделки, минимизирующее вероятность наступления негативных событий в дальнейшем на жизненном цикле клиента (например, договоренности с клиентом не закреплены в документах, в результате клиент отказывается платить или платит меньше, что то не учтено было на этапе продажи, потом в консалтинг сваливалось проблемой - счета на выставились, формулировки не устраивают клиента, документов не хватает, проблему начинают исправлять вместе с бэк, разработчиками)

Критерии завершения проекта

Какие ключевые точки оценки результатов проекта?

Удовлетворённость результатами проекта всеми вовлечёнными в него сторонами (нематериальный результат)

Цикл сделки:

  • количество итераций согласования
  • количество времени на согласование
  • количество вносимых клиентом правок/запросов на изменения

Валидировать - каковы значения сейчас? Что будет для нас успехом, какое изменение метрик?
