Основная информация

AdRiver - Это рекламный сервис для:

  • валидации трафика (робот смотрит на баннер? конкурент?), 
  • контроля brand-safety (можно ли показывать застройщика ФСК на порно-сайтах? а на форуме про разорившихся застройщиков?)
  • управление медийной частью медиасплита (структура кампаний, метки, расходы)
  • измерения медийной-охватной рекламы (сколько людей видело баннер/видео?, сколько уникальных?, сколько досмотрело видео до конца? сколько кликало?)

Мы пересекаемся по 4-му пункту функционалом, но наше главное отличие в том - что мы матчим эти показы с CJM покупателей/звонивших, показываем конверсии в воронке после просмотра, a AdRiver показывет просто статистику показов/кликов

Мы знакомы с коллегами, общаемся на уровне основателей и менеджеров поддержки, в прямую не конкурируем, а скорее дополняем друг друга там где мы не можем измерять или не умеем закрывать остальные их задачи.

По всем сервисам Postview работает 1 синхрон: описан тут 2. Post-view

Как подключить

  1. Наши клиенты самостоятельно заключают договора с AdRiver, мы не участвуем в этих процессах
    1. Чтобы был корректный матчинг (он у нас идет через google client id) - все сайты клиента должны быть зарегистрированы в AdRiver и на каждом сайте клиента должен быть установлен код AdRiver (он для каждого сайта свой - выдается после регистрации сайта в AdRiver)
  2. Чтобы данные скачивались Интеграцию необходимо подключать в облаке. В коробках подключаем "плюшевую" интеграцию, чтобы данные участвовали в анализе налету и показывались в CJM. (В коробках скачивание данных организовано через наше облако, для скачивание через отдельные интеграции требуется включать синхроны поствью и отлаживать как будут вести себя коробки => пока этого не требуется, клиенты не требуют подключения исключительно в коробке)
  3. При подключении интеграции есть актуальная инструкция:
  4. Пройти по инструкции в описании интеграции, заполнить адрес и сохранить
    1. Адрес папки правильнее задать самим при запросе, например "Просьба передавать post-view данные в FTP аккаунт Smartis https://files.adriver.ru/Smartis.ftp/ в папку LSR". Тогда вам не надо будет никого дергать и дополнительно уточнять что прописать в интеграции => в этом примере прописываем https://files.adriver.ru/Smartis.ftp/LSR/
    2. Если по каким-то причинам не говорят в какую папку сложили, или вы забыли указать в какую складывать → просим Валеру или Игоря посмотреть папки на FTP Adriver
      1. Для Валеры: в prod env есть хост/пароль/юзер
  5. Со следующего дня, если в указанную папку Adriver выложит файл (бывают случаи что забывают или не успевают) - до 17 часов должно успевать скачиваться "за вчера"
    1. Если данных очень много (как у ФСК по 20млн/день) => данные могут завершать весь процесс через день
  6. Рекомендуется свериться с личным кабинетом AdRiver за 3-7 дней статистики, нормальная сходимость 95%


Как пользоваться

Данные Adriver как и остальные сервисы складываются в системный показатель "Просмотры медийных размещений"

Если нужно отдельно выделять данные этого сервиса - есть соответствующий фильтры:

  • Медийка: Сервис Post-view
  • Медийка: Тип события
    • Клик
    • Показ
    • Старт - Окончание загрузки и начало воспроизведения конкретного рекламного креатива
    • Просмотр 25% - Первая четверть рекламного ролика
    • Просмотр 50% - Середина рекламного ролика
    • Просмотр 75% - Третья четверть рекламного ролика
    • Просмотр 100% - Окончание просмотра рекламного ролика

Чтобы данные Adriver участвовали в анализе по МА и показывались в CJM - должна быть в кабинете Smartis неудаленная интеграция (статус не важен). Поэтому в коробках подключаем "плюшевую интеграцию"

Работа с placement

Если клиенту нужно анализировать эффективность рекламы не только по источнику, но и по конкретному месту размещения баннера, Adriver позволяет автоматически определять эту позицию с помощью макроса {placement}.

Для этого специалист, управляющий рекламным кабинетом Adriver, должен настроить передачу этого параметра.

Со стороны Smartis для анализа таких данных используется параметр utm_content.

Плейсмент – место на сайте, где размещена реклама (баннер в шапке, боковой блок, вставка в тексте).

Макрос – переменная, заменяемая системой на реальное значение ( {placement} → banner_top )

Загрузка данных за ретро период

Агентство или застройщик, работающий с AdRiver, должен запросить у AdRiver лог данных для загрузки. После получения информации лог необходимо передать нам.

Обычно лог включает в себя два типа файлов:

  • Статистика показов – 12345.logad.csv.gz
  • Лог сайтов (влияет только на матчинг с куками) – 12345.logsite.csv.gz

Лог хранится ограниченное время, поэтому важно сразу после получения информации проверить наличие файлов и скачать их. Для этого создаём задачу в консультацию - Шаблон №104: AdRiver - импорт отдельных файлов

Если клиент передаёт большой объём старых данных (например, за полгода или год), объём хранения может значительно увеличиться, что повлияет на стоимость тарифа. Поэтому важно заранее согласовать условия: превышение установленного лимита либо оплачивается отдельно, либо требует перехода на более подходящий тарифный план.

Важные ньюансы:

  1. Канал/Площадка по просмотру из Adriver определяется по utm условиям наших площадок (проходят через наш центральный механизм определения источников по utm)
  2. У AdRiver для некоторых площадок есть специфичная разметка без utm но с другими спец параметрами. Это у нас не учтено в интеграции, данные будут складываться в прямой трафик, если там нет utm
  3. Значительная часть показов (до 80-90%) Adriver приходит без google client id, поэтому даже попытаться сматчить мы не сможем. Это означает что пользователь видевший рекламу еще не заходил на сайты клиента. Это главный минус этой интеграции из-за которой мы рекомендуем пользоваться нашим пикселем 

  4. Если данные не сходятся - в большинстве случаев это означает, что какие-то рекламные кампании не делегированы на стороне AdRiver для выгрузки к нам в логи (через владельцев аккаунта Adriver надо делать запрос "делегировать такие то кампании, чтобы выгружались логи post-view в папку Smartis и предоставить выгрузку данных по этим кампаниям до даты включения выгрузки").
