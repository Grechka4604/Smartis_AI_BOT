О Циан, вводная информация

Циан - это большая экосистема продуктов на рынке недвижимости для разных потребителей:

  • Для рекламодателей: реклама на медийных площадях их сайта + мобильного приложения => Медийная реклама
  • Для рекламодателей: размещение фида квартир и ЖК на своем сайте и мобильном приложении с последующим обращением клиента через заявки и звонки => Базы Недвижимость
  • Для рекламодателей: Циан.Аудитории - по сути ведут рекламу в пользу застройщика, используя данные которые они знают о своих юзерах, в собственных рекламных аккаунтах Циан в Яндекс.Директе/ВК и т.д.
  • Для Риэлторов: Сервисы работы с объявлениями и их реализацией на первичном и вторичном рынке (в том числе реклама этих объявлений в пользу риэлтора)
  • Для людей: Циан.Сделка и т.д.

Далее речь идет о том как работает интеграция с Медийной рекламой на ЦИАН

Важно: Решение работает со стандартными форматами медийки на Циан, спец.форматы размечаются отдельно через обращение к менеджеру ЦИАН, а не Smartis. 

Как выглядят разные форматы, можно посмотреть тут https://promo.cian.site/developers. 

Какая ценность от интеграции post-view Циан

  1. Оценка реальной эффективности медийной рекламы на ЦИАН  на сквозной воронке (показатели ДРР, СРА с учетом моделей атрибуции, Конверсия в сделки, средний чек, цикл сделки) для управления медиасплитом в целом (перераспределение рекламного бюджета с целью получения большей выручки/прибыли)
  2. Управление медийными рекламными кампаниями на Циан (какие форматы выбрать, какие креативы лучше, какие рекламные кампании более эффективны, улучшились ли показатели при росте просмотров, на каких устройствах больше конверсия и т.д.)

Как работает интеграция post-view ЦИАН

  1. Застройщик размещает медийную рекламу на ЦИАН 

  2. Пользователь видит на ЦИАН медийную рекламу 

    1. На web-ресурсах ЦИАН (сайт cian.ru, другие партнерские сайты, которые купил циан)

      1. ЦИАН присваивает ему cian_client_id и сохраняет в логах, что именно видел пользователь, когда и другие свойства события и размещения.

      2. Когда этот пользователь зайдет на любой из сайтов любого клиента Smartis с включенной интеграцией - мы произведем cookie-матчинг за счет установки безопасного пикселя на сайт и сохраним связку cian_client_id => smartis_client_id

        1. cookie-матчинг может быть как после просмотренной рекламы пользователем, так и до нее - то есть пользователь мог зайти на какой то из сайтов застройщиков раньше - тогда мы все равно сохраним связку cian_client_id => smartis_client_id

      3. Мы ежедневно скачиваем из ЦИАН по специальному закрытому API (эксклюзивно только для нас сделанному) - логи всех зафиксированных событий в хранилище post-view

      4. Когда мы анализируем конверсию по источникам, зная smartis_client_id (pid) → мы находим cian_client_id → по нему все события на веб-ресурсах циан

    2. В мобильном приложении ЦИАН

      1. ЦИАН знает за счет регистрицации / авторизации / оставленных заявок / звонков через колтрекинг связку пользователь мобильного приложения => телефон 
      2. Мы ежедневно скачиваем данные о всех событиях в мобильном приложении Циан по специальному закрытому API получая в качестве идентификатора пользователя зашифрованный хитрым сложным алгоритмом (на основе SHA256 со специальными ключами шифрации, которые известны только серверу циан и серверу смартис) телефон
      3. Когда мы анализируем конверсию по источникам, зная телефон → мы его шифруем в том же алгоритме → по полученному идентификатору находим все события в мобильном приложении ЦИАН этого пользователя

Cookie-матчинг на web-сайтах застройщика

    1. В целом этот процесс аналогичен например размещению пикселя AdRiver/Weborama +ранее DCM, только наш еще сильнее защищает застройщика, и не требует запросов к разработчикам сайтов и ручной установки пикселей.

Циан MobileApp Post-view           

Поскольку логика обмена данными другая, нужно отдельное разрешение клиента на такой обмен, и поскольку клиент может закупать отдельно рекламу на веб и в мобильном приложении - интеграции для Web и для MobileApp - подключаются отдельно.                                     

  1. Перед тем как подключить интеграцию - обязательно проговариваем с клиентом чем как именно происходит обмен данными и получаем от него явное письменное согласование:
    1. Предварительно на встрече рассказываем:
      1. для матчинга медийной рекламы в мобильных приложениях нужно подключить отдельную интеграцию
      2. для матчинга эта интеграция будет запрашивать у закрытого API Циана post-view события по списку зашифрованных телефонов
      3. Алгоритм шифрации необратим и очень надежен (SHA256 со специальными ключами шифрации, и многократными циклами шифрования, которые известны только серверу циан и серверу смартис). Шифрование одного телефона длится 3 секунды (чтобы перебрать все мобильные телефоны и получить базу таких хешей надо 34 722 дня) + ключи шифрования периодически меняются. Это делает невозможным за счет перебора сопоставить телефоны третьим лицам вне целевого применения.
      4. И самое главное за счет чего застройщик может быть спокоен за свои данные - когда мы спрашиваем ЦИАН API:
        1. мы не передаем информацию о том какого застройщика/ЖК этот зашифрованный телефон => это обобщенный набор всех застройщиков подключенных к решению
        2. мы не передаем информацию о том, что за событие стоит за этим телефоном - это обобщенный набор всех событий обладающих телефоном (включая например спам звонки)
      5. Юридически:
        1. между Циан и Смартис заключен НДА в рамках, которого есть у ЦИАН есть юридическая ответственность 

    2. Если клиент устно согласовывает - проговариваем

      Устно проговариваем на встрече закрепить договоренность

      для соблюдения нашего регламента я направлю по итогам встречи письмо с запросом согласования включения интеграции, просьба ответить согласовано

    3. Отправляем письмом:

      1. По итогам встречи направляю запрос на согласование включения интеграции Циан Post-view

        Текст письма для согласования клиентом включения интеграции
        По итогам встречи направляю запрос на согласование включения интеграции Циан Post-view для Mobile APP:

- В обмен данными между API Циан и Smartis будут добавляться необратимо зашифрованные телефоны ваших обращений и конверсий (включая спам звонки) без упоминания Вас как застройщика/ЖК, и без упоминания о том что это за обращения/конверсии

Просьба ответным письмом согласовать подключение
      2. После получения подтверждения ответным письмом - фиксируем скрин, в котором будет видно кто/когда подтвердил - и складываем в документацию по клиенту в конфлюенсе
  2. Когда согласие получено и зафиксировано - в разделе Интеграции подключаем интеграцию "ЦИАН Post-view MobileApp" (не требует дополнительных настроек).
  3. По всем обращениям и CRM данным начиная со дня подключения интеграции начнут матчиться данные.

Функционал Циан Post-view в личном кабинете Smartis

  1. в CJM клиента добавляется информация обо всех просмотрах медийки застройщика на ресурсах ЦИАН
  2. События post-view от циан доступны в показателе "Просмотры медийных размещений"
    1. если в личном кабинете используется несколько интеграций - для фильтрации данных от циан используйте фильтр "Медийка: Сервис Post-view" = Циан Web/Mobile
  3. Их можно группировать и фильтровать по:
    1. время (день/неделя/месяц/квартал/год/день недели/час)
    2. источники (канал/площадка/кампания/категория кампаний/объявление/Ключевые слова/форматы медийки)
    3. utm-метки
    4. Устройство
    5. Медийка: Место показа баннера
    6. Медийка: Тип события
    7. ВНИМАНИЕ!
      По "Объекты" эти данные не фильтруются и не привязываются. В планах такой функционал есть, на конец 2024года.
  4. Для построения сквозных отчетов с учетом post-view данных от Циан
    1. интеграция должна быть включена и не удалена
    2. используется показатели обрашений "(по моделям атрибуции)" или CRM показатели
    3. используется модель атрибуции "с post-view"

Ньюансы

  1. Спецформаты циану необходимо размечать отдельно. 
    1. Застройщик/РА самостоятельно обращается к менеджеру ЦИАН. Для рекламодателя это стоит денег, мы не участвуем в этом вопросе, только "подсвечиваем", что со спец форматами надо вести по нетиповому проекту с циан
    2. На нашей стороне распознавание данных со спец форматах опирается на "наименование баннера", которое циан обычно делает в виде "7877/4314887 cr:FSK1025" где 7877 - тот самый идентификатор застройщика на Циан

  2. Циан в качестве просмотров нам передает (далее цитата от Циан): 

Мы передаем видимые показы. То есть в момент доскрола до баннера уходит событие. Другие метрики по воронке мы отдавать не можем, так как мы не используем стандартные шаблоны адфокса, а кастомные не трекаются

3. Циан присылает менее 0,1% "в будущее", на дни, месяцы и даже иногда на годы в будущее. Если клиенты об этом спрашивают - отвечайте "так присылает циан, поправить не можем, но это несущественная погрешность."

4. Почему у меня есть события даже до дня подключения интеграции?

    1. Ранее для тех клиентов у которых в личных кабинетах Smartis уже заведены сайты - мы даже без включенной интеграции сохраняем события post-view. И только с момента включения интеграций начинаем их применять при матчинге конверсий
    2. Интеграция с Mobile App может получить события по хешу телефона за 180 дней до конверсии, такие события мы так же сохраним и они будут в днях до подключения интеграции

Достоверность по Cian 

  1. у РА/рекламодателя на стороне Cian есть рекламный кабинет (adfox), с которым иногда пользователи могут сверяться
  2. На текущий момент сходимость с этим рекламным кабинетам ~80%
  3. При критичных запросах на отклонения:
    1. Если речь о недавних днях (~последние 14 дней) - Вспоминаем не было ли сообщений от мониторинг-процесса (в общий чат разработчики пишут о критичных моментах которые должны знать менеджеры) 
    2. Если п.а не помог:
      1. у нас должны быть зарегистрированы все сайты, на которые ведется медийная реклама в циан (куда ведут баннеры)
      2. Запрашиваем у РА список ссылок на которые ведут баннеры - проверяем что все сайты у нас зарегистрированы, а если в ссылках встретится страницы на cian - вспоминаем информацию из раздела "как подключить" + договариваемся с РА настраивать заранее такие страницы
      3. Если найдены незарегистрированные у нас сайты - регистрируем и создаем тикет "Циан / Обновить client_id по новым зарегистрированным сайтам"
  4. Отдельные строки по достоверности Cian в разделе "Мониторинг системы" - не актуальны, и будут удалены в ближайших релизах. 
    1. Разработка мониторит поток данных самостоятельно сразу по всем клиентам, если возникают локальные проблемы - устраняем в течение 1-2 дней, если проблемы большего масштаба - сообщаем в общем чате
