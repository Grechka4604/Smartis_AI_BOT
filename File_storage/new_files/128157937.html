Цель: регулярный мониторинг показателей клиентов через дашборды. 

Участники процесса: аккаунт-менеджер, руководитель отдела.

Инициатор: руководитель отдела

Инфо поле: дашборды/https://metabase.uis.st/dashboard/2 

KPI процесса: 

KPI 1 - 

Как контролируем: выборочный контроль дашбордов и созданных задач в Jira; 

Результат: результатом выполнения задачи будет ежедневный анализ дашбордов по итогам которого будут формироваться рекомендации для клиентов 

Владелец процесса: руководитель отдела

Статус процесса: работает.

 

Краткое содержание процесса.

  1. Ежедневно аккаунт-менеджер CoMagic Premium c 9:00 до 09:15 проверяет дашборды по своим клиентам, по трём показателям: 
    1.1. Количество пропущенных вызовов; 
    1.2. Процент охвата; 
    1.3. Ошибки в отчётах;  
  2. Исключение: 
    2.1. Если сотрудник находится в отпуске или на больничном, сотрудник его замещающий проверяет не только своих клиентов, но и клиентов своего коллеги, далее действует по шагам №3 и №4;
  3. На основании полученных данных при необходимости проверяет информацию в личном кабинете; 
  4. Ставит на себя задачи в Jira. Задача должна содержать краткое описание проблемы и что необходимо выяснить/согласовать с клиентом;
