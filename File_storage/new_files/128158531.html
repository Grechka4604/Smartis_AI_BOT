Предпосылки

  1. Необходимость получать данные о звонке, и при необходимости дополнительно их выгружать с помощью специального id;

  2. Необходимость объединять данные из различных систем коммуникаций

  3. Недополучение данных из нашей системы.
    Реализация подобного решения у колтача и колкипера.

Программа клиента работает через id обращения. 

В процессе нужно исключить любые случаи, когда мы не передаем не полную информацию по обращению. 

У клиента платформа, по потокам E1. Нам необходимо это передать в сипе, и затем отправить клиенту.  

Цели

Получение данных о звонке

Необходимо сделать так, чтобы в SIP передавался id сессии звонка, такой же, как в hook или в API. 

Критерии успешности/Требования

Список ожидания от решения (бизнес требования, технические требования)

  1. Получение данных о звонках по sip-trunk:
    -В целом необходимо передавать:
    номер А(АТС), номер В (номер абонента), call session id, время звонка, продолжительность звонка
    Это возможно реализовать только с помощью доработки на ВАТС, или только для Автомир. Для остальных клиентов это трогать нельзя.
  2. Важно что бы в звонке были номер АТС, номер абонента, call ID, время звонка, продолжительность.
