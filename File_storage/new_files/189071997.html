Что необходимо клиенту (запрос)? 

Обычно клиент приходит с таким запросом: “Хочу смотреть показатель в срезе по дням/неделям и по источникам, но не удобно смотреть в двухсоставной группировке, очень большая легенда и визуально выглядит не очень”

Что клиенту необходимо: 

  • Фильтровать дашборд по конкретному источнику 

  • Альтернатива двусоставной группировке  

  • Смотреть эффективность по одной площадке в дашборде  

 

Зачем это нужно клиенту, в каком процессе это участвует?  

  • Удобная оценка эффективности каналов/площадок 

  • Оперативное управление источниками 

  • Детализация данных 

  • Удобная визуализация отчетов по конкретному источнику 

 

Техническое решение:  

  1. Если клиенту нужна многосоставная группировка (канал/неделя) 

Чтобы это не выглядело так, в формате 2 графиков для сравнения

Затем в следующий виджет выводим эти же показатели, но в разрезе по каналам: 



Далее, чтобы смотреть первый виджет по каналам, необходимо просто во втором отчете нажать на название канала.

 

При этих действиях выйдет значок фильтра, указывая, по какому источнику на данный момент фильтруется весь дашборд.

 
Описание готового решения (презентация клиенту):

Цель решения:

Предоставить клиенту возможность анализировать ключевые CRM-показатели в двух разрезах: по неделям и по каналам, с удобной интерактивной фильтрацией для детализации данных. 

Как это работает: 

1) Два виджета с группировкой данных: 

а. Первый виджет: отображает CRM -показатели (например, количество лидов, продажи, конверсии) для двух и более объектов (например, объект А и объект Б) с группировкой по неделям. Это позволяет сравнивать динамику показателей по неделям для каждого объекта. 

b. Второй виджет (таблица): отображает те же CRM-показатели, но в разрезе каналов (например, социальные сети, контекстная реклама, email). Это помогает понять, какие каналы наиболее эффективны для каждого объекта. 

  2) Интерактивная фильтрация: 

a. Клиент может выбрать конкретный канал во втором виджете, нажав на его название. 

b. При выборе канала автоматически активируется фильтр, который применяется ко всему дашборду, включая первый виджет и.. 

c. В интерфейсе появится значок фильтра, указывающий, по какому каналу в данный момент отфильтрованы данные.

Преимущества решения: 

  • Удобство анализа: Клиент может быстро переключаться между общим анализом по неделям и детализацией по каналам. 

  • Интерактивность: Фильтрация в реальном времени позволяет глубже изучать данные без необходимости создания дополнительных отчетов. 

  • Гибкость: Возможность сравнивать показатели для двух объектов в одном дашборде. 

  • Наглядность: Значок фильтра помогает отслеживать, какие данные отображаются в текущий момент.

Пример использования
Клиент может: 

     —Сначала оценить общую динамику лидов по неделям для двух объектов. 

     —Затем детализировать данные, выбрав конкретный канал (например, контекстную рекламу), чтобы увидеть, как этот канал влияет на показатели по неделям. 

     —Быстро переключаться между каналами, чтобы сравнить их эффективность.

Итог: 

Решение позволяет клиенту эффективно анализировать данные в двух ключевых разрезах (неделя и канал), фильтровать дашборд по конкретному источнику, а также клиента есть альтернатива двусоставной группировке.
