Предпосылки (что послужило причиной для проекта, какую проблему решаем)

Основной показатель, за который отвечает маркетинг - генерация новых лидов, которых взяли в работу ( В CRM создаются, как Интересы в статусе "в работе").
Лиды привлекаются по разным направлениям - департаментам компании:

  • Аренда,
  • Продажа,
  • Загородная недвижимость,
  • Зарубежная

По каждому департаменту есть целевые значения стоимости потенциального клиента. 

Сейчас получается, что слишком высокая стоимость через цифровые каналы и она как-то непредсказуемо пляшет. 


Цель (ожидаемый результат и критерии успеха)

Требования

Создать автоматизированный отчет, показывающий конверсию источников рекламы в этапы воронки продаж. 


1) Отчет позволяет принимать решение по стоимости привлечения клиента до конкретного этапа. Конверсия и стоимость лида. 

2) Разрезы статистики:
По объектам
По департаментам (продажа, аренда, загородная, зарубежная)

3) Отчет собирается в двух представлениях
Относительно времени привлечение лида
Относительно времени закрытия сделки

4) Динамика ключевых показателей (стоимость лида на разных этапах в разрезе источника и департамента)

5) Нужны пайчарты по долям каналов привлечения. Непонятно, зачем?

6) Нужна возможность сравнить ключевые показатели с предыдущими периодами. 

Нужно следить за тем, что изменилось, рынок или проблемы с процессами в компании. 

7) В отчетах должны быть плановые и фактические расходы по рекламным кампаниям.

Как работают сейчас

Настроена передача данных между CoMagic и CRM-MS Dynamics. В карточку уже передаются данные о рекламных кампаниях. 

Работает не напрямую. С CRM интегрирована телефония на Asterisk. 

СRM запрашивает у нас данные о источнике по времени и номеру телефона. 

Раз в сутки делает повторную проверку. 


Источник могут понять только по 30% сделок, остальное приходится собирать руками и на это тратит время лично директор по маркетингу. 

На сбор актуального отчета требуется несколько дней, клиент руками ищет соответствие между сделкой в CRM и источником в CoMagic.

Сколько человек работает?

CRM MS Dynamics

АТС - Asterisk

  1. Телефония интегрирована с CRM и при звонке автоматом создается сущность Звонок
    1. Для всех остальных типов обращений карточки создаются руками - ВЫЯСНИТЬ ИХ ДОЛЮ
    2. Звонку проставляется метка Целевой/Нецелевой 
    3. В Звонок добавляются источники из CoMagic - рекламная кампания
  2. Если Звонок Целевой, то создается сущность Интерес
    1. Есть дополнительные статусы у Интереса, которые проставляются автоматически, в зависимости от работы внутри карточки Интереса. 
    2. Основной статус для маркетинга - в работе. 
  3. Если Интерес продвигается дальше по воронке, то создается отдельная сущность - Показ
  4. Есть отдельная сущность - Переговоры
  5. Финальный этап - Сделка

Интегратор Никфорт готов добавить к Звонку любые дополнительные признаки, которые нужны, чтобы мы потом их могли выгружать и класть к себе в отчеты.
