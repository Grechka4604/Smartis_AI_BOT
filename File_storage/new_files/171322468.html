Основная информация 

АМО CRM - система CRM для контроля продаж. В Smartis с этой CRM нативная интеграция (работает по API и подключается без участия разработчиков). Из АМО мы получаем всю информацию о клиентах - сделки, их статусы, контактные данные и другие параметры, которые для каждого покупателя заполняются в CRM.


Видео  с обзором сервиса и его подключением

Данные из АМО CRM не содержат информацию о том, с какого источника (рекламного, органического) "пришел" клиент. На основе информации о рекламном источнике предыдущих касаний клиента с рекламой (просмотры медийки, сессии, обращения) строится аналитика по CRM данным в разрезе каналов, площадок, кампаний и т.д.
То есть, в Smartis работает атрибуция CRM данных (в зависимости от выбранной Модели Атрибуции)

В Амо есть 2 сущности контакты и сделки.


Если клиент использует Списки AMO CRM для загрузки и хранения данных из других систем, мы также можем забирать эти данные. Такая доработка сделана для AFI (справка тут: ). Данные по спискам загружаются в сущность Элементы списков AmoCRM.

Подключение интеграции

Перед подключением нужно запросить у клиента доступы (логин и пароль) в АМО. Либо подключать может сам клиент, используя свои доступы. Лучше через админский подключать. 
ВАЖНО! Если АМО будет подключен не через админский доступ, то ФИО ответственных менеджеров получить будет невозможно - будут передаваться только их ID.

  1. Для подключения интеграции в кабинете Smartis необходимо пройти в раздел Администрирование/Интеграции и среди доступных для подключения выбрать АМО.

  2.  Нажать "Подключить". Откроется вкладка Амо CRM, где нужно будет либо авторизоваться под логином, либо (если уже авторизованы) пройти следующие шаги: выбрать нужный аккаунт, нажать "Разрешить"

  3. После этого интеграция подключится и появится в разделе "Доступные интеграции".
  4. Для того, чтобы данные из Амо воспроизводились в Smartis, после подключения интеграции автоматически создастся коннектор Smartis.СRM
    Справка по коннекотору: 

Настройка интеграции после подключения

После подключения необходимо зайти в раздел интеграции, в настройки интеграции АМО и выбрать дополнительные условия для настройки, Поле объекта и проекта для продукта и Если сделка перемещалась более одного раза на этап:* и выбрать первый или последний этап. 

Привязка интеграции к сайту

Для того, чтобы данные по Амо отобразились в статистике проекта в Smartis, необходимо привязать интеграцию к нужному проекту. Данные мы всегда привязываем к базовой единице - сайту.

  1. Для привязки необходимо пройти в раздел "Проекты и сайты", выбрать нужный сайт и зайти в его настройки. 
  2. В доступных для привязки интеграциях необходимо выбрать интеграцию АмоCRM и привязать ее к сайту. 

  3. После того, как привязали интеграцию к сайту, никакие дополнительные действия не нужны - интеграция отобразится в разделе "Привязанные к сайту интеграции"

  4. Из самой интеграции Амо привязки по разным условиям для отдельных сайтов настроить невозможно - эти настройки выполняются в коннекторе Smartis.СRM, который также нужно привязать к сайту (выбрать его из раздела доступных интеграций и нажать "Привязать к сайту"). О том, как настроить привязки к сайт с помощью коннектора - читать тут: 

Для корректной привязки к сайтам важно знать бизнес-логику клиента - как именно он хочет смотреть данные в Smartis и как он их делит на объекты. Некоторые клиенты выделяют как отдельный объект нежилые типы недвижимости - паркинг, кладовки, коммерческие помещения. Также клиенты с Амо могут хранить данные по 1 ЖК в разных воронках продаж и привязывать к сайту нужно будет только данные из определенных воронок. В каких доп полях хранится нужная информация для привязки лучше уточнить заранее.


Отчеты по данным АМО

Показатели

При подключении интеграции создаются системные показатели (автоматиечски). Они будут отображаться в списке показателей в конструкторе отчетов, когда выбран проект, объект или сайт, к которому привязана интеграция. 
Показатели будут отображаться в отдельной категории - AmoCRM


Кол-во сделок - отображает кол-во сущности "Сделки" в Амо. Сделка в Амо не равно "Договор купли-продажи", это просто все созданные карточки клиентов, которые могут находиться на разных этапах воронки продаж. Это основной показатель Амо, на основе которого строятся показатели для отчетов в Smartis.
Кол-во сделок с лидов за период - отображает кол-во сделок, которые были созданы из лидов [для отчетов не используется]
Сумма сделок - отображает суммы сделок (то есть данные из поля Сумма из Амо)
Сумма сделок с лидов за период - отображает суммы сделок (то есть данные из поля Сумма из Амо), по сделкам, которые были созданы из лидов [для отчетов не используется]
Кол-во сделок с лидов за период [Кол-во уникальных] - отображает кол-во сделок с уникальными id, которые были созданы из лидов [для отчетов не используется]
Кол-во сделок [Кол-во уникальных] - отображает кол-во сущности "Сделки" в Амо с уникальными id. 
Цикл сделки AmoCRM - отображает цикл сделки (в днях), расчитанный в Амо CRM. [для отчетов не используется]
Цикл сделки AmoCRM с лидов - отображает цикл сделки (в днях), расчитанный в Амо CRM. По сделкам, которые были созданы из лидов [для отчетов не используется]
Задачи AmoCRM - отображает задачи для менеджеров по сделкам, которые создаются в Амо.

Также для каждого клиента создадутся автоматически показатели по воронке продаж. Эти показатели будут находиться в отдельных категориях, которые называются как воронки в Амо. Для каждой воронки будут созданы метрики Кол-во седлок, Кол-во сделок с лидов, Сумма сделок и Сумма сделок с лидов для каждого этапа воронки (у разных клиентов могут быть разные, в каждой воронке также этапы могут быть разные).
На скрине - пример созданных автоматически категорий показателей под каждую воронку Амо у клиента.

Детализация данных Амо CRM

Для данных Амо CRM доступна детализация (исходные данные). Они такие же, как у других сервисов CRM.

Основные свойства, передаваемые в даных Амо:

  • Данные о свойствах CRM-элементов: внешний ID, дата создания, тип CRM-элемента, ссылка на сделку в Амо
  • Данные о клиенте: номер телефона, email, ФИО, статус на сейчас.
  • Данные о рекламном источнике обращения: канал, площадка, кампания (эти столбцы заполняются в зависимости от выбранной модели атрибуции. есл
  • Данные из доп полей: суммы сделок, объект (ЖК), ответственный менеджер, этап воронки, площадь, комнатность и другие параметры.

Общая справка по детализации данных CRM:
6. Исходные данные (Детализация, Сырые данные)

  • Для Амо CRM в детализации доступны также наборы данных (столбцы) всем доп полям, которые передает нам клиент из Амо. Эти столбцы можно просматривать и при необходимости фильтровать. Набор этих данных зависит от набора доп полей в Амо у клиента.

В Smartis можно переключиться в Амо для просмотра отдельных сделок. Чтобы это сделать, в детализации данных выберете столбец Ссылки. По клику на ссылку откроется Амо (у пользователя с доступом к данному ЛК сразу откроется нужная сделка)


  • Для Амо и для Битрикса мы отображаем в таблице детализации сырых данных и в CJM "Статус на сейчас". Это статус на момент последней синхронизации этой сделки (в амо обычно это каждые 15-30 минут)

Это нужно чтобы понимать куда продвинулся лид при изучении его истории, и удобно для отладки показателей с наложенными фильтрами.

Цвет подписи статуса - берется из самого Амо (как там настроено, так мы и отображаем)

Достоверность AmoCRM


отображается в разделе Мониторинг системы. О его работе можно почитать тут: 9. Администрирование: Мониторинг системы

По данному сервису мы получаем информацию по таким сущностям:

  • контакты (контактные данные клиентов)
  • сделки 
  • элементы списков АМО (если используется). На данный момент есть только у клиента 

    Достоверность ретро данных по АМО считается автоматически за последние 3 дня ежедневно.

    В информации о достоверности показано, сколько данных у нас и сколько данных в API Амо, а также дата последнего обновления.


Для АМО нормальной считается ситуация, если контактов "у нас" больше чем "всего". Если контактов больше, то достоверность будет все равно 100% по этой сущности. Однако если у нас больше сделок - % достоверности снизится. 

Также может быть такое, что сделок "у нас" больше чем "всего". Это связано с тем, что у амо нет контроля удаленных. Эта ситуация считается нормальной, если доля таких расхождений не более 3%.
