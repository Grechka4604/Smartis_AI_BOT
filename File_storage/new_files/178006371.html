Двухфакторная аутентификаци для доступа к облачному сервису

При авторизации юзера по клиенту в ЛК #1904 показывается доп окошко, после успешного ввода логина и пароля, где нужно ввести код из почты, отправленный юзеру на почту из смартис. При не успешном вводе кода пользователь не может авторизоваться в ЛК.

Также есть кнопка "отправить код повторно"-  при нажатии на нее приходит новый код, если код не пришел с первого раза. Интервал ожидания 30 секунд для запроса нового. кода


Хранить код можно в колонке loginControlSum.


Проверка требований к паролям

В связи с требоавниями КБ клиента, добавили настройки для проверки паролей.

для окна создания пароля должны показываться требования к формату пароля:

- пароль не должен повторять предыдущие 4 пароля для данной учетной записи.

- обязательная длина пароля должна быть не менее 12 символов;

- пароль должен содержать в себе символы как минимум трех категорий из четырех: буквы нижнего регистра (от a до z), буквы верхнего регистра (от A до Z), цифры (от 0 до 9) и спецсимволы (например: $, #, %);
