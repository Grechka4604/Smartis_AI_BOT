Общая информация

Яндекс Директ - рекламный сервис, с помощью которого клиенты размещают контекстную рекламу и рекламу других типов (РСЯ, например https://yandex.ru/support/partner/). У нас нативная интеграция с Яндекс Директ, которая позволяет получать и отображать в Smartis данные о кампаниях, объявлениях и ключевых словах из рекламного кабинета.


Если у вашего клиента есть данные в старом кабинете Директа, к которому нет доступа, но при этом данные клиенту нужно получать в смартис в ретро периодах, то это можно сделать через импорт данных из старого кабинета.
Что для этого нужно:

  • получить от клиенты выгрузки со всеми необходимыми показателями (расход, клики, показы)
  • создать импортируемые показатели по кликам и показам
  • загрузить данные по Директу: 
    • клики - в импортируемый показатель Клики
    • показы - в импортируемый показатель Показы
    • расход - в импортируемый показатель Бюджет медиапланов (факт) в площадку Яндекс Директ
  • для того, чтобы старые данные отображались в отчетах, необходимо создать показатели по кликам, расходу и по показам, который включает импортированные вручную данные + данные которые поступают из Директа по апи интеграции. Чтобы это сделать, создается новый показатель который строится по формуле Клики (импорт) + Клики Директа (с показами и расходами аналогично)
  • эти показатели нужно добавить во все отчеты, где нужно отслеживать данные по Директу.



Какие данные по Яндекс Директ доступны в Smartis:

  • показы
  • клики
  • расход с НДС

Обратите внимание, что на данный момент в Smartis нет интеграции с Промо-страницами.

У Яндекс Директ есть еще сущность "ad_group" (группы объявлений) и группы кампаний. На данный момент мы в Smartis никак их не отображаем (но данные с информацией по этим сущностям получаем).

Расходы по Яндекс Директ, которые мы получаем из интеграции, всегда загружаются в системную площадку Яндекс Директ (id 11), канал = Контекстная реклама.


Данные по обращениям и сессиям, которые поступили с рекламы в Яндекс Директ также привязываются к этой площадке по умолчанию. 

Если для вашего клиента нужно учитывать сессии и обращения по медийной рекламе Яндекс Директ (реклама в РСЯ) в отдельной площадке - можно создать площадку Яндекс РСЯ и задать для привязки условия. Однако в таком случае расходы и клики из рекламного кабинета все равно будут попадать в площадку Яндекс Директ (id 11), канал = Контекстная реклама.

Подключение интеграции

Структура кабинетов Яндекс Директ у клиентов может быть разная. Вот возможные варианты:

  • к одному логину в Яндекс Директ относится несколько рекламных кабинетов
    Логин: ИвановИван@yandex.ru, кабинеты доступные логину: ЖК 1 (рекламные кампании в кабинете по ЖК 1), ЖК 2 (рекламные кампании в кабинете по ЖК 2),
    ЖК 3 (рекламные кампании в кабинете по ЖК 3)

  • к одному логину в Яндекс Директ относится 1 логин, и в нем ведутся рекламные кампании по 1 объекту/сайту
    Логин: ИвановИван@yandex.ru, кабинеты доступные логину: ЖК 1, рекламные кампании в кабинете по ЖК 1

  • к одному логину в Яндекс Директ относится 1 логин, и в нем ведутся рекламные кампании по нескольким объектам/сайтам
    Логин: ИвановИван@yandex.ru, кабинеты доступные логину: "Общий", рекламные кампании в кабинете по ЖК 1, ЖК 2, ЖК 3

В любом из этих случаев мы для подключения интеграции используем логин с доступами в кабинет/кабинеты, и далее привязываем к сайту либо каждый логин из общего кабинета отдельно, либо единственный кабинет с логином к 1 сайту, либо если реклама в кабинете по нескольким объектам - 1 логин ко всем нужным сайтам + прописываем условия привязки по названиям кампаний.

  1. Получить у клиента доступ в кабинет Яндекс директ, либо если подключает сам клиент не нужно. Для подключения интеграции со Smartis достаточно прав на чтение. Документация Яндекса о правах доступа:https://yandex.ru/dev/direct/doc/dg/objects/roles.html

  2. В разделе Интеграции в Smartis выбрать интеграцию с Яндекс Директ в разделе "Доступные для подключения" и нажать "подключить"


  3. Откроется страница авторизации - нужно ввести свои данные и пароль к кабинету Яндекс Директ


  4. После ввода логина и пароля к кабинету Яндекс Директ, отобразится окно с информацией о том, к каким данным Smartis получит доступ (на скрине пример с полными доступами, при доступе на чтение информация может отличаться). На этом этапе нажать "войти" (желтая кнопка). Если доступы верные, появится сообщение "Авторизация прошла успешно" и окно закроется.



  5. После подключения интеграции, она отобразится в разделе "Подключенные интеграции" с названием = логин, через который вы подключились.


Когда интеграция подключена, по ней начинается автоматически обновление данных. По умолчанию мы забираем данные из интеграции с начала предыдущего месяца. Если нужно обновить больший период, воспользуйтесь ручным обновлением данных (справка тут  )

Если интеграция отключилась и по ней отображается ошибка: Ошибка авторизации - это значит, что владелец кабинета сменил пароль или логин для доступа к данным кабинета. Если интеграцию подключал менеджер Smartis, нужно получить актуальные данные для авторизации от клиента. Если подключал клиент - нужно чтобы шаги ниже выполнил клиент.


нужно кликнуть иконку "Обновить авторизацию" и далее пройти путь подключения интеграции как описано выше пункт 1-5 (но привязывать заново к сайтам ее не нужно).

Если с момента отключения интеграции прошло менее 2 месяцев, то данные по интеграции обновятся автоматически, с первого дня прошлого месяца. Если больше - воспользуйтесь обновлением данных вручную через раздел Мониторинг системы.

Привязка интеграции к сайту

Данные по интеграции нельзя просматривать в отчете, пока интеграция не привязана к сайтам в личном кабинете. Интеграцию нужно привязать ко всем сайтам, по которым ведется реклама из данного рекламного кабинета.
Например, у вас рекламный кабинет Яндекс Директа, в котором ведут рекламу по ЖК #1 - значит, ее нужно привязать к сайту этого ЖК.
Если по 1 ЖК у вас несколько сайтов, но все они входят в 1 объект в Smartis, интеграцию достаточно привязать только к 1 из этих сайтов (любому).

  1. Зайти в раздел Администрирование → Проекты и сайты → Сайты и выбрать нужный сайт



  2. По клику на название сайта откроется меню для привязки интеграций. В разделе Интеграции доступные для привязки найдите нужную и нажмите "Привязать с логином" 
  3. В выпадающем списке логинов выберите нужный и кликните на него (выбрать модно только 1 логин единовременно). После выбора логина страница обновится и привязанная интеграция отобразится в разделе Привязанные к сайту интеграции


  4. Для каждой интеграции можно настроить условия привязки - например, если вам нужно получать из интеграции для выбранного сайта только определенные кампании, вы можете прописать такие условия. Для этого кликните на иконку шестеренки на плашке интеграции. Здесь можно указать условия для привязки к сайту определенных рекламных кампаний в виде регулярного выражения. В примере мы указали, что кампании у которых в названии есть "бренд" или "brand" к сайту привязывать не нужно.
    Такая настройка используется, если в рекламном кабинете рекламные кампании по нескольким объектам ЖК или как в нашем случае - есть общие рекламные кампании по всему бренду клиента и их нужно просматривать в Smartis в отдельном объекте.



  5. Если вам нужно отвязать интеграцию от сайта, кликните на иконку как на скриншоте


Отчеты по данным Яндекс Директ

Для клиентов мы обычно строим вот такие отчеты по контекстной рекламе.

Анализ эффективности контекстной рекламы

Дашборд Анализ эффективности контекстной рекламы необходим для:

  1. Более детального анализа контекстной рекламы от показов до суммы сделок;

  2. Анализа эффективности данных в разрезе кампаний/объявлений/ключевых слов. 

  3. Отслеживания динамики данных по дням;

  4. Анализа эффективности данных в разрезе категорий кампаний (бренд/не бренд, поиск/РСЯ, Гео и т.д.)



Одноцифровые виджеты состоят из:

  • Расходов ЯД;

  • Показов ЯД;

  • Кликов ЯД.

Динамика данных состоит из:

  • Расходов ЯД;

  • Валовых обращений;

  • Целевых обращений.



Воронка в разрезе поиска/РСЯ, Бренд/Не бренд (и другие категории кампаний по желанию) состоит из показателей воронки:

  • Расхода ЯД;

  • Показов ЯД;

  • Кликов ЯД;

  • Валовые обращения;

  • CPL: Валовых обращений;

  • CRM: Целевые обращения;

  • CPL: Целевого обращения;

  • CRM: Встречи;

  • CRM: Брони;

  • CRM: Успешные сделки;

  • CRM: Сумма сделок;

  • %ДРР.


Данные отчеты так же строим в разрезе кампаний и объявлений. 

В каждом виджете можно с помощью проваливания детализировать данные до кампаний/объявлений/ключевых слов. 

Подробнее про детализацию и отчеты по объявлениям и ключевым словам по Яндекс Директ:

Показатели Яндекс Директ

После привязки интеграции к сайту, при выборе проекта или объекта, в которых этот сайт состоит, или сайта отдельно в конструкторе отчетов, у вас в списке показателей появятся системные показатели Яндекс Директ. Они будут находится в категории "Я.Директ" (создается тоже автоматически). Показатели системные, их нельзя изменить. Но на их основе можно создать пользовательские показатели с помощью формул (про них справка тут  )

По данным можно построить отчет в группировках, которые доступны при выборе показателей. Площадка, по которой будут строиться данные - всегда будет только Яндекс Директ, даже если у вас выбран канал "Весь трафик". Если будет выбран канал не Весь трафик и не Контекстная реклама, то в отчете отобразится 0.

Данные Яндекс Директ невозможно посмотреть в виде детализированных данных, поэтому для просмотра отчетов по данным Яндекс Директ доступны все виды визуализации кроме "Исходные данные".

Расходы из Яндекс Директ поступают сразу с НДС. Если вам нужно учитывать расходы без НДС, то нужно:

  • создать новый показатель "Яндекс Директ расход без НДС"
  • тип данных = формула
  • в формуле прописать такое значение: системный показатель [Я.Директ расход с НДС] *100 / 120

Фильтры для Яндекс Директ

кроме основных системных фильтров, для Яндекс Директ доступны особые:

  • Яндекс Директ: Блок показа объявления - позволяет отфильтровать данные по спецразмещениям
  • Яндекс Директ: Возрастная группа пользователя - позволяет отфильтровать данные по возрастной группе 
  • Яндекс Директ: Пол пользователя - фильтрует по полу пользователя

  • Яндекс Директ: Тип площадки - фильтрация по типу площадки (Поиск или РСЯ)

  • Яндекс Директ: идентификатор кампании - позволяет фильтровать данные по campaign id (cid) - этот параметр передается из кабинета ЯД, обычно указывается в utm метках. 
  • Яндекс Директ: идентификатор объявления - позволяет фильтровать данные по id объявления из Яндекс Директ. 

Эти фильтры применяются только к показателям с данными Яндекс Директ. Если в отчете выбраны и другие показатели, к ним фильтр не применится.


Детализация источников по Яндекс Директ (проваливание)

по данным Яндекс Директ доступны такие уровни детализации:

  • Категории кампаний (пользовательские группы, настраиваются тут  )
  • Кампании - открывает срез по кампаниям
  • Объявления - открывает срез по объявлениям. Отчет по объявлениям можно смотреть не только по данным Яндекс Директ, но и по данным сервисов обращений и CRM. То есть можно посмотреть статистику как отработало и сколько обращений, сделок и других конверсий поступило с каждого объявления Яндекс Директ.

  • Ключевые слова/форматы медийки - открывает срез по ключевым словам (для поисковых кампаний) или форматам медийки (для кампаний в РСЯ). Отчет по ключевым словам можно смотреть не только по данным Яндекс Директ, но и по данным сервисов обращений и CRM. То есть можно посмотреть статистику как отработало и сколько обращений, сделок и других конверсий поступило с каждого ключевого слова Яндекс Директ.


    Таким образом, данные по Яндекс Директ можно просматривать в срезе до ключевых слов.

Особенности получения названия рекламных Смарт-Кампании из Я.Директа:

В связи с особенностью API Яндекса, по Смарт-кампаниям мы не получаем названия кампаний из рекламного кабинета. В связи с этим мы их переименовываем в Smartis.

В отчетах данные кампании отображаются с таким же неймингом, как в кабинете ЯД.


Как это реализовано технически:
Берем ID кампаний, по которым не смогли получить названия. Запрашиваем из ЯД отчет по кампаниям с фильтром по ID.

В отчете по кампаниям добавляем

FieldNames": [

+ "CampaignName",

]

и получаем названия. 

Группа "Динамические объявления" (ранее "Не указано") по Яндекс Директ в группировке "Объявления" включает в себя данные по объявлениям, по которым ЯД не возвращает наименование и идентификатор объявления. Как правило, туда попадают объявления, созданные в мастере кампаний или РК МКБ.
Сам ЯД время от времени меняет типы кампаний, в которых генерируются динамические объявления. Однако суть остается прежней - эти объявления генерируются автоматически, и у них нет ни названия ни идентификатора.
