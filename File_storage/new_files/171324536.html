Основная информация

В Smartis есть 2 интеграции с ВК - VK Ads (новый кабинет) и VK Business (старая версия кабинета). У некоторых клиентов подключены оба. С апреля 2023 года VK Ads стал основным кабинетом и клиенты перенесли рекламу в него.

VK - рекламный сервис, с помощью которого клиенты размещают рекламу в социальной сети ВК.

При настройке таргет рекламы в ВК можно выбрать один из двух типов объявлений — тизеры или нативная реклама. Тизеры — это объявления с изображением и текстом. Нативная реклама — это посты, которые встраиваются в ленту пользователей и сообществ.

У нас нативная интеграция с VK по API, которая позволяет получать и отображать в Smartis данные о расходам, кликах, показах, кампаниях/объявления, лидах из рекламного кабинета.

Лучше не подключать несколько агентских аккаунтов VK, у которых есть доступ к одинаковым логинам/кампаниям, так как потом привязки по логинам переопрделяются. По факту часть привязок станет недоступной в каком-то из подключенных логинов.

Если у вашего клиента есть данные в старом кабинете ВК, к которому нет доступа, но при этом данные клиенту нужно получать в смартис в ретро периодах, то это можно сделать через импорт данных из старого кабинета.
Что для этого нужно:

  • получить от клиенты выгрузки со всеми необходимыми показателями (расход, клики, показы)
  • создать импортируемые показатели по кликам и показам
  • загрузить данные по ВК: 
    • клики - в импортируемый показатель Клики
    • показы - в импортируемый показатель Показы
    • расход - в импортируемый показатель Бюджет медиапланов (факт) в площадку ВК
  • для того, чтобы старые данные отображались в отчетах, необходимо создать показатели по кликам, расходу и по показам, который включает импортированные вручную данные + данные которые поступают из ВК по апи интеграции. Чтобы это сделать, создается новый показатель который строится по формуле Клики (импорт) + Клики ВК (с показами и расходами аналогично)
  • эти показатели нужно добавить во все отчеты, где нужно отслеживать данные по ВК.



Какие данные по VK  доступны в Smartis:

  • показы
  • клики
  • расход (с НДС)
  • лиды


Если ранее у клиента не была подключена интеграция с ВК по лидам и обращения по лидам ВК поступали из колл-трекинга, после подключения ВК лидов будут дубли этих обращений. В таком случае чтобы не было дублей, требуется проверить, чтобы в utm_content обращений из колл-трекинга были указаны идентификаторы объявлений. Если они там есть - то все ВК лиды "сольются" с обращениями колл-треикнга и дублей не будет. Отсутствие в обращениях из колл-трекинга utm_content влияет на то, что при построении отчета по CRM данным в группировке Объявления не будут учитываться CRM сущности, которые атрибутированы к обращениям из колл-трекинга (тк у них не указан utm_content).



Расходы и другие данные по VK, которые мы получаем из интеграции, всегда загружаются в системную площадку vk.com реклама #40, канал = Реклама в соц.сетях

Данные по обращениям и сессиям, которые поступили с рекламы в VK также привязываются к этой площадке по умолчанию. 

Важно! Нельзя удалять эту площадку и переносить данные в другую площадку, даже если клиенты очень сильно просят переименовать площадку. 

Подключение интеграции

Структура кабинетов VK у клиентов может быть разная. Вот возможные варианты:

  • к одному логину в VK относится несколько рекламных кабинетов
    Логин: ИвановИван@mail.ru, кабинеты доступные логину: ЖК 1 (рекламные кампании в кабинете по ЖК 1), ЖК 2 (рекламные кампании в кабинете по ЖК 2),
    ЖК 3 (рекламные кампании в кабинете по ЖК 3)

  • к одному логину в VK относится 1 логин, и в нем ведутся рекламные кампании по 1 объекту/сайту.
    Логин: ИвановИван@mail.ru, кабинеты доступные логину: ЖК 1, рекламные кампании в кабинете по ЖК 1

  • к одному логину в VK относится 1 логин, и в нем ведутся рекламные кампании по нескольким объектам/сайтам
    Логин: ИвановИван@mail.ru, кабинеты доступные логину: "Общий", рекламные кампании в кабинете по ЖК 1, ЖК 2, ЖК 3

В любом из этих случаев мы для подключения интеграции используем логин с доступами в кабинет/кабинеты, и далее привязываем к сайту либо каждый логин из общего кабинета отдельно, либо единственный кабинет с логином к 1 сайту, либо если реклама в кабинете по нескольким объектам - 1 логин ко всем нужным сайтам + прописываем условия привязки по названиям кампаний.

  1. Получить у клиента доступ в кабинет VK, либо если подключает сам клиент не нужно. Для подключения интеграции со Smartis достаточно прав на чтение. 
  2. В разделе Интеграции в Smartis выбрать интеграцию с VK в разделе "Доступные для подключения" и нажать "подключить"

  3. Откроется страница авторизации - нужно ввести свои данные и пароль к кабинету VK
  4. После прохождения авторизации интеграция подключится - появится в разделе "Подключенные интеграции"
  5. Когда интеграция подключена, по ней начинается автоматически обновление данных. По умолчанию мы забираем данные:

    VK Business (с начала года по текущий день)
    VK Ads (синхроним только аккаунты). Сами данные нужно обновлять вручную

    Если нужно обновить больший период, воспользуйтесь ручным обновлением данных (справка тут 9. Администрирование: Мониторинг системы )

    Если интеграция отключилась и по ней отображается ошибка: Ошибка авторизации - это значит, что владелец кабинета сменил пароль или логин для доступа к данным кабинета. Если интеграцию подключал менеджер Smartis, нужно получить актуальные данные для авторизации от клиента. Если подключал клиент - нужно чтобы шаги ниже выполнил клиент.

  6. При подключении подтягиваются сразу все показатели Расходы, Показы, Клики и Лиды, дополнительно для подтягивания лидов в новой интеграции ВК не нужно ничего настраивать. 


Привязка интеграции к сайту

Данные по интеграции нельзя просматривать в отчете, пока интеграция не привязана к сайтам в личном кабинете. Интеграцию нужно привязать ко всем сайтам, по которым ведется реклама из данного рекламного кабинета. Информация о привязке актуальна как для старого, так и для нового кабинета VK.

Например, у вас рекламный кабинет VK, в котором ведут рекламу по ЖК #1 - значит, ее нужно привязать к сайту этого ЖК.
Если по 1 ЖК у вас несколько сайтов, но все они входят в 1 объект в Smartis, интеграцию достаточно привязать только к 1 из этих сайтов (любому).

  1. Зайти в раздел интеграции, в шестеренку и выбрать нужные аккаунты в интеграции:
  2. Зайти в раздел Администрирование → Проекты и сайты → Сайты и выбрать нужный сайт.
  3. По клику на название сайта откроется меню для привязки интеграций. В разделе Интеграции доступные для привязки найдите нужную и нажмите "Привязать к сайту" или "Привязать с логином". Если подключали весь кабинет - будет много логинов в списке. Если подключили 1 логин - то в списке отобразится только он. 
  4. Если у вас кнопка "Привязать с логином", то в выпадающем списке логинов выберите нужный и кликните на него (выбрать модно только 1 логин единовременно).Если "Привязать к сайту"- выбирать логин не нужно. После этого страница обновится и привязанная интеграция отобразится в разделе Привязанные к сайту интеграции.

  5. Для каждой интеграции можно настроить условия привязки - например, если вам нужно получать из интеграции для выбранного сайта только определенные кампании, вы можете прописать такие условия. Для этого кликните на иконку шестеренки на плашке интеграции. Здесь можно указать условия для привязки к сайту определенных рекламных кампаний в виде регулярного выражения. 
    Такая настройка используется, если в рекламном кабинете рекламные кампании по нескольким объектам ЖК или есть общие рекламные кампании по всему бренду клиента и их нужно просматривать в Smartis в отдельном объекте.

    Наименование кампании соответствует регулярному выражению - если в названии рекламной кампании в VK указаны названия ЖК, то можно прописать условие (в виде регулярного выражения) в этом поле (учтите все возможные варианты написания - на русском и английском языке, сокращения и прочее). Не влияет на привязку лидов.

    Наименование кампании не соответствует регулярному выражению - если нужно привязать все рекламные кампании, кроме определенных - заполнить в виде регулярного выражения это поле. Не влияет на привязку лидов.



    Привязка Кликов, показов и расходов по ВК выполняется по названию кампании. То есть чтобы определить какие кампании нужно привязать к сайту для сбора этих данных, нужно построить отчет по данным показателям в группировке "Кампания".



    Наименование формы соответствует регулярному выражению для лидов - для привязки форм (лидов) ВК. Указанные здесь условия применимы ТОЛЬКО для привязки лидов, на привязку кликов/расходов не влияет.

    Привязка лидов выполняется не по "Кампании" а по "РК Трекинга". То есть для того чтобы увидеть данные для привязки форм/лидов, вам нужно построить показатель VK Лиды в группировке "РК Трекинга".



    Привязка лидов ВК также происходит по этой логике - вы указываете выражение в нужном поле и в отчете получаете лиды, которые подходят под заданные условия.



  6. Если вам нужно отвязать интеграцию от сайта, кликните на иконку как на скриншоте

Показатели VK

После привязки интеграции к сайту, при выборе проекта или объекта, в которых этот сайт состоит, или сайта отдельно в конструкторе отчетов, у вас в списке показателей появятся системные показатели VK. Они будут находится в категории "VK Business" (создается тоже автоматически). Показатели системные, их нельзя изменить. Но на их основе можно создать пользовательские показатели с помощью формул (про них справка тут 1. Показатели и их настройка )

Smartis автоматически рассчитывает НДС для VK.

Если вам нужно учитывать расходы без НДС, то нужно:

  • создать новый показатель "VK расход без НДС"
  • тип данных = формула
  • в формуле прописать такое значение: системный показатель [VK расход с НДС] *100 / 120

По данным можно построить отчет в группировках, которые доступны при выборе показателей. Площадка, по которой будут строиться данные - всегда будет только VK даже если у вас выбран канал "Весь трафик". Если будет выбран канал не Весь трафик и не Реклама в соц.сетях, то в отчете отобразится 0.

Данные VK расходам, кликам и показам невозможно посмотреть в виде детализированных данных, поэтому для просмотра отчетов по данным VK доступны все виды визуализации кроме "Исходные данные".

Детализация доступна только по VK Лиды - так как лиды это обращения. Есть специальные формы для обращений, которые можно заполнить в VK. Они такие же, как данные по заявкам у других сервисов обращений - их можно просматривать в конструкторе отчетов или в данных по показателям обращений по клику из отчета или дашборда, а также в базе обращений.

Фильтры для VK

Для данных VK применяются все системные фильтры, которые отображаются при выборе показателей этого сервиса. Особых фильтров для VK нет.

Детализация источников по VK (проваливание)

из отчета по каналам или по площадкам по данным VK доступны такие уровни детализации:

  • Категории кампаний (пользовательские группы, настраиваются тут 9. Администрирование: Конфигуратор )
  • Кампании - открывает срез по кампаниям
  • Объявления - открывает срез по объявлениям. Отчет по объявлениям можно смотреть не только по данным VK, но и по данным сервисов обращений и CRM. То есть можно посмотреть статистику как отработало и сколько обращений, сделок и других конверсий поступило с каждого объявления VK.

  • Из площадки можно провалиться в кампании и категории кампаний. В объявления можно провалиться только кампании. Если нужен отчет по всем кампаниям VK по объявлениям, воспользуйтесь группировкой "Объявления".
