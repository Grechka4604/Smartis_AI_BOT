Этот функционал позволяет, опираясь на название источника, указанное в названии РК из Яндекс Директ, создавать собственные рекламные источники - каналы и площадки (чтобы данные Яндекс Директа попадали в разные рекламные площадки).

Функционал доступен по ссылке в облаке: https://my.smartis.bi/admin/configurator , вкладка "Пользовательские источники".

В отличие от группировок категорий кампаний, данный функционал выполняет проверку по названию канала, площадки и кампании. Если у данных нет названия рекламной кампании, она относится в категорию Остальное.

То есть, кроме поиска по условию в названии РК, можно выполнять поиск по названию канала или площадки. Пример настройки: категория Реклама - в нее входят все каналы рекламы. В категорию Не реклама входят каналы, которые приводят не рекламный трафик.

Принцип работы этого функционала такой же, как у конфигуратора группировок по категориям камапний:

  1. Создать новую группировку. Например, площадки Яндекс Директ
  2. Перевести ее в статус "Активно"
  3. Добавить "категории" - категория это название источника. Например, РСЯ
  4. Выбор операции:
    1. содержит
    2. соответствует регулярному выражению
    3. не соответствует регулярному выражению

  5. Заполнить значение: указываем название канала и рядом условие из названия рекламной камапнии, по которому будем выделять источник. Например, 

    *rsya.*

  6. Добавить другие категории для выделения других источников. Их может столько, сколько нужно выделить источников. Важно, что у разных клиентов могут отличаться "метки" источника, прописанные в названии рекламной кампании. Поэтому перед заполнением уточните, какую метку выбирать для заполнения поля "Значение"

После того, как заполнили таблицу - нажать "Сохранить".


Просмотр данных по пользовательским источникам в отчетах

Чтобы посмотреть данные в настроенной разбивке, в списке группировок отчета нужно выбрать название пользовательской группировки, которое вы указали при настройке.
