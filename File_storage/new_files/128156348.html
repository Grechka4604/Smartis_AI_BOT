Цель: Завершить процесс однозначно для нас и клиента.

Участники процесса: клиент, Руководитель проекта, руководитель консалтинга, ОПР.

KPI процесса: Подтверждение со стороны клиента

Как контролируем: Обмен итогами встречи в митинг репорте

Владелец процесса: руководитель консалтинга

 

Краткое описание процесса

  1. Команда разработки передает решение менеджеру проекта с набором материалов
    1. Подтверждение о завершении разработки и этапа тестирования
    2. Описание решения и его возможностей
  2. Руководитель проекта передает клиенту
    1. Подготовить демонстрацию итогов проекта
    2. Срок, когда клиент успеет проверить работу решения
    3. Время для личной встречи/созвона для демонстрации решения
  3. Руководитель проекта проводит ретроспективу с командой - фиксирует, что можно было сделать лучше в проекте.
