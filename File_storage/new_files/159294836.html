Структура обновления статусов должна быть организована следующим образом: 


1. ПРОЕКТЫ ВНЕДРЕНИЯ


1.1. По проекту внедрения обязательно должен быть указан дедлайн. Дедлайн внутренний или согласованный с клиентом, при этом указывается в первую очередь тот, который нам наиболее критичен. Например, дедлайн согласованный с клиентом проект необходимо сдать до 30.12.2022, но наш внутренний мы понимаем, что нам необходимо сдать до 20.12.2022, т.к. например дальше сотрудник уходит в отпуск. Следовательно в Asana прописывается дедлайн 20.12.2022. 


1.2. При работе с проектами внедрения используются два статуса "по плану" и "отстаёт". 

По плану - означает, что ход выполнения работ указывает на то, что сдача проекта произойдёт до указанного дедлайна или раньше. 

Отстаёт - означает, что мы прогнозируем (ориентируясь на ход работ), что по каким-то причинам проект будет сдан позже установленного дедлайна. 

Если PM указывает статус проекта "отстаёт", то обязательно в статусе необходимо прописать тезисно набор задач/проблем, которые сейчас блокируют для своевременной сдачи проекта. 


2. ПРОЕКТЫ ПОДДЕРЖКИ

1.1. При актуализации статусов проектов поддержки используется два статуса "по плану" и "под угрозой". 

По плану - означает, что выполняются следующие пункты по клиенту: 

  • мы знаем какая цель у нас с клиентом (цель согласована и зафиксирована в карточке клиента); 
  • мы понимаем как выглядит план её достижения; 
  • клиент использует сервис (есть регулярные входы, входящие запросы); 
  • нет ошибок/багов/проблем, которые блокирует использование сервиса в штатном режиме. 

Под угрозой - означает, что какая-то часть задач описанных выше не выполняется: 

  • мы не знаем нашу общую цель с клиентом/или знаем, но она не подтверждена с клиентом; 
  • клиент не пользуется сервисом (нет входов, не задаёт вопросы); 
  • есть ошибки/баги/проблемы, которые блокируют работу клиента с сервисом/его штатное использование. 

1.2. Как актуализируем статусы. 

Если ставим статус "по плану", то указываем: 

  • текущую цель над которой работаем; 
  • план реализации (какие пункты выполнены, какие пункты открыты - дедлайны по данным пунктам); 
  • если есть какие-то важные задачи/детали/нюансы по клиенту, которые в данный момент не блокируют, но потенциально могут повлиять, то отмечаем. 

Если ставим статус "под угрозой", то указываем: 

  • какой из пунктов по статусу "по плану" сейчас не выполняется; 
  • прописываем план проработки; 


ВАЖНО - статусы по проектам обновляются еженедельно до 18:00 понедельника. Подведение статусов проводим каждый вторник в 10:30
