Цель: проверка состояния здоровья клиента с помощью чек листа.

Участники процесса:  руководитель группы

Инициатор: руководитель группы

Инфо поле: Confluence

KPI процесса:

  • Регулярность - 1 раз за 2 недели, по понедельникам.

Результат: результатом процесса является

  1. Отчет о проверке здоровья клиента в профиле клиента в Confluence
  2. В случае проблем со здоровьем клиента - инициированные "лечебные касания" клиента или "лечебный проект".

Как контролируем: Сводные данные в таблице Контроль CS - Карты здоровья 2.0. Ссылка на Контроль CS

Владелец процесса: руководитель направления Коллтрекинг

Статус процесса: эксплуатация.


Описание процесса для Group Head

  1. В карточке клиента в Conflunce добавить карточку здоровья 
  2. Перенести в сводную таблицу Контроль CS. Ссылка на Контроль CS
  3. Запланировать задачи/проекты по повышению значения здоровья.

Описание процесса для руководителя направления

  1. Выборочная проверка 3-4 карты на каждого групп хеда
  2. Актуализировать список клиентов, если появились новые или были передачи
  3. Добавить раздел Карты здоровья в карточку клиента в Confluence
