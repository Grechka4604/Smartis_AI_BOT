Описание проекта. 

Предпосылки.

Сейчас в юните Смартис если клиент использует несколько продуктов из разных направлений, то с ним работает 2 или 3 аккаунт-менеджера/PM.

В процессе работы это влечёт ряд проблем, вот некоторые их них:

  1. сотрудники из разных направлений не знают задачи друг друга;

  2. не понимают как выполняемые задачи влияют на задачи другого направления;

  3. клиент воспринимает нас как набор отдельных специалистов, которые не действуют единой командой.

  4. не работаем на достижение глобальной цели клиента, не знаем, что это за цель.


Objective: Получить опыт работы с клиентом единой командой на примере 10-ти клиентов.

Key Results:

  1. Модель работы в формате единой команды описана, согласована и выложена в conf;

  2. Подтверждена гипотеза: если мы будем работать с клиентом в формате единой команды, то это позволит увеличить его совокупный чек/или по одному из продуктов на 30%. Должно быть тезисно сформулировано как это конкретно влияет;

  3. Модель может быть масштабирована на всю клиентскую базу;

  4. Минимум по 3 клиентам мы можем говорить, что получили позитивный опыт работы в формате единой команды

- оценка клиента;

- сформированный план развития клиента в выручке на 30%;


Как это влияет на верхнеуровневый OKR:

  1. запущенный формат единой команды влияет на восприятие нас как экосистемы с стороны конкретных клиентов/рынка;

  2. для достижения цели 2022 - очень важно сохранить текущий объём выручки, реализация данного проекта повысит LTV текущих клиентов;

  3. работа в единой команде помогает увеличит чек клиента на 30%, по сравнению с клиентами где эта работа не проводится;
