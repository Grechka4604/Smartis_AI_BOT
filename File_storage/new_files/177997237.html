Какую задачу клиента решает:

  • посмотреть, по каким объектам покупают/бронируют недвижимость клиенты, которые обращались/посещали сайт конкретного проекта (в каких жк застройщика покупают квартиру клиенты, которые изначально интересовались ЖК Ромашка)
  • посмотреть, как двигаются по воронке продаж заявки/сессии/звонки или что угодно другое, которые поступают по 1 объекту, а далее переходят в другой (сотрудник/аг-во ведет конкретные ЖК, и их работа приносит целевые лиды/сделки или что-то еще. При этом не все эти целевые события происходят по тем ЖК, которые они ведут. В таком случае часто возникает запрос посмотреть, не уходят ли привлеченные клиенты в другие ЖК данного застройщика)

Для построения такого отчета рассмотрим несколько вариантов.

  1.  Если нужны перетекания из объекта А в другие объекты по сессиям (клиент заходил на сайт проекта А, в какие проекты он совершал визит/бронь/сделку).
    Здесь выбираем нужные метрики + фильтр по странице входа. Строим все в группировке Объект (по CRM) - если нужны только CRM данные, либо в группировке Объекты (если нужны НЕ только CRM данные).


  2. Перетекания по обращениям (звонили по объекту А, в какие проекты он совершал визит/бронь/сделку)
    1. Выбираем нужные метрики: обращения (или целевое обращение) + другие метрики по воронке продаж, которые нам нужны. Строим отчет в группировке Объект - если нужны только CRM данные, либо в группировке Объекты (если нужны НЕ только CRM данные). 
    2. Далее нужно выбрать фильтр по наименованию кампании, в котором будет указан нужный нам ЖК (важно! если клиент не соблюдает разметку и НЕ указывает ЖК в названии кампаний, то фильтрация будет некорректной)



  3. Если нужны перетекания по конкретным РК и площадке аг-ва/сотрудника-маркетолога.
    Разберем на примере запроса: контекстолог ведет кампании в Директе по ЖК Улыбка, нужно посмотреть как по воронке продаж по всем объектам продвигаются клиенты, которые пришли по этим кампаниям.

    1. Выбираем нужные метрики: заявкой с рекламы считается обращения (или целевое обращение) + другие метрики по воронке продаж, которые нам нужны. Строим отчет в группировке Объект - если нужны только CRM данные, либо в группировке Объекты (если нужны НЕ только CRM данные). В качестве площадки выбираем нужную нам - Яндекс Директ
    2. Далее нужно выбрать фильтр по наименованию кампании, в котором будет указан нужный нам ЖК(который ведет этот конкретный директолог). Если вам нужно выбрать кампании аг-ва - то можно выбирать в фильтре по кампаниям метку этого агентства.
    В примере на скриншоте видно, что целевые обращения по кампаниям директолога не ковнертировались в сделки в выбранном периоде.
