Цель создания данного перечня - стандартизация подходов к решению типовых рабочих задач в Smartis (решаем одну и ту же задачу одним инструментом) и решение проблем совместимости (каждый сотрудник может продолжить работу над файлом, созданным другим сотрудником, используя только ПО из нижеприведенного списка).

При работе сотрудники Smartis могут использовать инструменты из этого перечня для выполнения своих рабочих задач (компания предоставляет инструменты из этого списка по мере необходимости).

Инструменты, не входящие в этот перечень, использовать запрещается.

При необходимости использования каких-либо других инструментов вне этого перечня (например, нестандартная задача, требующая другого инструментария), решение о возможности использования инструмента вне этого списка принимает руководитель подразделения, в котором работает сотрудник.


1. Коммуникации

  • Teams 
    • Набор инструментов для собраний, чатов (для чатов в Smartis используем Telegram, чаты в Teams - для переписки с коллегами из других бизнес-юнитов), звонков и совместной работы. 
    • Инструмент для работы с календарем (планирование встреч - нужен регламент).
  • Telegram - инструмент для работы рабочих групп сотрудников, объединяемых вокруг какой-то задачи (работа с определенным клиентом, проектом или задачей), а также для быстрой коммуникации сотрудников.

Ключевые проблемы/инсайты (по опросу)

  • Confluence - неудобство, хаос (на при этом пользуемся и видим ценность).
  • Нет структуры и нет централизованного ответственного за управление знаниями компании. ("Каждый отдел хранит в своем сундучке и так им удобно.")
  • Нет системы развития / переаттестации сотрудников (этим вообще не занимаемся!)
  • Нет доступа к знаниям других отделов / бизнес-юнитов (и это не болит?)
